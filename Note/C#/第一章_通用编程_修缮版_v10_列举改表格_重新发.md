## 通用编程

### 基础格式

```c#
using System;

namespace StudentManagerApp
{
    // 第一章统一示例：StudentManagerApp
    // 这里展示最基础的“程序骨架 + 类型 + 类”的组合（不追求功能完整）
    class Program
    {
        static void Main(string[] args)
        {
            var s = new Student(id: 1, name: "Alice", age: 20);
            Console.WriteLine($"Student => Id={s.Id}, Name={s.Name}, Age={s.Age}");
        }
    }

    class Student
    {
        public int Id;
        public string Name;
        public int Age;

        public Student(int id, string name, int age)
        {
            Id = id;
            Name = name;
            Age = age;
        }
    }
}
```

### 预编译与命名空间

#### 预编译

**基础内容描述**  
预编译指令是在“编译阶段”起作用的开关与标记，它不改变运行时逻辑，而是决定哪些代码会被编译进最终程序集。第一章只需要把它当成“条件包含代码”的机制：用来区分 Debug/Release、打开/关闭某些调试输出、在特定条件下给出 warning/error。

**基础用法列举**  
常见指令包括 `#define/#undef/#if/#elif/#else/#endif` 以及 `#region/#endregion`、`#warning/#error`、`#nullable`。实践上：把调试输出包在 `#if DEBUG` 内，避免发布版携带多余日志；用 `#region` 组织长文件（尤其在学习阶段）。

**示例代码**  
见本节代码块（已统一到 StudentManagerApp 语境或保持指令示例原样）。

​	预处理器指令指导编译器在实际编译开始之前对信息进行预处理。通过这些指令，可以控制编译器如何编译文件或编译哪些部分。常见的预处理器指令包括条件编译、宏定义等。所有的预处理器指令都是以 **#** 开始，且在一行上，只有空白字符可以出现在预处理器指令之前。预处理器指令不是语句，所以它们不以分号 **;** 结束。C# 编译器没有一个单独的预处理器，但是，指令被处理时就像是有一个单独的预处理器一样。在 C# 中，预处理器指令用于在条件编译中起作用。与 C 和 C++ 不同的是，它们不是用来创建宏。一个预处理器指令必须是该行上的唯一指令。通过预编译可以实现：1. **提高代码可读性**：使用`#region`可以帮助分隔代码块，提高代码的组织性。2. **条件编译**：通过`#if`等指令可以在开发和生产环境中编译不同的代码，方便调试和发布。3. **警告和错误**：通过`#warning`和`#error`可以在编译时提示开发人员注意特定问题。

以下是C# 中可用的预处理器指令：

| 指令         | 描述                                                         |
| :----------- | :----------------------------------------------------------- |
| `#define`    | 定义一个符号，可以用于条件编译。                             |
| `#undef`     | 取消定义一个符号。                                           |
| `#if`        | 开始一个条件编译块，如果符号被定义则包含代码块。             |
| `#elif`      | 如果前面的 `#if` 或 `#elif` 条件不满足，且当前条件满足，则包含代码块。 |
| `#else`      | 如果前面的 `#if` 或 `#elif` 条件不满足，则包含代码块。       |
| `#endif`     | 结束一个条件编译块。                                         |
| `#warning`   | 生成编译器警告信息。                                         |
| `#error`     | 生成编译器错误信息。                                         |
| `#region`    | 标记一段代码区域，可以在IDE中折叠和展开这段代码，便于代码的组织和阅读。 |
| `#endregion` | 结束一个代码区域。                                           |
| `#line`      | 更改编译器输出中的行号和文件名，可以用于调试或生成工具的代码。 |
| `#pragma`    | 用于给编译器发送特殊指令，例如禁用或恢复特定的警告。         |
| `#nullable`  | 控制可空性上下文和注释，允许启用或禁用对可空引用类型的编译器检查。 |

**#define** 用于定义符号（通常用于条件编译），**#undef** 用于取消定义符号。这样，通过使用符号作为传递给 #if 指令的表达式，条件指令用于测试符号是否为真。如果为真，编译器会执行 **#if** 和下一个指令之间的代码。一个以 **#if** 指令开始的条件指令，必须显示地以一个 **#endif** 指令终止。

```c#
#define DEBUG

#if DEBUG
    Console.WriteLine("Debug mode");
#elif RELEASE
    Console.WriteLine("Release mode");
#else
    Console.WriteLine("Other mode");
#endif
```

**#warning** 用于生成编译器警告，**#error** 用于生成编译器错误。

```
#warning This is a warning message
#error This is an error message
```

**#region** 和 **#endregion** 用于代码折叠，使代码更加可读。**#line** 用于更改文件行号和文件名的编译器输出。**#pragma** 用于向编译器发送特殊指令。最常见的用法是禁用特定的警告。

```c#
#region MyRegion
    // Your code here
#endregion

#line 100 "MyFile.cs"
    // The next line will be reported as line 100 in MyFile.cs
    Console.WriteLine("This is line 100");
#line default
    // Line numbering returns to normal
    
#pragma warning disable 414
    private int unusedVariable;
#pragma warning restore 414
```

#### 命名空间

**基础内容描述**  
命名空间用于“给类型分组并避免命名冲突”。同名的 `Student` 可以在不同命名空间里共存，这在大型项目里非常关键。第一章把命名空间理解为“文件夹/模块层级”就足够：它帮助你在后续工程化阶段把代码按领域拆分。

**基础用法列举**  
`using Xxx;` 引入命名空间；`namespace StudentManagerApp { ... }` 定义命名空间；可用别名 `using IO = System.IO;` 简化长名；同名冲突时用全限定名 `System.Collections.Generic.List<int>`。


补充：`Main(string[] args)` 的 `args` 用于接收命令行参数（例如 `StudentManagerApp --seed 10`），它让控制台程序从“交互式”扩展到“可脚本化”。需要表达程序是否成功执行时，可以在 Main 里 `return` 一个退出码（0 表示成功，非 0 表示失败）。现代写法可用 `static int Main(string[] args)`。

**示例代码**  
见本节代码块（using/namespace 示例）。

​	**命名空间**的设计目的是提供一种让一组名称与其他名称分隔开的方式。在一个命名空间中声明的类的名称与另一个命名空间中声明的相同的类的名称不冲突。命名空间的定义是以关键字 **namespace** 开始，后跟命名空间的名称。为了调用支持命名空间版本的函数或变量，会把命名空间的名称置于前面。**using** 关键字表明程序使用的是给定命名空间中的名称。该指令告诉编译器随后的代码使用了指定命名空间中的名称。命名空间可以嵌套，使用点（.）运算符访问嵌套的命名空间的成员。

```c#
// 1.a 使用 using 引入命名空间
using System;
using SomeNameSpace;
using SomeNameSpace.Nested;
// 1.b 为命名空间起别名
using Nested = SomeNameSpace.Nested;

// 2. 定义一个命名空间
namespace StudentManagerApp
{
    public class MyClass 
    {
        static void Main() 
        {
            // 3.此时不需要使用完整的路径即可使用 System 下的函数
            Console.WriteLine("In SomeNameSpace");
            Nested.NestedNameSpaceClass.SayHello();
        }
    }

    // 4.内嵌命名空间
    namespace StudentManagerApp   
    {
        public class NestedNameSpaceClass 
        {
            public static void SayHello() 
            {
                Console.WriteLine("In Nested");
            }
        }
    }
}
```

### 类型、变量与常量

#### 类型转换


**基础内容描述**  
类型转换的核心是“把一种表示方式变成另一种表示方式”，转换可能带来信息丢失（例如 double 转 int 截断小数）、溢出（数值范围不够）、或者失败（字符串不是数字）。第一章阶段不追求背全规则，只要建立三个习惯：能避免就避免隐式丢失；用户输入优先 TryParse；把“可能失败”的转换写成可控分支，而不是靠异常当流程。

**基础用法列举**  
显式强转：`(int)doubleValue`，要意识到会截断；需要四舍五入就 `Math.Round` 后再转。数值之间也可用 `Convert.ToInt32/ToDouble` 做更明确的转换。字符串到数字：优先 `int.TryParse` / `double.TryParse`，失败就提示用户重输。对象与字符串：`obj.ToString()` 会走对象的 ToString 覆写；需要格式化输出更常用插值字符串 `$"{x}"`。第一章只把“常见方向”记清：**字符串↔数值、数值↔数值、对象→字符串**。

**示例代码**  
```csharp
using System;

namespace StudentManagerApp
{
    class TypeConversionDemo
    {
        static void Main()
        {
            // 1) 用户输入（字符串）→ 数字：优先 TryParse
            Console.Write("请输入学生年龄：");
            string raw = Console.ReadLine()?.Trim() ?? "";
            if (!int.TryParse(raw, out int age))
            {
                Console.WriteLine("年龄必须是整数。");
                return;
            }

            // 2) 数值↔数值：注意精度与截断
            double scoreAvg = 89.6;
            int truncated = (int)scoreAvg;                 // 89（截断）
            int rounded   = (int)Math.Round(scoreAvg);     // 90（四舍五入）

            // 3) 对象 → 字符串：依赖 ToString（可覆写）
            var s = new Student(1, "Alice", age);
            Console.WriteLine($"Student={s} | truncated={truncated} | rounded={rounded}");
        }
    }

    class Student
    {
        public int Id;
        public string Name;
        public int Age;

        public Student(int id, string name, int age) { Id = id; Name = name; Age = age; }

        public override string ToString() => $"#{Id}-{Name}(Age={Age})";
    }
}
```

#### 类型


**基础内容描述**  
“类型”是 C# 里最底层的规则：它决定一个值在内存里怎么放、能做哪些运算、默认值是什么、以及赋值/传参时是“复制内容”还是“复制引用”。第一章只需要把类型当作“数据的语法”：写对类型，编译器才能帮你挡住大量错误；选对类型，程序的精度/性能/可维护性会更稳。

**基础用法列举**  
最常见的值类型：`int/long`（整数范围不同）、`double`（近似浮点）、`decimal`（十进制高精度，金额常用）、`bool`、`char`、`DateTime`（后面会用到）。最常见的引用类型：`string`、数组、`class`。值类型赋值会复制值；引用类型赋值复制的是“引用”，多个变量可能指向同一个对象。引用类型可以是 `null`，值类型默认不能为 `null`，需要“可空”用 `int?` 这类写法。第一章把这三句话记牢：**值类型复制值、引用类型复制引用、null 只属于引用/可空值类型**。

**示例代码**  
```csharp
using System;

namespace StudentManagerApp
{
    class TypeDemo
    {
        static void Main()
        {
            int a = 20;      // 值类型：复制的是“值”
            int b = a;
            b++;
            Console.WriteLine($"a={a}, b={b}");

            Student s1 = new Student(1, "Alice", 20); // 引用类型：复制的是“引用”
            Student s2 = s1;
            s2.Age++;
            Console.WriteLine($"s1.Age={s1.Age}, s2.Age={s2.Age}");

            int? maybeAge = null; // 可空值类型：允许没有值
            Console.WriteLine(maybeAge ?? -1);
        }
    }

    class Student
    {
        public int Id;
        public string Name;
        public int Age;
        public Student(int id, string name, int age) { Id=id; Name=name; Age=age; }
    }
}
```


**变量类型**：**值类型、引用类型、指针类型**

**值类型**：值类型变量可以直接分配给一个值。它们是从类 **System.ValueType** 中派生的。值类型直接包含数据。比如 **int、char、float**，它们分别存储数字、字符、浮点数。当您声明一个 **int** 类型时，系统分配内存来存储值。如需得到一个类型或一个变量在特定平台上的准确尺寸，可以使用 **sizeof** 方法。表达式 *sizeof(type)* 产生以字节为单位存储对象或类型的存储尺寸。

```c#
// 1. 使用sizeof获取数据类型长度
Console.WriteLine("Size of int: {0}", sizeof(int));
```

**引用类型**：引用类型指的是一个内存位置，它不包含存储在变量中的实际数据，但它们包含对变量的引用。如果内存位置的数据是由一个变量改变的，其他变量会自动反映这种值的变化。**内置的** 引用类型有：**object**、**dynamic** 和 **string**。

```c#
// 1.object 类型
object obj;
obj = 100; // 这是装箱
// 2. dynamic 类型
dynamic d = 20; // 运行时推测类型
// 3. string 类型
string str = @"C:\Windows"; // 逐字处理""内的字符
string str = "C:\\Windows"; // 转义处理\后的字符
string str = @"<script type=""text/javascript"">
    <!--
    -->
</script>"; //逐字符后允许随意换行表示字符串
```

**指针类型**：指针类型变量存储其他变量类型的内存地址。本质上引用类型是扩展了部分操作的指针类型。

**类型转换**：将一个数据类型的值转换为另一个数据类型的过程。可以分为两种：**隐式类型转换**和**显式类型转换**（也称为强制类型转换）。隐式转换是不需要编写代码来指定的转换，编译器会自动进行。例如，从 int 到 long，从 float 到 double 等。从小的整数类型转换为大的整数类型，从派生类转换为基类。将一个 byte 类型的变量赋值给 int 类型的变量，编译器会自动将 byte 类型转换为 int 类型，不需要显示转换。显式类型转换，即强制类型转换，需要程序员在代码中明确指定，强制转换会造成数据丢失。本身有许多方法提供了类型转换，例如ToType()，Convert，Parse。

```c#
// 0. 强制转换
double doubleValue = 3.14;
int intValue = (int)doubleValue; // 强制从 double 到 int，数据可能损失小数部分
// 1. Convert 类提供了一组静态方法，可以在各种基本数据类型之间进行转换。
string str = "123";
int num = Convert.ToInt32(str);
// 2. Parse 方法用于将字符串转换为对应的数值类型，如果转换失败会抛出异常。
string str = "123.45";
double d = double.Parse(str);
// 3. TryParse 方法类似于 Parse，但它不会抛出异常，而是返回一个布尔值指示转换是否成功。
string str = "123.45";
double d;
bool success = double.TryParse(str, out d);

if (success) {
    Console.WriteLine("转换成功: " + d);
} else {
    Console.WriteLine("转换失败");
}

// 4. C# 还允许你定义自定义类型转换操作，通过在类型中定义 implicit 或 explicit 关键字。
using System;
public class Fahrenheit
{
    public double Degrees { get; set; }

    public Fahrenheit(double degrees)
    {
        Degrees = degrees;
    }

    // 隐式转换从Fahrenheit到double
    public static implicit operator double(Fahrenheit f)
    {
        return f.Degrees;
    }

    // 显式转换从double到Fahrenheit
    public static explicit operator Fahrenheit(double d)
    {
        return new Fahrenheit(d);
    }
}
public class Program
{
    public static void Main()
    {
        Fahrenheit f = new Fahrenheit(98.6);
        Console.WriteLine("Fahrenheit object: " + f.Degrees + " degrees");

        double temp = f; // 隐式转换
        Console.WriteLine("After implicit conversion to double: " + temp + " degrees");

        Fahrenheit newF = (Fahrenheit)temp; // 显式转换
        Console.WriteLine("After explicit conversion back to Fahrenheit: " + newF.Degrees + " degrees");
    }
}
```

#### 变量

**基础内容描述**  
变量是“有名字的存储位置”。第一章关注两点：作用域（在哪能用）与生命周期（什么时候存在）。局部变量在方法内创建与销毁；字段跟随对象或类型（static）存在。稳定的变量命名与作用域控制会直接影响可读性。

**基础用法列举**  
声明：`int age;` 初始化：`int age = 20;` 多变量：`int a=1,b=2;` 作用域：`{}` 内有效；推荐：变量尽量“离使用点近”，避免超大作用域变量被多处改写。


补充理解：局部变量的作用域只在最近的一对 `{}` 内有效，出了块就不可见；字段的生命周期跟随对象实例（或类型的 static 字段跟随整个程序生命周期）。习惯上把变量尽量声明在“最靠近使用处”的地方，能减少误改与阅读负担。


补充最小代码组织习惯：一段逻辑超过 20~30 行就考虑拆方法；复杂 if/switch 分支在“为什么要这么判断”的地方加一句短注释；常量（如年龄上限）不要散落在多处，集中为 `const` 或 `static readonly`，避免改一次漏一处；命名优先表达含义（`studentById` 比 `dict1` 可读）。

**示例代码**

**变量定义**：`<data_type> <variable_list>;`在这里，data_type 必须是一个有效的 C# 数据类型，可以是 char、int、float、double 或其他用户自定义的数据类型。variable_list 可以由一个或多个用逗号分隔的标识符名称组成。变量的命名需要遵循一些规则：1. 变量名可以包含字母、数字和下划线。2. 变量名必须以字母或下划线开头。3. 变量名区分大小写。4. 避免使用 C# 的关键字作为变量名。

**变量作用域**：1. 在方法、循环、条件语句等代码块内声明的变量是局部变量，它们只在声明它们的代码块中可见。2. 块级作用域，即使用大括号 **{}** 创建的任何块都可以定义变量的作用域。3. 方法的参数也有其自己的作用域，它们在整个方法中都是可见的。4. 在类的成员级别定义的变量是成员变量，它们在整个类中可见，如果在命名空间级别定义，那么它们在整个命名空间中可见。5. 静态变量是在类级别上声明的，但它们的作用域也受限于其定义的类。

#### 常量

**基础内容描述**

#### 调试与排错

**基础内容描述**  
调试的目标不是“看程序跑没跑”，而是把程序的状态在关键时刻“停下来并看清楚”。第一章阶段只要掌握一套稳定流程：先复现问题（输入什么会错），再定位问题发生的具体行（断点/异常提示），再观察关键变量值（监视/局部变量/调用栈），最后用最小修改验证修复是否生效。以 StudentManager 为例，最常见的错误来自输入解析失败、集合状态不一致（List 与 Dictionary 没同步）、以及分支走错（switch/if 条件写反或漏了 default）。

**基础用法列举**  
在 IDE（VS / Rider / VS Code + C# 扩展）里通常用到：断点（Breakpoint）、继续运行（Continue）、逐语句执行（Step Over/F10）、单步进入（Step Into/F11）、跳出当前方法（Step Out）、监视表达式（Watch）、局部变量窗口（Locals）、调用堆栈（Call Stack）。排错时优先在“状态变化点”打断点，例如 `AddStudent()` 里写入集合之前/之后、`RemoveStudent()` 里移除之前/之后、`TryParse` 失败分支里。对于异常：先看异常类型与消息，再看堆栈最上方属于自己代码的那一行；不要直接在 catch 里吞掉异常，否则会让定位更难。

**示例代码**  
下面的代码不需要额外写“调试代码”，调试动作是在运行时完成的。建议按注释提示在指定行打断点，观察变量与集合状态变化（Students.Count / StudentById.Count / id/name/age）。

#### 方法

**基础内容描述**  
方法是“把一段可重复的逻辑封装成可调用单元”的工具，它决定了代码能否被组织得清晰、可复用、可测试。第一章阶段把方法掌握到“会拆函数、会传参、会返回值、会复用”的程度即可：输入进来是参数，处理过程在方法内部完成，输出出去是返回值；如果需要把结果写回到外层变量，就用 `ref/out`（但第一章只要求理解与会用，不追求滥用）。

**基础用法列举**  
方法签名由“访问修饰符 + 返回类型 + 方法名 + 参数列表”组成；`void` 表示不返回值。常见参数形态包括：普通参数、可选参数（带默认值）、命名参数（调用时 `name: value`）、`ref/out` 参数（把变量引用传入，让方法写回）。方法还支持重载：同名但参数列表不同。工程里更常见的实践是：重载用在“同一语义、不同输入形态”的场景；如果只是为了省事让一个方法承担多种职责，通常会让代码变难读。

**示例代码**  
下面的方法全部服务于 StudentManagerApp：用统一的输入模板读数字/文本，用单独的方法完成添加、删除、查找等操作。注意：第一章先追求“清晰”，不追求花哨。

```csharp
using System;
using System.Collections.Generic;

namespace StudentManagerApp
{
    class Program
    {
        static readonly List<Student> Students = new();
        static readonly Dictionary<int, Student> StudentById = new();

        static void Main()
        {
            while (true)
            {
                Console.WriteLine("\n1) Add  2) Remove  3) Find  4) List  0) Exit");
                int cmd = ReadInt("Command: ");

                switch (cmd)
                {
                    case 1: AddStudent(); break;
                    case 2: RemoveStudent(); break;
                    case 3: FindStudent(); break;
                    case 4: ListStudents(); break;
                    case 0: return;
                    default: Console.WriteLine("Unknown command."); break;
                }
            }
        }

        static void AddStudent()
        {
            int id = ReadInt("Id: ");
            string name = ReadText("Name: ", allowEmpty: false);
            int age = ReadInt("Age: ", min: 0, max: 150);

            // 复用：把校验拆成方法，避免 AddStudent 太长
            if (!ValidateNewStudent(id, name, age, out var error))
            {
                Console.WriteLine(error);
                return;
            }

            var s = new Student(id, name, age);
            Students.Add(s);
            StudentById[id] = s;

            Console.WriteLine("Added.");
        }

        static bool ValidateNewStudent(int id, string name, int age, out string error)
        {
            // out 参数：用于把错误信息“写回”给调用方
            if (StudentById.ContainsKey(id))
            {
                error = "Id already exists.";
                return false;
            }
            if (string.IsNullOrWhiteSpace(name))
            {
                error = "Name cannot be empty.";
                return false;
            }
            if (age < 0 || age > 150)
            {
                error = "Age out of range.";
                return false;
            }

            error = "";
            return true;
        }

        static void RemoveStudent()
        {
            int id = ReadInt("Id to remove: ");
            if (!StudentById.TryGetValue(id, out var s))
            {
                Console.WriteLine("Not found.");
                return;
            }

            StudentById.Remove(id);
            Students.Remove(s);
            Console.WriteLine("Removed.");
        }

        static void FindStudent()
        {
            int id = ReadInt("Id to find: ");
            if (StudentById.TryGetValue(id, out var s))
                Console.WriteLine(s);
            else
                Console.WriteLine("Not found.");
        }

        static void ListStudents()
        {
            if (Students.Count == 0)
            {
                Console.WriteLine("(empty)");
                return;
            }

            foreach (var s in Students)
                Console.WriteLine(s);
        }

        static int ReadInt(string prompt, int? min = null, int? max = null)
        {
            while (true)
            {
                Console.Write(prompt);
                string? t = Console.ReadLine();

                if (!int.TryParse(t, out var v))
                {
                    Console.WriteLine("Invalid integer.");
                    continue;
                }

                if (min.HasValue && v < min.Value) { Console.WriteLine($"Too small (min={min})."); continue; }
                if (max.HasValue && v > max.Value) { Console.WriteLine($"Too large (max={max})."); continue; }

                return v;
            }
        }

        static string ReadText(string prompt, bool allowEmpty = true)
        {
            while (true)
            {
                Console.Write(prompt);
                var t = (Console.ReadLine() ?? "").Trim();
                if (allowEmpty || t.Length > 0) return t;
                Console.WriteLine("Value cannot be empty.");
            }
        }
    }

    record Student(int Id, string Name, int Age);
}
```

#### 输入输出与解析

**基础内容描述**  
控制台程序的“交互稳定性”主要取决于两件事：输入是否被可靠地解析，输出是否清晰地表达状态。输入解析的核心问题是：文本到类型的转换（`TryParse`）、空值处理（`ReadLine()` 可能返回 null）、以及格式容错（用户可能输入空格、中文、混合字符）。输出表达的核心问题是：能否用一致的提示语把“成功/失败/下一步要做什么”讲清楚，避免用户迷路。

**基础用法列举**

| 项目 | 列举 |
|---|---|
| 读取输入 | `Console.ReadLine()`；输出：`Console.WriteLine()` / `Console.Write()`；解析：`int.TryParse` / `double.TryParse` / `DateTime.TryParse`。对字符串做预处理：`Trim()`、判空 `string.IsNullOrWhiteSpace`。对结构化输出：字符串插值 `$"Id={id}"`、格式化 `{0}`。当输入不可解析时，优先走“提示 + 重试”的正常路径，而不是 `try/catch`。 |
| 补充 | 写控制台小程序时经常需要生成测试数据或随机行为（例如“猜数字”）。可使用 `Random`：`var rng = new Random(); int x = rng.Next(min, max);`。注意随机数对象通常复用同一个实例，不要在短循环里反复 `new Random()`（可能产生相同种子导致分布异常）。在 StudentManager 里可用随机 id/name/age 快速填充数据，便于测试遍历、查找与删除逻辑。 |

补充输出：控制台输出常用两种写法：插值字符串 `$"Id={id}"` 可读性高；占位符 `Console.WriteLine("Id={0}", id)` 更老派但也常见。输入解析建议始终 `Trim()` 处理空格，并统一错误提示语，避免交互混乱。

**示例代码**  
仍然复用 StudentManagerApp 的输入模板。下面补充一个常见的“输入一行并拆分”的场景：例如一次性输入 `id,name,age`，用分隔符解析成三个字段（第一章只需要会写，不追求完美）。

```csharp
// 仍在 StudentManagerApp 中
static bool TryReadStudentLine(out int id, out string name, out int age)
{
    Console.Write("Input student (id,name,age): ");
    var line = (Console.ReadLine() ?? "").Trim();

    id = 0; name = ""; age = 0;

    if (line.Length == 0) return false;

    var parts = line.Split(',');
    if (parts.Length != 3) return false;

    if (!int.TryParse(parts[0].Trim(), out id)) return false;
    name = parts[1].Trim();
    if (!int.TryParse(parts[2].Trim(), out age)) return false;

    if (string.IsNullOrWhiteSpace(name)) return false;
    if (age < 0 || age > 150) return false;

    return true;
}
```

### 集合（List / Dictionary 入门）

#### List/Dictionary 入门

**基础内容描述**  
数组适合“长度固定”的数据，但管理系统类的小程序往往需要动态增删，因此第一章就需要认识 `List<T>` 与 `Dictionary<TKey, TValue>`。`List<T>` 负责“有序的一组元素”，擅长遍历与按顺序展示；`Dictionary<TKey, TValue>` 负责“键到值的映射”，擅长按 id 快速查找。把两者组合使用，就能同时获得“列表展示的顺序性”和“查找的效率”，这也是你后面做后端/游戏时非常常见的数据组织方式。

**基础用法列举**

| 集合类型 | 常用成员/操作 | 记忆点与注意事项 |
|---|---|---|
| `List<T>` | `Add`、`Insert`、`Remove`、`RemoveAt`、`Contains`、`IndexOf`、`Sort`、`Reverse`、`Count`、`Clear`、`foreach` 遍历。 | `List.Add` 是追加到末尾，`Insert(index, item)` 会把元素插到指定位置并整体移动后续元素；`Remove(item)` 按值移除首次匹配项，`RemoveAt(index)` 按下标移除。 |
| `Dictionary<TKey,TValue>` | `Add`、`Remove`、`ContainsKey`、`TryGetValue`、索引器 `dict[key]`（仅在 key 必存在时）、`Keys/Values`、`Count`。 | `Dictionary` 访问 key 时，`dict[key]` 在 key 不存在会抛异常，因此交互程序优先 `TryGetValue` 或 `ContainsKey`。 |
| 组合使用习惯 | — | 增删改时保持两者同步；查找用 `TryGetValue`；展示用 `List` 的遍历；不要用异常处理“找不到 key”这种可预期情况。 |


**示例代码**

    {

        return false;
    }

    StudentById[s.Id] = s;

    error = "";
    return true;
}

static bool TryRemoveStudent(int id)
{

        return false;

### 异常与异常处理

#### 异常处理

**基础内容描述**  
异常处理是“兜底机制”：当出现不可预期或无法继续的错误时，用异常把错误信息带出去。第一章要区分：输入错误是正常路径（TryParse），真正异常才用 try/catch。并知道 finally 的用途：释放资源、记录日志、恢复状态。

**基础用法列举**  
`try { ... } catch (FormatException ex) { ... } finally { ... }`；重抛 `throw;`；自定义异常后面工程化再学。第一章只要会捕获常见异常并输出信息即可。

异常是在程序执行期间出现的问题。C# 中的异常是对程序运行时出现的特殊情况的一种响应，比如尝试除以零。异常提供了一种把程序控制权从某个部分转移到另一个部分的方式。C# 异常处理时建立在四个关键词之上的：**try**、**catch**、**finally** 和 **throw**。**try**块标识了一个将被激活的特定的异常的代码块，后跟一个或多个 catch 块。**catch**块在程序通过异常处理程序捕获异常。**finally**块用于执行给定的语句，不管异常是否被抛出都会执行。**throw**块用于在问题出现时，抛出一个异常。

```c#
using System;

namespace StudentManagerApp
{
    class ExceptionDemo
    {
        static void Main()
        {
            try
            {
                int id = int.Parse("not-an-int"); // 故意制造错误
                Console.WriteLine(id);
            }
            catch (FormatException ex)
            {
                Console.WriteLine("Input format error: " + ex.Message);
            }
            finally
            {
                Console.WriteLine("finally: cleanup/log here");
            }
        }
    }
}
```

#### 异常


**基础内容描述**  
异常（Exception）是运行时错误的“对象化结果”：除了错误消息，还包含异常类型与调用堆栈，用于定位问题。把异常理解成“程序遇到意外情况时的报警单”更直观：它会一路向上冒泡，直到被某一层 catch 住，或者直接导致程序终止。第一章要建立的直觉是：异常不是普通分支；输入校验要走返回值（TryXxx），真正意外才让异常传播。

**基础用法列举**  
常见异常类型：`FormatException`（格式不对，常见于 Parse）、`ArgumentException/ArgumentOutOfRangeException`（参数不合法）、`InvalidOperationException`（当前状态不允许操作）、`NullReferenceException`（空引用访问，尽量通过判空与短路避免）。异常对象常用属性：`Message`（人能读的原因）、`StackTrace`（定位线索）。重抛时用 `throw;` 保留原堆栈；`throw ex;` 会丢失关键堆栈信息（第一章只要知道不要这么写即可）。

**示例代码**  
```csharp
using System;

namespace StudentManagerApp
{
    class ExceptionDemo
    {
        static void Main()
        {
            try
            {
                // 模拟：把用户输入当作年龄解析（可能失败）
                string raw = "abc";
                int age = int.Parse(raw); // 触发 FormatException
                Console.WriteLine(age);
            }
            catch (FormatException ex)
            {
                Console.WriteLine($"输入格式错误：{ex.Message}");
                // 需要定位时再输出堆栈
                // Console.WriteLine(ex.StackTrace);
            }
        }
    }
}
```


​	C# 异常是使用类来表示的。C# 中的异常类主要是直接或间接地派生于 **System.Exception** 类。**System.ApplicationException** 和 **System.SystemException** 类是派生于 System.Exception 类的异常类。**System.ApplicationException** 类支持由应用程序生成的异常。所以程序员定义的异常都应派生自该类。**System.SystemException** 类是所有预定义的系统异常的基类。如果异常是直接或间接派生自 **System.Exception** 类，则可以抛出一个对象。可以在 catch 块中使用 throw 语句来抛出当前的对象。

下表列出了一些派生自 System.SystemException 类的预定义的异常类：

| 异常类                            | 描述                                           |
| :-------------------------------- | :--------------------------------------------- |
| System.IO.IOException             | 处理 I/O 错误。                                |
| System.IndexOutOfRangeException   | 处理当方法指向超出范围的数组索引时生成的错误。 |
| System.ArrayTypeMismatchException | 处理当数组类型不匹配时生成的错误。             |
| System.NullReferenceException     | 处理当依从一个空对象时生成的错误。             |
| System.DivideByZeroException      | 处理当除以零时生成的错误。                     |
| System.InvalidCastException       | 处理在类型转换期间生成的错误。                 |
| System.OutOfMemoryException       | 处理空闲内存不足生成的错误。                   |
| System.StackOverflowException     | 处理栈溢出生成的错误。                         |
