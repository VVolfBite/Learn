# Authentication vs Authorization

> 认证回答“你是谁”，授权回答“你能做什么”。这两个问题经常连续出现，但它们绝不是一回事。

## 背景与作用

认证是确认请求方身份的过程，比如判断一个请求是否真的来自某位用户或某个服务。授权则是在身份被确认之后，决定这个身份是否有权访问某个资源、执行某个操作。一个用户可以认证成功，但仍然没有足够权限执行某个敏感动作。

如果没有认证，系统连请求来自谁都无法确定，谈不上后续控制。如果只有认证没有授权，系统虽然知道来的是谁，却可能让任何已登录用户随意访问本不该访问的数据。它们通常一起出现，是因为大多数业务都先要识别身份，再要控制访问范围。

## 场景与边界

这是安全基础里最经典的区分题。面试官借它看候选人是否真的理解安全边界。工作里一旦设计后台管理、分角色功能、多租户系统或服务间调用，这个区分就会变得非常具体：登录成功不等于能看所有数据，也不等于能执行所有操作。

最常见的误区是把“登录”当成全部安全控制。登录通常只完成认证，还远远没完成授权。另一个误区是只在页面层做授权。真正的授权检查应当落实在服务和资源访问边界上，否则换一条请求路径就可能绕过去。

## 面试表达

认证是确认身份，授权是决定权限。认证通过之后，系统仍然需要判断这个身份能访问哪些资源、执行哪些操作。很多业务里两者会连续出现，但职责不同：前者解决“是谁”，后者解决“能做什么”。

## 快速回忆

- Authentication：你是谁。
- Authorization：你能做什么。
- 登录成功不代表有全部权限。
- 授权检查要落在真正的资源边界。
