# API Security 与 Web Security - 高频判断

## 身份与权限

- 认证回答“你是谁”，授权回答“你能做什么”。
- Cookie、Session、Token 不是同一层概念。
- JWT 方便传递身份，但不代表适合所有场景。

## 传输与浏览器机制

- HTTPS 保护传输，不替代应用层安全。
- CORS 是浏览器跨源机制，不是后端鉴权。
- CSP 是附加防线，不是主防线。

## 输入与请求风险

- 外部输入默认不可信。
- XSS 和 SQL 注入都和“把数据当指令”有关。
- CSRF 利用的是浏览器自动带凭证。

## 整体治理

- OAuth 2.0 偏授权，OIDC 偏认证。
- OWASP 更像风险地图，不是死记目录。
- API 安全是系统性边界治理，不是单点配置。
