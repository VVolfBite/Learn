# HTTPS and TLS

> HTTPS 的意义不是给网址前面加一把锁，而是让通信在传输途中更难被偷看、篡改和冒充。

## 背景与作用

HTTPS 可以理解成运行在 TLS 保护之上的 HTTP。TLS 负责在通信双方之间建立受保护的连接，提供机密性、完整性和一定程度的身份认证；HTTP 仍然负责请求和响应的语义。两者结合后，客户端和服务端之间的内容就不再像明文 HTTP 那样暴露在链路上。

HTTPS 主要缓解三类风险：别人偷看你传输了什么，别人中途改写你传输的内容，以及你连上的到底是不是声称的那个服务。对于登录、支付、个人信息、管理后台等场景，这些都是最基本的要求。没有 HTTPS，再好的登录方案和接口设计都可能在链路上被削弱。

## 场景与边界

HTTPS 是后端和运维共同关注的基础设施能力。面试会问，是因为它太基础也太容易被泛化。工作里更现实：证书部署、反向代理、内部服务访问、混合内容和跳转策略都会和 HTTPS 有关。理解它解决了什么，也能帮助你理解它没有解决什么。

最常见的误区是把 HTTPS 当成“系统已经安全”的证明。HTTPS 解决的是传输安全，不解决授权错误、业务逻辑漏洞、注入攻击和越权访问。另一个误区是只记握手名词，不知道它的业务意义。对复习而言，更重要的是把 HTTPS 和机密性、完整性、身份认证三件事对应起来。

## 面试表达

HTTPS 是在 HTTP 之上通过 TLS 提供传输保护的方式。它主要让通信内容更难被窃听和篡改，同时帮助客户端确认自己连到的服务身份。它非常重要，但它解决的是传输层面的安全，不是应用层所有安全问题的总开关。

## 快速回忆

- HTTPS = HTTP + TLS。
- 核心价值是机密性、完整性、身份认证。
- 它很基础，但不是万能盾牌。
- 传输安全和应用安全是两层问题。
