# API Security 与 Web Security - 目录

这一组笔记讨论的是后端最常见、也最容易被误解的安全基础。它不是安全岗手册，也不追求覆盖所有攻防细节，而是帮助后端开发建立最稳定的安全心智：身份是怎么确认的，请求为什么会被冒用，浏览器为什么会限制跨域，为什么 HTTPS 很重要却又不是万能盾牌。

模块包含 Security Basics、Authentication vs Authorization、Session Cookie Token、JWT、OAuth 2.0、OpenID Connect、HTTPS and TLS、CORS、CSRF、XSS SQL Injection and Input Risks、CSP、OWASP Risks、API Security Best Practices，以及最后的高频判断。阅读顺序建议先看身份与传输安全，再看浏览器相关风险，最后回到整体风险框架和 API 设计实践。

这一组没有把 Basic Auth、2FA、SAML 单独拆成文件，但会在身份体系相关卡片里顺带交代它们的定位：Basic Auth 适合理解最基础的 HTTP 认证思路，2FA 强调“登录不只看密码”，SAML 则更多出现在企业级单点登录与身份联邦场景中。
