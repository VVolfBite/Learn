# CI_CD 基础与 Testing - 高频判断

## 测试主线

- 测试的核心价值是降低改动风险。
- 单元测试快，集成测试更接近真实协作。
- 功能测试关注业务闭环，不只是 UI。

## 分层与替身

- 分层是成本分配，不是教条模板。
- Mock、Stub、Fake 都是替身，但侧重点不同。
- 过度 mock 会让测试失真。

## CI / CD

- CI 是自动化质量门禁，不只是跑测试。
- CD 关注稳定交付，不等于自动直接上线。
- 流水线分阶段是为了更早定位问题。

## 总体判断

- 没有测试，系统越复杂越不敢改。
- 没有 CI/CD，验证和上线都会越来越依赖人工经验。
- 真正稳的工程实践，靠的是持续的小反馈，而不是上线前一次性大检查。
