# CI_CD 基础与 Testing - 目录

这一组笔记讨论的是后端工程里最基础的一层质量保障：怎样验证改动没有把系统弄坏，怎样把验证过程变成稳定、可重复的流水线。它不写具体平台命令，而是建立一种更重要的判断：测试为什么分层，CI 为什么是门禁，CD 为什么不是“按一下就上线”。

模块包含 Testing Basics、Unit Testing、Integration Testing、Functional Testing、Mock Stub Fake、Testing Boundaries and Layers、CI Basics、CD Basics、Common Pipeline Stages，以及最后的高频判断。建议先建立测试层次感，再看 CI/CD 如何把这些检查串起来。
