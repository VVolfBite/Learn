## 存储层次

### 必要性

**理想的存储器:**和 CPU 一样快、容量无限大、成本为零。

**现实:**速度快的(寄存器、SRAM)成本极高容量极小,容量大的(硬盘)速度极慢。单一存储器无法满足所有需求。

**解决方案:分层设计**

```
        寄存器 (最快,最小,最贵)
          ↑ ALU 直接操作
       L1 Cache
          ↑ 缓存最热数据
       L2 Cache
          ↑ 缓存热点数据
       L3 Cache
          ↑ 缓冲 CPU 和内存
      主存(RAM)
          ↑ 运行中的程序
     外存(SSD/HDD) (最慢,最大,最便宜)
```

**核心思想:**快的存储器容量小,只放热点数据;慢的存储器容量大,存放全部数据。利用局部性原理,让大部分访问命中快速存储器。

**效果**：用户感觉容量大(接近硬盘)速度快(接近 Cache),实际成本远低于全用快速存储器。因为程序访问数据具有局部性。虽然 Cache 只缓存千分之一的数据,但这千分之一恰好是最热的数据,覆盖了 90%-99% 的访问。

**设计原则:**
- 相邻层次速度差距不能太大(通过 Cache 缓冲 CPU 和内存)
- 容量逐层递增,成本逐层递减
- 上层内容是下层的子集(寄存器数据来自 Cache,Cache 来自内存,内存来自硬盘)

### 层次结构

**寄存器**在 CPU 内部,是最快的存储单元。访问时间 0.3-1 纳秒,容量只有几十到几百字节。x86-64 有 16 个通用寄存器,每个 8 字节,总共约 128 字节。寄存器用触发器实现,成本极高,占用芯片面积,所以数量有限。但配合 Cache 已经够用。**Cache** 也在 CPU 内部,分 L1、L2、L3 三级。L1 最快(1-2 纳秒),容量最小(32-64 KB),分指令 Cache 和数据 Cache,每个核心独享。L2 稍慢(3-10 纳秒),容量稍大(256-512 KB),每个核心独享。L3 相对慢(10-20 纳秒),容量较大(8-32 MB),所有核心共享。Cache 用 SRAM 实现,不需要刷新,对程序员透明,硬件自动管理。作用是缓解 CPU 和内存的速度差距。

**主存(内存)**在 CPU 外部,通过总线连接。访问时间 50-100 纳秒,容量 4GB-64GB,断电数据丢失。用 DRAM 实现,电容存储会漏电,需要定期刷新。内存存储运行中的程序和数据,是 CPU 和硬盘的桥梁。

**外存(硬盘)**是独立设备,通过 I/O 总线连接。机械硬盘(HDD)访问时间 5-10 毫秒,容量 1TB-10TB,顺序读写快但随机读写慢。固态硬盘(SSD)访问时间 50-150 微秒,容量 256GB-4TB,随机读写性能好。外存断电数据保留,用于长期存储文件和程序。

### 性价对比

访问速度差距巨大。寄存器 1 纳秒,L1 Cache 2-4 倍慢,L2 Cache 10-20 倍慢,L3 Cache 20-40 倍慢,内存 100-200 倍慢,SSD 慢 5 万到 15 万倍,HDD 慢 500 万到 1000 万倍。

假设访问寄存器需要 1 秒,那访问 L1 Cache 要 2-4 秒,L2 Cache 要 10-20 秒,L3 Cache 要 20-40 秒,内存要 1.5-3 分钟,SSD 要约 1 天,HDD 要约 2-3 个月。

容量每向下一层增长约 10-1000 倍。寄存器几百字节,L1 Cache 几十 KB,L2 Cache 几百 KB,L3 Cache 几十 MB,内存几十 GB,SSD/HDD 几 TB。从寄存器到硬盘,容量增长约 10^11 倍。

成本每向下一层降低约 10-100 倍。SRAM(Cache)约 $10,000/GB,DRAM(内存)约 $5-10/GB,SSD 约 $0.1-0.3/GB,HDD 约 $0.02-0.05/GB。从 Cache 到硬盘,单位成本降低约 10 万倍。

| 存储层次 | 速度 | 容量 | 成本 | 易失性 | 位置 | 管理方式 |
|----------|------|------|------|--------|------|----------|
| 寄存器 | 最快 | 最小 | 最高 | 易失 | CPU 内 | 编译器/程序员 |
| L1 Cache | 很快 | 很小 | 很高 | 易失 | CPU 内 | 硬件自动 |
| L2 Cache | 快 | 小 | 高 | 易失 | CPU 内 | 硬件自动 |
| L3 Cache | 较快 | 较小 | 较高 | 易失 | CPU 内 | 硬件自动 |
| 内存 | 中等 | 中等 | 中等 | 易失 | CPU 外 | 操作系统 |
| SSD | 慢 | 大 | 低 | 非易失 | 独立设备 | 操作系统 |
| HDD | 很慢 | 很大 | 很低 | 非易失 | 独立设备 | 操作系统 |

### 工作流程

程序启动时,存储在硬盘上。双击运行,操作系统把程序加载到内存。CPU 从内存读取指令,指令被缓存到 L3、L2、L1 Cache。CPU 从 L1 Cache 读取指令到指令寄存器,然后执行。

读取变量 x 时,CPU 先检查寄存器有没有,没有就查 L1 Cache,还没有就查 L2、L3,都没有才从内存读取。x 被加载到 L3、L2、L1 Cache,最后加载到寄存器。CPU 用寄存器中的 x 进行运算。再次读取 x 时,直接从寄存器拿,无需访问 Cache 或内存。

修改变量 x 时,CPU 在寄存器中修改,写回 L1 Cache(标记为脏),稍后写回 L2、L3 Cache,最终写回内存。如果需要持久化,操作系统将内存数据写回硬盘。

各层之间有包含关系:硬盘存所有数据,内存存运行中的程序和数据(是硬盘的子集),L3 Cache 存最近访问的数据(是内存的子集),L2 Cache 存更热的数据(是 L3 的子集),L1 Cache 存最热的数据(是 L2 的子集),寄存器存正在运算的数据(是 L1 的子集)。上层修改数据后需要同步到下层,使用写直达或写回策略。多核 CPU 用 MESI 协议保证 Cache 一致性。

为什么这样设计有效?因为程序访问数据具有局部性。时间局部性:最近访问的数据很可能再次访问。空间局部性:访问某个地址后很可能访问相邻地址。虽然只有少量数据在 Cache 中,但大部分访问都命中 Cache,典型命中率 90%-99%。

假设 L1 Cache 命中率 95%,访问时间 2ns,未命中访问内存 100ns。平均访问时间 = 0.95 × 2ns + 0.05 × 100ns = 6.9ns。相比直接访问内存(100ns),提升约 14 倍!

成本效益也很明显。假设需要 16GB 存储,全用 SRAM 要 $160,000。分层设计:32MB L3 Cache $320 + 16GB 内存 $160 = $480。成本降低约 333 倍,性能接近全 SRAM!

## 高频判断

**存储层次中,离 CPU 越近,速度越快** → 对。寄存器最快,然后是 Cache、内存、硬盘。

**存储层次中,离 CPU 越近,容量越大** → 错。离 CPU 越近容量越小。寄存器最小,硬盘最大。

**存储层次中,离 CPU 越近,成本越高** → 对。寄存器和 Cache 用 SRAM,成本极高;硬盘成本最低。

**Cache 的内容是内存的子集** → 对。Cache 缓存内存中的热点数据,是内存的子集。

**寄存器比 L1 Cache 快** → 对。寄存器约 0.3-1 纳秒,L1 Cache 约 1-2 纳秒。

**内存比 SSD 快** → 对。内存约 50-100 纳秒,SSD 约 50-150 微秒,内存快约 1000 倍。

**存储层次的设计利用了局部性原理** → 对。局部性原理保证大部分访问命中快速存储器,使分层设计有效。

**L3 Cache 比 L1 Cache 容量大但速度慢** → 对。L3 约 8-32MB,L1 约 32-64KB;L3 约 10-20ns,L1 约 1-2ns。

**所有 CPU 核心共享 L1 Cache** → 错。L1 和 L2 每个核心独享,L3 所有核心共享。

**硬盘是易失性存储** → 错。硬盘是非易失性存储,断电数据保留。寄存器、Cache、内存是易失性存储。

**访问寄存器比访问内存快约 100 倍** → 对。寄存器约 1 纳秒,内存约 50-100 纳秒,快约 50-100 倍。

**存储层次的目的是实现"又快又大又便宜"** → 对。通过分层设计,用户感觉容量大、速度快、成本低。

**CPU 可以直接访问硬盘** → 错。CPU 只能直接访问寄存器、Cache 和内存,不能直接访问硬盘。

**内存容量越大,程序运行越快** → 不完全对。内存充足可以减少页面置换,但不会直接提升 CPU 运算速度。速度主要受 Cache 影响。

**SRAM 比 DRAM 快但贵** → 对。SRAM 用于 Cache,速度快但成本高;DRAM 用于内存,速度较慢但成本低。

**存储层次中,上层的数据一定在下层存在** → 对。这是包含性原则,寄存器数据来自 Cache,Cache 数据来自内存,内存数据来自硬盘。

**HDD 比 SSD 慢但容量大成本低** → 对。HDD 访问时间毫秒级,SSD 微秒级;但 HDD 单位成本更低,容量可以更大。

**存储层次的每一层都由硬件自动管理** → 错。寄存器由编译器分配,Cache 由硬件管理,内存和硬盘由操作系统管理。
