## 核心主线：程序如何执行

理解计算机组成原理的关键是理解程序执行的完整流程。

**第一步：程序装入内存**  
程序存储在硬盘上，运行前需要加载到内存。CPU 不能直接访问硬盘，必须通过内存。

**第二步：CPU 取指执行**  
CPU 从内存读取指令，经过取指、译码、执行三个阶段。PC（程序计数器）指向下一条指令，IR（指令寄存器）存储当前指令。

**第三步：数据在存储层次间流动**  
寄存器（最快最小）→ Cache（缓冲热点数据）→ 内存（运行中的程序）→ 硬盘（长期存储）。离 CPU 越近越快越小越贵。

**第四步：中断处理**  
CPU 执行过程中可能被中断打断，处理紧急事件（如 I/O 完成），然后恢复执行。

**为什么 Cache 有效？**  
因为程序访问具有局部性：刚访问的数据很可能再次访问（时间局部性），相邻的数据很可能被访问（空间局部性）。Cache 命中率通常在 90%-99%。

**数据如何表示？**  
整数用补码（统一加减运算），小数用浮点数（科学计数法，有精度限制），字符用 ASCII/Unicode。

```
硬盘中的程序 
  ↓ 加载
内存中的程序(代码+数据)
  ↓ CPU 读取
Cache 缓存热点指令和数据
  ↓ 加载到
寄存器中的操作数
  ↓ 送入
ALU 执行运算
  ↓ 结果写回
寄存器 → Cache → 内存 → 硬盘
```

## 正式主稿（按学习顺序）

### 第一部分：程序执行主线

**[1. 计算机如何工作](计算机组成原理%20-%20计算机如何工作.md)**  
面试问法："程序在计算机里是怎么执行的？"  
核心：五大部件、程序加载、指令执行循环  
关键点：CPU 不能直接访问硬盘，必须通过内存

**[2. CPU 与指令执行](计算机组成原理%20-%20CPU与指令执行.md)**  
面试问法："CPU 是如何执行指令的？"  
核心：取指→译码→执行，流水线技术  
关键点：PC 存储下一条指令地址，IR 存储当前指令

### 第二部分：存储体系主线

**[3. 寄存器](计算机组成原理%20-%20寄存器.md)**  
面试问法："寄存器和内存有什么区别？"  
核心：CPU 内部，最快，容量最小  
关键点：ALU 只能操作寄存器中的数据

**[4. RAM](计算机组成原理%20-%20RAM.md)**  
面试问法："内存的作用是什么？"  
核心：CPU 和硬盘的桥梁，存储运行中的程序  
关键点：易失性，断电数据丢失

**[5. CPU Cache](计算机组成原理%20-%20CPU_Cache.md)**  
面试问法："Cache 为什么快？为什么有效？"  
核心：缓解 CPU 和内存速度差距，利用局部性原理  
关键点：L1/L2/L3 三级结构，命中率决定性能

**[6. 存储层次](计算机组成原理%20-%20存储层次.md)**  
面试问法："为什么需要存储层次？"  
核心：速度、容量、成本的平衡，实现"又快又大又便宜"  
关键点：离 CPU 越近越快越小越贵

**[7. 局部性原理](计算机组成原理%20-%20局部性原理.md)**  
面试问法："什么是局部性原理？为什么重要？"  
核心：时间局部性 + 空间局部性，Cache 有效性的基础  
关键点：顺序访问数组 vs 随机访问链表

**[8. 中断](计算机组成原理%20-%20中断.md)**  
面试问法："中断是什么？为什么需要中断？"  
核心：暂停当前程序处理紧急事件，提高 CPU 利用率  
关键点：外中断（异步）vs 内中断（同步）

### 第三部分：数据表示主线

**[9. 二进制基础](计算机组成原理%20-%20二进制基础.md)**  
面试问法："为什么计算机使用二进制？"  
核心：物理实现简单，抗干扰能力强  
关键点：进制转换，位运算技巧

**[10. 补码](计算机组成原理%20-%20补码.md)**  
面试问法："为什么计算机使用补码？"  
核心：统一加减运算，消除负零，硬件简化  
关键点：负数补码 = 反码 + 1

**[11. 浮点数基础](计算机组成原理%20-%20浮点数基础.md)**  
面试问法："浮点数为什么会有误差？"  
核心：精度有限，无法精确表示某些十进制数  
关键点：0.1 + 0.2 ≠ 0.3

### 第四部分：面试冲刺

**[12. 组成原理高频判断](计算机组成原理%20-%20高频判断.md)**  
180+ 道高频判断题  
易混淆点对比（寄存器 vs 内存 vs 磁盘、Cache vs 主存）  
面试前快速过一遍

## 常见误区

**误区 1：把组成原理当硬件课学**  
错误：深入学习 CPU 设计、电路原理  
正确：理解基本概念，能回答面试问题

**误区 2：只背概念不理解关系**  
错误：寄存器是什么、Cache 是什么...  
正确：它们之间的关系？为什么需要它们？

**误区 3：忽视高频对比**  
错误：单独学每个概念  
正确：重点掌握对比（寄存器 vs 内存、原码 vs 补码）

**误区 4：不串主线**  
错误：碎片化知识点  
正确：能串起"程序如何执行"的完整流程
