# 可观测性基础 - 高频判断

## 日志越多越好吗

不一定。日志太少，问题看不清；日志太多，关键线索会被噪音淹没，检索成本和存储成本也会一起上升。更重要的是关键位置有没有留下可检索、可关联的信息。

## 指标、日志、链路追踪各自擅长什么

日志擅长还原事件细节，指标擅长看趋势和告警，链路追踪擅长把一次请求跨服务的路径串起来。真正的线上排障很少只靠其中一种。

## Request ID 和 Correlation ID 是一回事吗

经常会一起用，但侧重点不同。Request ID 更偏一次请求链路内的唯一标识，Correlation ID 更适合跨同步与异步边界去关联同一个业务动作。

## 告警为什么不能越多越好

因为人真正能及时响应的注意力是有限的。大量无效告警只会让团队习惯性静音，最后把真正严重的问题也淹掉。
