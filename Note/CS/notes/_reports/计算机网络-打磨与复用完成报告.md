### 2. 改为自然说明
**改进示例**：

**改前**：
```
**本章核心概念**：
1. **分层思想**：降低复杂度、标准化接口、提高灵活性
2. **数据传输**：封装（发送端）、解封装（接收端）

**必须掌握**：
- 为什么要分层？（降低复杂度、标准化、灵活性）
- 数据如何在网络中传输？（逐层封装和解封装）
```

**改后**：
```
网络协议栈分为五层：应用层、传输层、网络层、链路层、物理层。每一层解决不同的问题，向上层提供服务。

数据从应用层向下传输时，每一层都会添加自己的头部信息（封装）；到达目的地后，再逐层去掉头部（解封装）。
```

### 3. 用自然段代替列表
**改前**：
```
**分层的好处**：
1. **降低复杂度**：将复杂问题分解为多个简单问题
2. **易于维护**：每层独立，修改某一层不影响其他层
3. **标准化**：每层有明确的接口和协议标准
4. **灵活性**：可以针对不同需求选择不同的协议实现
```

**改后**：
```
分层的核心是降低复杂度。应用层只需要关心如何表达数据（HTTP、DNS），传输层关心如何可靠传输（TCP），网络层关心如何跨网络路由（IP），链路层关心如何在相邻节点间传输（以太网），物理层关心如何传输比特流。

面试常问为什么要分层，答案就是：降低复杂度、标准化接口、提高灵活性。
```

## 五、补充的最值钱内容

### 1. 传输时延 vs 传播时延的自然说明
**补充位置**：网络基础文档

**补充内容**：
```
容易混淆的是传输时延和传播时延。传输时延是数据"进入"链路的时间，传播时延是信号"在"链路上传播的时间。增加带宽可以减少传输时延，但无法减少传播时延；缩短距离可以减少传播时延，但无法减少传输时延。
```

**价值**：这是面试高频易混淆点，用自然语言说清楚比列表更容易记住

### 2. 传输层 vs 网络层的自然说明
**补充位置**：分层模型文档

**补充内容**：
```
传输层和网络层容易混淆。传输层是进程到进程的通信，使用端口号；网络层是主机到主机的通信，使用 IP 地址。传输层关心可靠传输和流量控制，网络层关心路由选择和分组转发。
```

**价值**：这是面试高频易混淆点，直接说明比表格对比更自然

### 3. 路由器为什么只需要三层
**补充位置**：分层模型文档

**补充内容**：
```
路由器只有下面三层（物理、链路、网络），因为它只需要根据 IP 地址转发数据包。交换机只有下面两层（物理、链路），因为它只需要根据 MAC 地址转发帧。

面试常问为什么路由器只需要三层，答案就是：路由器的功能是路由选择和分组转发，只需要工作在网络层及以下，不需要为进程提供服务（传输层）或运行应用程序（应用层）。
```

**价值**：这是面试高频问题，直接给出答案比分点列举更实用

### 4. 分组交换为什么更好
**补充位置**：网络基础文档

**补充内容**：
```
现代互联网使用分组交换，因为数据传输具有突发性，分组交换能更好地利用网络资源。
```

**价值**：直接说明原因，比列举优缺点更清楚
