核心原因:ALU 直接连接寄存器,不连接内存。

```

Cache 速度接近寄存器(2-20ns),容量比寄存器大得多(几 MB)。
```

## 三、如何把二进制/补码/浮点数和 CPU 计算联系起来

### 1. 二进制 → CPU 只能处理二进制

**修改位置：二进制基础.md**

原文只说"物理实现简单"，现在明确联系到 CPU：

```
为什么 CPU 只能处理二进制?因为 CPU 内部是电路,电路只有两种稳定状态:
高电平(1)和低电平(0)。

所有数据(整数、小数、字符、图片)在计算机中都必须转换成二进制,CPU 才能运算。

布尔代数只有真(1)和假(0),与、或、非等逻辑运算可以直接用电路实现。
CPU 内部的 ALU 就是用这些逻辑门电路搭建的。
```

### 2. 补码 → 简化 CPU 的 ALU 设计

**修改位置：补码.md**

原文说"统一加减运算"，现在明确联系到 ALU：

```
减法可以转换为加法:a - b = a + (-b)。
这意味着 CPU 的 ALU 只需要实现加法器,不需要单独的减法器。

使用原码:
  ALU 需要:加法器、减法器、符号判断电路、绝对值比较电路

使用补码:
  ALU 只需要:加法器

减法转换为加法,硬件大大简化,成本降低,速度更快!
这就是为什么所有现代 CPU 都使用补码表示有符号整数。
```

### 3. 浮点数 → CPU 的浮点寄存器和 FPU

**修改位置：浮点数基础.md**

原文只说"精度有限"，现在明确联系到 CPU 硬件：

```
为什么 CPU 需要浮点数?因为整数只能表示整数,无法表示小数、很大的数、很小的数。
CPU 需要处理科学计算、图形渲染等任务,必须支持浮点运算。
现代 CPU 都有专门的浮点运算单元(FPU)。

CPU 浮点寄存器位数有限。单精度 32 位,尾数 23 位,约 7 位十进制精度。
双精度 64 位,尾数 52 位,约 15-16 位十进制精度。
CPU 的 FPU(浮点运算单元)只能在这个精度范围内计算。

经典问题:0.1 + 0.2 ≠ 0.3。
因为 0.1 和 0.2 在 CPU 浮点寄存器中都无法精确表示,
存储时已经有舍入误差,FPU 执行加法后误差累积,结果不等于 0.3。
```

### 2. Cache 为什么快

**修改位置：高频判断.md**

```
物理距离近。Cache 在 CPU 内部,与 ALU 直接相连。

访问简单。Cache 访问无需复杂的地址转换。
容量小查找快。
```

```

```

### 6. 为什么需要存储层次

**修改位置：存储层次.md**

强化核心矛盾：

```
理想的存储器:和 CPU 一样快、容量无限大、成本为零。
现实:速度快的(寄存器、SRAM)成本极高容量极小,容量大的(硬盘)速度极慢。

解决方案:分层设计。快的存储器容量小,只放热点数据;
慢的存储器容量大,存放全部数据。
```

### 7. 局部性原理为什么重要

**修改位置：局部性原理.md**

强调 Cache 有效性的基础：

```
Cache 的有效性完全依赖于局部性原理。
Cache 容量很小(几 MB),内存很大(几 GB),Cache 只能缓存内存的千分之一。
如果程序访问完全随机,Cache 命中率会很低,Cache 就没用了。

但实际上程序访问具有局部性,虽然 Cache 只缓存千分之一的数据,
但这千分之一恰好是最热的数据,覆盖了 90%-99% 的访问。

局部性原理是整个存储层次设计的基础。没有局部性,Cache 就没有意义,
CPU 性能会下降 10-100 倍。
```

### 8. 中断是什么/为什么需要中断

**修改位置：中断.md**

量化 CPU 利用率提升：

```
没有中断,CPU 在等待 I/O 的 5ms 内本可以执行 500 万条指令!
有了中断,CPU 利用率从不到 1% 提升到接近 100%。

这就是为什么所有现代操作系统都基于中断机制。
```
