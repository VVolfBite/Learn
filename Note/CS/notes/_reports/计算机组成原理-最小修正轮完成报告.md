### 1. 压缩了"CPU 基础"的机械分点

**修改位置：CPU与指令执行.md**

**原文结构：**
```
## CPU 基础
### 1. CPU 的组成

- 执行逻辑运算(与或非、比较)

...

```

**修改后：**
```
## CPU 基础

CPU(中央处理器)是计算机的核心,负责执行程序指令。内部主要包括:

运算器(ALU):执行加减乘除、与或非等运算。ALU 只能操作寄存器中的数据,不能直接对内存运算。

控制器:程序计数器(PC)记录下一条指令在哪,指令寄存器(IR)存放当前指令...

CPU 的主要工作就是不断重复"取指→译码→执行"循环。
```

**效果：**
- 去掉了"1/2/3"式的机械编号
- 去掉了"定义/特点/作用"式的分点
- 直接说 CPU 是什么、有什么、做什么
- 最后一句点明主线:"取指→译码→执行"循环

### 2. 简化了指令执行三阶段的描述

**修改位置：CPU与指令执行.md**

**原文：**每个阶段都有"目标"、"详细步骤"、"1/2/3/4/5/6"式的分点

**修改后：**
```
### 2. 取指阶段 — 从内存读取指令
PC 中存着下一条指令的地址,假设 PC = 0x1000。CPU 把地址发送到地址总线,
同时发送"读"控制信号。内存收到信号后,从地址 0x1000 读取指令...

### 3. 译码阶段 — 解析指令含义
指令译码器分析 IR 中的指令,提取操作码确定操作类型,提取操作数确定数据位置...

### 4. 执行阶段 — ALU 运算并写回
从 R2 和 R3 读取数据到 ALU,ALU 执行加法运算,产生结果并更新状态标志...
```

**效果：**
- 去掉"目标"、"详细步骤"等模板化标题
- 用自然的段落描述流程
- 标题直接说明这个阶段在做什么

### 3. 简化了"CPU、内存、硬盘"关系的描述

**修改位置：计算机如何工作.md**

**原文：**每个部件都用粗体标题 + 分点列表

**修改后：**
```
CPU 是计算机的"大脑",内部有寄存器(最快)、Cache(高速缓存)、ALU(运算单元)...

内存是 CPU 和硬盘之间的桥梁。为什么需要内存?因为硬盘太慢(毫秒级),
CPU 等不起;寄存器太小(几百字节),放不下完整程序。

硬盘负责长期保存。容量最大(TB 级),速度最慢,但断电数据还在。

总线是连接各部件的公共通道。数据总线传输数据(双向)...
```

**效果：**
- 去掉粗体标题的强调
- 用自然的段落串联
- 直接说明为什么需要这些部件
