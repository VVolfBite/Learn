```
硬盘中的程序 → 内存中的程序 → Cache缓存 → 寄存器 → ALU运算

**存储体系主线:**

```

### 2. 强化因果关系

**在目录页明确:**
- 为什么需要内存? → 硬盘太慢,CPU 太快
- 为什么需要 Cache? → 内存仍比 CPU 慢 50-100 倍
- 为什么需要寄存器? → ALU 只能操作寄存器数据
- 为什么 Cache 有效? → 程序访问具有局部性

##  三、补充的高频问法

### 程序执行类(3 个)
1. **"程序在计算机里是怎么执行的?"**
   - 提供 1 分钟简洁版和 3 分钟详细版答案
   - 关键点: CPU 不能直接访问硬盘

2. **"CPU 是如何执行指令的?"**
   - 取指→译码→执行三阶段
   - 关键点: PC 存储下一条指令地址

3. **"为什么需要内存?"**
   - CPU 和硬盘的桥梁
   - 速度差距: 硬盘毫秒级,CPU 纳秒级

### 存储体系类(4 个)
4. **"寄存器、内存、磁盘有什么区别?"**
   - 7 维度对比表(位置、速度、容量、易失性等)

5. **"Cache 为什么快?"**
   - 4 个原因: 物理距离近、使用 SRAM、容量小、访问简单

6. **"Cache 为什么有效?"**
   - 局部性原理 + 命中率计算(90%-99%)

7. **"为什么需要存储层次?"**
   - 反证法: 为什么不全用寄存器/内存/硬盘
   - 用数据说明: 成本降低 1000 倍,速度提升 14 倍

### 原理解释类(2 个)
8. **"什么是局部性原理?"**
   - 时间局部性 + 空间局部性
   - 为什么重要: Cache 有效性的基础

9. **"中断是什么?为什么需要中断?"**
   - 3 个原因: 提高 CPU 利用率、避免轮询、实现多任务

### 数据表示类(3 个)
10. **"为什么计算机使用二进制?"**
    - 物理实现简单,抗干扰能力强

11. **"为什么计算机使用补码?"**
    - 3 个优势: 统一加减运算、消除负零、硬件简化

12. **"浮点数为什么会有误差?"**
    - 0.1 + 0.2 ≠ 0.3 的原因
    - 解决方案: 使用误差范围比较
