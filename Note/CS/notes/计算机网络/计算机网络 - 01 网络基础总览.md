## 网络分层

​	网络协议栈分为五层：应用层、传输层、网络层、链路层、物理层。每一层解决不同的问题，向上层提供服务。想象一下，如果没有分层，一个应用程序要发送数据，就得自己处理所有事情：怎么把数据变成电信号、怎么在网线上传输、怎么找到目标主机、怎么保证数据不丢失、怎么让对方应用程序理解这些数据。这太复杂了。

​	分层的核心是降低设计复杂度。应用层只需要关心如何表达数据（HTTP、DNS），传输层关心如何可靠传输（TCP），网络层关心如何跨网络路由（IP），链路层关心如何在相邻节点间传输（以太网），物理层关心如何传输比特流。每一层只做好自己的事，通过标准化的接口与上下层交互。这样的分层设计让每一层可以独立演进。比如可以在同一个 IP 网络上运行不同的应用协议，也可以在不同的物理介质上运行相同的网络协议。HTTP 不需要关心底层是光纤还是无线网络，IP 协议也不需要关心上层是 HTTP 还是 FTP。

## 数据如何传输

​	浏览器访问网页时，数据会经历一个封装和解封装的过程。

​	**发送端自上而下封装**：应用层生成 HTTP 请求 → 传输层添加 TCP 头部（包含端口号）→ 网络层添加 IP 头部（包含 IP 地址）→ 链路层添加 MAC 头部（包含 MAC 地址）→ 物理层转换为比特流发送。

​	**接收端自下而上解封装**：物理层接收比特流 → 链路层去掉 MAC 头部 → 网络层去掉 IP 头部 → 传输层去掉 TCP 头部 → 应用层处理 HTTP 请求。

​	每一层只能看到自己的头部信息，这就是层次化封装的体现。传输层看到的是端口号，网络层看到的是 IP 地址，链路层看到的是 MAC 地址。路由器只有下面三层（物理、链路、网络），因为它的工作是根据 IP 地址转发数据包。它不需要知道数据包里装的是什么应用数据，也不需要关心 TCP 连接的状态。交换机更简单，只有下面两层（物理、链路），根据 MAC 地址转发帧。

## 两种交换方式

网络中有两种交换方式：电路交换和分组交换。

​	**电路交换**类似打电话。通信前先建立一条专用线路，通信过程中独占这条线路，通信结束后释放。这种方式的好处是延迟小、质量稳定，但问题是线路利用率很低。想象一下打电话时的沉默时刻，线路还是被占用着，但实际上没有传输任何数据。统计显示，电路交换的线路利用率往往不到 10%。

​	**分组交换**把数据分成一个个小的数据包，每个数据包独立传输。同一条线路可以同时传输多个数据包，就像高速公路上同时跑着很多车。路由器收到数据包后先存储，然后转发到下一跳，这叫存储转发。分组交换的优点是线路利用率高，特别适合突发性数据传输。你发邮件的时候，数据瞬间发出去，然后线路就可以给别人用了。缺点是存在排队时延（数据包在路由器排队等待转发），网络拥塞时可能丢包。现代互联网使用分组交换，因为数据传输具有突发性。浏览网页、看视频、发消息，都是一阵一阵的，不需要一直占用线路。

## 网络时延的组成

​	数据从发送到接收，总时延由四部分组成：**总时延 = 排队时延 + 处理时延 + 传输时延 + 传播时延**

​	**排队时延**：数据包在路由器队列中等待的时间。这是最不确定的部分，网络拥塞时会显著增加。就像高速公路堵车，你不知道要等多久。**处理时延**：路由器处理数据包的时间，包括分析头部、查找路由表、差错检验。现代路由器处理速度很快，通常只有微秒级。**传输时延**：数据包从主机进入链路所需的时间，等于数据包长度除以传输速率（带宽）。比如 1MB 的文件，在 100Mbps 的网络上，传输时延是 1MB / 100Mbps = 0.08 秒。注意，这与距离无关，只与带宽有关。**传播时延**：信号在链路上传播所需的时间，等于链路长度除以传播速度（接近光速，约 2×10⁸ m/s）。比如北京到上海 1000 公里，传播时延约 5 毫秒。这与带宽无关，只与距离有关。

​	容易混淆的是传输时延和传播时延。传输时延是数据"进入"链路的时间，传播时延是信号"在"链路上传播的时间。打个比方，传输时延是车队通过收费站的时间（车队越长、收费站越窄，时间越长），传播时延是车在高速公路上行驶的时间（距离越远，时间越长）。增加带宽可以减少传输时延，但无法减少传播时延；缩短距离可以减少传播时延，但无法减少传输时延。这就是为什么光纤入户能提高下载速度（减少传输时延），但访问国外网站还是慢（传播时延无法改变）。
