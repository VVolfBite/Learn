## OSI 七层模型

​	OSI（Open System Interconnection）模型是国际标准化组织（ISO）提出的理论模型，从上到下分为七层：应用层、表示层、会话层、传输层、网络层、数据链路层、物理层。

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/0fa6c237-a909-4e2a-a771-2c5485cd8ce0.png" width="450"/> </div><br>

​	OSI模型试图完整描述网络通信的所有功能，所以分得很细。上面三层（应用层、表示层、会话层）都是为应用程序服务的。应用层提供具体的网络服务（HTTP、DNS），表示层处理数据格式转换和加密，会话层管理会话连接。实际应用中这三层往往合并在一起，因为很多功能可以在应用层一起实现。比如HTTPS就把加密（表示层功能）和HTTP（应用层功能）合在一起了。

**各层的作用**：
- **应用层**：为应用程序提供网络服务（HTTP、FTP、SMTP、DNS）
- **表示层**：数据格式转换、加密解密、压缩解压
- **会话层**：建立、管理、终止会话
- **传输层**：端到端的可靠传输（TCP、UDP）
- **网络层**：路由选择和跨网络传输（IP）
- **数据链路层**：相邻节点间的可靠传输（以太网、PPP）
- **物理层**：传输比特流（电信号、光信号）

**OSI模型的问题**：概念清晰但过于复杂，实际应用中很少严格按照七层实现。面试时记住七层的名称和顺序即可，重点在传输层和网络层。

## TCP/IP 四层模型

TCP/IP是实际使用的模型，分为四层：应用层、传输层、网络层（互联网层）、网络接口层。

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/48d79be8-085b-4862-8a9d-18402eb93b31.png" width="250"/> </div><br>

TCP/IP模型是在互联网发展过程中自然形成的，更注重实用性。它把OSI的上三层合并为应用层，把下两层合并为网络接口层，保留了最核心的传输层和网络层。

**各层的作用**：
- **应用层**：对应OSI的应用层+表示层+会话层，提供各种网络服务
- **传输层**：对应OSI的传输层，提供端到端通信（TCP、UDP）
- **网络层**：对应OSI的网络层，提供主机间通信（IP、ICMP、ARP）
- **网络接口层**：对应OSI的数据链路层+物理层，处理硬件细节

**TCP/IP模型的优点**：更简洁实用，是互联网的基础，所有互联网协议都基于这个模型。你每天用的HTTP、TCP、IP都是TCP/IP协议族的一部分。

## 五层协议

五层协议是为了学习方便而提出的折中模型，结合了OSI的清晰层次和TCP/IP的实用性。从上到下：应用层、传输层、网络层、数据链路层、物理层。五层模型保留了OSI模型中最重要的五层，去掉了表示层和会话层（这两层的功能通常在应用层实现），同时把物理层和数据链路层分开（便于理解硬件和软件的分工）。

**各层的数据单位**：
- **应用层**：报文（Message）
- **传输层**：报文段（Segment，TCP）或用户数据报（Datagram，UDP）
- **网络层**：分组（Packet）或数据报（Datagram）
- **数据链路层**：帧（Frame）
- **物理层**：比特流（Bit）

**传输层和网络层的区别**：

- **传输层**：进程到进程的通信，使用端口号，关心可靠传输和流量控制
- **网络层**：主机到主机的通信，使用IP地址，关心路由选择和分组转发

打个比方，网络层负责把数据送到主机，传输层负责把数据送到进程。

## OSI 与 TCP/IP 的关系

OSI是理论模型，TCP/IP是实际模型。两者的对应关系：

**OSI七层 → TCP/IP四层**：
- 应用层 + 表示层 + 会话层 → 应用层
- 传输层 → 传输层
- 网络层 → 网络层
- 数据链路层 + 物理层 → 网络接口层

OSI模型是先设计后实现，追求完美但过于复杂；TCP/IP模型是先实现后总结，追求实用但层次不够清晰。实际工程中，大家都用TCP/IP协议，但讨论时会借用OSI的层次划分。

## 数据在各层的传递

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/0fa6c237-a909-4e2a-a771-2c5485cd8ce0.png" width="450"/> </div><br>

**发送端自上而下封装**：

​	应用层生成报文 → 传输层添加TCP/UDP头部（包含源端口、目的端口）→ 网络层添加IP头部（包含源IP、目的IP）→ 链路层添加MAC头部和尾部（包含源MAC、目的MAC）→ 物理层转换为比特流。每一层都会添加自己的头部信息，这叫封装。就像寄快递，你先把东西装进盒子（应用层），快递员加上快递单（传输层），分拣中心加上路由标签（网络层），最后装上卡车（链路层和物理层）。

**接收端自下而上解封装**：

​	物理层接收比特流 → 链路层去掉MAC头尾 → 网络层去掉IP头部 → 传输层去掉TCP/UDP头部 → 应用层处理报文。每一层只能看到自己的头部信息，这就是层次化封装的体现。传输层看到的是端口号，网络层看到的是IP地址，链路层看到的是MAC地址。

**路由器和交换机的层次**：

- **路由器**：只有下面三层（物理、链路、网络），因为它只需要根据IP地址转发数据包
- **交换机**：只有下面两层（物理、链路），因为它只需要根据MAC地址转发帧

**高频问题**：为什么路由器只需要三层？

答案：路由器的功能是路由选择和分组转发，只需要工作在网络层及以下。它不需要为进程提供服务（传输层的工作），也不需要运行应用程序（应用层的工作）。如果路由器要处理传输层和应用层，性能会大幅下降。
