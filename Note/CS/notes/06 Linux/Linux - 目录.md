# Linux

> 这一组笔记面向后端学习和复习，目标不是把命令手册抄一遍，而是建立一条够用的主线：文件系统怎么组织，权限怎么控制，Shell 怎么串命令，进程怎么运行，网络问题怎么排查。

## 这一组怎么读

Linux 内容如果只是按命令一个个记，很快就会变成碎片。更好的阅读顺序通常是先理解文件系统和权限，因为它们决定了“文件是什么、谁能碰它、为什么删不掉或进不去”；再去看 Shell、重定向和文本处理，因为这部分直接对应日常操作；之后再进入进程管理和网络排障，这两块最贴近后端实际工作里的问题定位。

你可以把这一组理解成一条从“对象是什么”到“问题怎么查”的链路。文件和目录回答系统如何组织内容，inode 和 block 回答文件在底层如何落盘，权限回答谁能做什么，Shell 和重定向回答如何高效组合命令，进程管理回答程序怎样运行，网络诊断则回答服务为什么不通。

## 知识地图

### 文件系统基础

- **[文件与目录](Linux%20-%20文件与目录.md)**：理解路径、目录结构和常见文件类型，先建立“Linux 如何组织文件”的整体感觉。
- **[inode、block 与文件系统](Linux%20-%20inode、block%20与文件系统.md)**：这是后端面试里的高频点，重点不是背定义，而是知道元信息和真正内容分别放在哪里。
- **[硬链接与软链接](Linux%20-%20硬链接与软链接.md)**：理解“多个名字指向什么”这件事，很多删除、移动、链接行为都和这里有关。

### 权限与身份

- **[用户、用户组与权限](Linux%20-%20用户、用户组与权限.md)**：核心是弄清 rwx 在文件和目录上的含义并不完全一样，以及删除、进入目录、执行文件分别看什么权限。

### Shell 与命令操作

- **[Shell 与重定向](Linux%20-%20Shell%20与重定向.md)**：这篇帮助你理解标准输入输出、重定向和管道，知道命令为什么能像积木一样串起来。
- **[常用命令与文本处理](Linux%20-%20常用命令与文本处理.md)**：聚焦后端常见场景，如查看日志、过滤信息、统计和简单压缩备份，而不是做命令百科。

### 进程与运行时

- **[进程管理](Linux%20-%20进程管理.md)**：重点看进程状态、查看方式、信号以及僵尸进程和孤儿进程的区别。
- **[进程间通信基础](Linux%20-%20进程间通信基础.md)**：建立 IPC 的入门认知，知道管道、信号、共享内存和套接字分别适合什么问题。

### 网络排障

- **[网络诊断与端口排查](Linux%20-%20网络诊断与端口排查.md)**：把服务“不通”这件事拆成连通性、监听状态、端口占用和进程归属几个角度去看。

### 高频判断

- **[高频判断](Linux%20-%20高频判断.md)**：面试前快速翻看的一页，帮助把前面零散概念重新收束成可直接回答的判断句。

## 面试里最常被追问什么

Linux 面试很少要求把命令背全，更常见的是围绕几个高频问题来追问：inode 和 block 分别存什么；软链接和硬链接的本质区别是什么；目录权限为什么会影响删除文件；僵尸进程和孤儿进程怎么区分；看到端口占用或服务不通时应该先查什么。把这些问题读通，Linux 这组就已经足够支撑大部分后端入门场景。

## 快速回忆

- Linux 不要按命令记，要按“文件—权限—Shell—进程—网络”这条主线记。
- 文件系统解决“内容怎么组织”，权限解决“谁能操作”，进程和网络解决“程序怎么跑、问题怎么查”。
- 目录页只负责导航，具体概念和高频判断放到对应卡片里看。
