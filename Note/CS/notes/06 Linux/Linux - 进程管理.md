## 进程状态（高频）

Linux 进程有以下几种状态：

**R（Running）**：正在运行或就绪（等待 CPU）

**S（Sleeping）**：可中断睡眠，等待事件（如 I/O），可被信号唤醒

**D（Disk Sleep）**：不可中断睡眠，通常在等待 I/O，不能被信号中断

**T（Stopped）**：停止状态，收到 SIGSTOP 信号，可用 SIGCONT 恢复

**Z（Zombie）**：僵尸进程，已结束但父进程未回收，占用 PID

**面试重点**：
- R 和 S 是正常状态
- D 通常很短暂，长时间 D 状态说明 I/O 有问题
- Z 是问题状态，需要处理

## 查看进程

### ps - 进程快照

```bash
# 查看所有进程
ps aux                    # BSD 格式（推荐）
ps -ef                    # System V 格式

# 查找特定进程
ps aux | grep java        # 查找 Java 进程
ps aux | grep nginx       # 查找 Nginx 进程

# 查看进程树
ps auxf                   # 显示进程树
```

**输出解释**：
```
USER  PID %CPU %MEM  VSZ   RSS TTY STAT START TIME COMMAND
root    1  0.0  0.1 225868 9208 ?  Ss   Jan01 0:03 /sbin/init
```

- **PID**：进程 ID
- **%CPU**：CPU 使用率
- **%MEM**：内存使用率
- **STAT**：进程状态（R/S/D/Z/T）
- **COMMAND**：命令

### top - 实时监控

```bash
top                       # 实时查看
top -p 1234               # 查看指定进程
```

**常用快捷键**：
- `M`：按内存排序
- `P`：按 CPU 排序
- `k`：杀死进程
- `q`：退出

## 进程控制

### kill - 发送信号

这一节主要把“kill - 发送信号”放回“Linux - 进程管理”这条主线里看。先抓住它在整个Linux知识体系中的位置，再去看后面的分类和结论，会更容易把零散的判断串起来。

```bash
kill PID                  # 发送 SIGTERM（15），优雅退出
kill -9 PID               # 发送 SIGKILL（9），强制杀死
kill -STOP PID            # 停止进程
kill -CONT PID            # 继续进程
```

**常用信号**：
- **SIGTERM（15）**：终止信号，进程可以捕获并优雅退出（推荐）
- **SIGKILL（9）**：杀死信号，进程无法捕获，立即终止（慎用）
- **SIGSTOP（19）**：停止信号，进程无法捕获
- **SIGCONT（18）**：继续信号，恢复停止的进程

**kill vs kill -9**：
- `kill`：优雅退出，进程可以清理资源
- `kill -9`：强制杀死，进程无法清理资源
- **原则**：优先使用 `kill`，只有在进程无响应时才用 `kill -9`

## 僵尸进程（高频考点）

### 什么是僵尸进程

**定义**：子进程已结束，但父进程未回收。

**产生原因**：
1. 子进程调用 `exit()` 结束
2. 内核保留退出状态等信息
3. 父进程没有调用 `wait()` 或 `waitpid()` 回收
4. 子进程变成僵尸进程

### 僵尸进程的特点

这一节主要把“僵尸进程的特点”放回“Linux - 进程管理”这张卡片的主线里看。先抓住它在整个Linux知识体系中的位置，再看后面的分点和对比，会更容易把零散结论串起来。

这一节主要把“僵尸进程的特点”放回“Linux - 进程管理”这条主线里看。先抓住它在整个Linux知识体系中的位置，再去看后面的分类和结论，会更容易把零散的判断串起来。

- **占用 PID**：进程号被占用，不能分配给新进程
- **不占用其他资源**：不占用 CPU、内存
- **状态为 Z**：`ps aux` 中显示为 `Z`
- **无法被 kill**：已经死了，无法再杀死
- **只能等父进程回收**：父进程调用 `wait()` 回收

### 如何处理僵尸进程

**方法 1：杀死父进程**
```bash
# 查看僵尸进程的父进程
ps aux | grep Z
ps -o ppid= -p 僵尸进程PID

# 杀死父进程
kill 父进程PID
# 僵尸进程会被 init（PID 1）收养并回收
```

**方法 2：修复父进程代码**（根本解决）
```c
// 父进程应该调用 wait() 回收子进程
pid_t pid = fork();
if (pid == 0) {
    // 子进程
    exit(0);
} else {
    // 父进程
    wait(NULL);  // 回收子进程
}
```

```bash
# 统计僵尸进程数量
ps aux | grep Z | wc -l

# 查看系统中的僵尸进程数量

# Tasks: 200 total, 1 running, 199 sleeping, 0 stopped, 5 zombie
```

## 孤儿进程（对比考点）

### 什么是孤儿进程

**定义**：父进程先结束，子进程被 init 收养。

**特点**：
- **父进程先结束**：父进程退出，子进程还在运行
- **被 init 收养**：PPID 变成 1（init 进程）
- **正常运行**：不影响子进程运行
- **会被正常回收**：init 会回收所有孤儿进程
- **不是问题**：孤儿进程是正常现象

### 僵尸进程 vs 孤儿进程

| 维度 | 僵尸进程 | 孤儿进程 |
|------|---------|---------|
| 定义 | 子进程结束但未回收 | 父进程先结束 |
| 状态 | 已结束（Z） | 正常运行 |
| PPID | 不变 | 变成 1 |
| 问题 | 是问题，占用 PID | 不是问题 |
| 处理 | 需要手动处理 | 自动处理 |

## 守护进程

**守护进程（Daemon）**：后台运行的进程，不依赖终端。

**特点**：
- 后台运行，不占用终端
- 不受终端关闭影响
- 通常由 init 启动
- 命名约定：通常以 d 结尾（如 sshd、httpd）

**创建守护进程**：
```bash
nohup command &           # 简单方式
nohup java -jar app.jar > /dev/null 2>&1 &
```

## 常见问题

**Q: 如何查看进程的 CPU 和内存使用率？**  
A: `top` 或 `ps aux`

**Q: 如何杀死一个进程？**  
A: `kill PID`，如果不行用 `kill -9 PID`

**Q: kill 和 kill -9 有什么区别？**  
A: kill 发送 SIGTERM，进程可以优雅退出。kill -9 发送 SIGKILL，强制杀死

**Q: 什么是僵尸进程？**  
A: 子进程已结束但父进程未回收，占用 PID，状态为 Z

**Q: 如何处理僵尸进程？**  
A: 杀死父进程，僵尸进程会被 init 收养并回收

**Q: 什么是孤儿进程？**  
A: 父进程先结束，子进程被 init 收养。不是问题

**Q: 僵尸进程和孤儿进程有什么区别？**  
A: 僵尸进程是子进程结束但未回收，孤儿进程是父进程先结束。僵尸是问题，孤儿不是

**Q: 如何让进程在后台运行？**  
A: `command &` 或 `nohup command &`

**Q: 如何查看后台任务？**  
A: `jobs`

**Q: 如何将后台进程调到前台？**  
A: `fg`
