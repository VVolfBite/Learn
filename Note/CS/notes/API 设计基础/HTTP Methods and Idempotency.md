# HTTP Methods and Idempotency

> HTTP 方法不只是“请求类型”，它们承载着调用方对操作语义的预期。幂等性则进一步告诉我们，多次重复发起同一请求时，系统语义是否稳定。

## 背景与作用

GET、POST、PUT、PATCH、DELETE 等方法是 HTTP 用来表达操作意图的基本语义。幂等性则描述一个请求执行多次后，系统结果是否和执行一次时保持同样语义。它关心的是重复请求的结果稳定性，而不是“会不会报错”或“服务器有没有做任何事”。

如果接口方法随意使用，调用方会很难判断这个请求是否安全、能否重试、缓存是否有意义。幂等性尤其重要，因为网络重试、客户端重复提交、服务端超时重放都可能让同一个动作被发送多次。设计得好，系统在这些情况下会更稳，调用方也更容易做自动化处理。

## 场景与边界

HTTP 方法和幂等性是 API 设计中的基础语义。面试喜欢问，是因为很多人会把它们当成死知识点，却说不清工程意义。工作里一旦出现重复提交、重试机制、状态更新和对账问题，幂等性就会从概念题变成现实设计题。

一个误区是把每个方法背成独立定义，却不知道为什么这些语义重要。另一个误区是把幂等性理解成“不会报错”或“完全不产生副作用”。更准确地说，幂等性关注的是多次执行同一请求后，资源结果是否保持稳定。比如 PUT 通常按整体替换语义设计，所以天然更容易做到幂等；POST 常用于创建或触发动作，因此默认不一定幂等，但也可以通过业务键或幂等键实现幂等。

## 面试表达

HTTP 方法表达的是接口操作语义，幂等性表达的是重复执行请求时结果是否稳定。两者都影响调用方如何理解接口、是否可以安全重试以及服务端如何设计状态变更。面试里比背定义更重要的是说清这些语义为什么对系统稳定性有价值。

## 快速回忆

- HTTP 方法体现操作语义。
- 幂等性关心重复请求的结果稳定性。
- POST 默认不一定幂等，但可以设计成幂等。
- 幂等性对重试和重复提交处理很关键。
