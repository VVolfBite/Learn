# 缓存基础 - 高频判断

## 什么时候不适合上缓存

数据更新极其频繁、强一致要求很高、命中模式又不稳定时，上缓存往往收益有限，反而会把问题从数据库侧转移到一致性和失效处理上。

## 命中率高就说明缓存设计好吗

不一定。命中率只是现象，不是完整结论。还要看命中的到底是不是高成本路径、回源压力是否真的下降、缓存内容是否值得占空间。

## 缓存和 Redis 是一回事吗

不是。缓存是思路和手段，Redis 只是常见实现之一。浏览器缓存、本地缓存、代理缓存都属于缓存，但不等于 Redis。

## 穿透、击穿、雪崩怎么区分

穿透是请求的数据本来就不存在；击穿是热点 key 在某个时刻失效；雪崩是大量 key 同时失效，或者缓存层整体出问题。
