# 硬链接与软链接

硬链接和软链接是 Linux 面试的高频考点，必须理解它们的区别。

## 什么是硬链接

**硬链接（Hard Link）**：多个文件名指向同一个 inode。

### 硬链接的特点

这一节主要把“硬链接的特点”放回“Linux - 硬链接与软链接”这条主线里看。先抓住它在整个Linux知识体系中的位置，再去看后面的分类和结论，会更容易把零散的判断串起来。

- **共享 inode**：多个文件名指向同一个 inode
- **共享数据**：修改任何一个文件，其他文件也会变化
- **删除安全**：删除一个文件名，其他文件名仍然有效
- **硬链接数**：inode 记录有多少个文件名指向它
- **只有硬链接数为 0，文件才真正删除**

### 创建硬链接

```ash
ln file1 file2            # 创建硬链接
# file1 和 file2 指向同一个 inode
```

### 查看硬链接

```ash
ls -li file1 file2
# 123456 -rw-r--r-- 2 user group 1024 Jan 1 12:00 file1
# 123456 -rw-r--r-- 2 user group 1024 Jan 1 12:00 file2
# inode 号相同（123456），硬链接数为 2
```

### 硬链接的限制

这一节看起来往往容易变成一串结论，其实先抓住“硬链接的限制”在Linux里的作用会更稳。后面的内容可以理解成围绕这个问题展开的几种常见判断和做法。

**不能跨文件系统**：
- 不同文件系统有不同的 inode 表
- inode 号只在同一文件系统内唯一

**不能链接目录**：
- 防止循环引用
- 只有 root 可以创建目录的硬链接（但不推荐）

**例子**：
```ash
ln /home/user/file /tmp/link
# 如果 /home 和 /tmp 在不同文件系统，会失败

ln /home/user/dir /tmp/link
# 不能链接目录，会失败
```

## 什么是软链接

**软链接（Soft Link / Symbolic Link）**：独立的文件，内容是目标文件的路径。

### 软链接的特点

这一节主要把“软链接的特点”放回“Linux - 硬链接与软链接”这条主线里看。先抓住它在整个Linux知识体系中的位置，再去看后面的分类和结论，会更容易把零散的判断串起来。

- **独立 inode**：软链接有自己的 inode
- **存储路径**：软链接的内容是目标文件的路径
- **类似快捷方式**：类似 Windows 的快捷方式
- **删除源文件**：软链接失效，变成悬空链接

### 创建软链接

```ash
ln -s file1 file2         # 创建软链接
# file2 是软链接，指向 file1
```

### 查看软链接

```ash
ls -li file1 file2
# 123456 -rw-r--r-- 1 user group 1024 Jan 1 12:00 file1
# 789012 lrwxrwxrwx 1 user group    5 Jan 1 12:00 file2 -> file1
# inode 号不同，file2 是软链接（l 开头）
```

### 软链接的优点

**可以跨文件系统**：
- 软链接存储的是路径，不依赖 inode

**可以链接目录**：
- 软链接可以指向目录

**可以链接不存在的文件**：
- 软链接可以先创建，目标文件后创建

**例子**：
```ash
ln -s /home/user/file /tmp/link
# 可以跨文件系统

ln -s /home/user/dir /tmp/link
# 可以链接目录

ln -s /path/to/future/file link
# 可以链接不存在的文件（悬空链接）
```

## 硬链接 vs 软链接

| 维度 | 硬链接 | 软链接 |
|------|--------|--------|
| inode | 共享 inode | 独立 inode |
| 数据 | 共享数据 | 存储路径 |
| 删除源文件 | 其他链接仍有效 | 链接失效 |
| 跨文件系统 | 不可以 | 可以 |
| 链接目录 | 不可以 | 可以 |
| 文件类型 | 普通文件 | 符号链接（l） |
| 大小 | 和源文件相同 | 路径长度 |

### 对比示例

**硬链接**：
```ash
echo \"hello\" > file1
ln file1 file2
ls -li file1 file2
# 123456 -rw-r--r-- 2 user group 6 Jan 1 12:00 file1
# 123456 -rw-r--r-- 2 user group 6 Jan 1 12:00 file2

cat file2
# hello

rm file1
cat file2
# hello（file2 仍然有效）
```

**软链接**：
```ash
echo \"hello\" > file1
ln -s file1 file2
ls -li file1 file2
# 123456 -rw-r--r-- 1 user group 6 Jan 1 12:00 file1
# 789012 lrwxrwxrwx 1 user group 5 Jan 1 12:00 file2 -> file1

cat file2
# hello

rm file1
cat file2
# cat: file2: No such file or directory（file2 失效）
```

## 使用场景

### 硬链接的使用场景

**备份文件**：
- 创建硬链接作为备份
- 删除原文件，备份仍然有效
- 不占用额外空间（共享数据）

**例子**：
```ash
ln important.txt important.txt.backup
# 备份文件，不占用额外空间
```

**快照**：
- 某些备份工具使用硬链接实现增量备份
- 只有变化的文件占用新空间

### 软链接的使用场景

**版本管理**：
- 软链接指向当前版本
- 升级时只需要修改软链接

**例子**：
```ash
ln -s /usr/local/app-1.0 /usr/local/app
# 升级到 1.1
ln -sf /usr/local/app-1.1 /usr/local/app
```

**路径简化**：
- 创建短路径指向长路径

**例子**：
```ash
ln -s /var/www/html/very/long/path /var/www/short
```

**跨文件系统链接**：
- 链接不同分区的文件

**例子**：
```ash
ln -s /mnt/data/file /home/user/file
```

## 目录的硬链接数

目录的硬链接数有特殊含义。

### 空目录的硬链接数

```ash
mkdir dir
ls -ld dir
# drwxr-xr-x 2 user group 4096 Jan 1 12:00 dir
# 硬链接数为 2
```

**为什么是 2**：
- 目录本身：dir
- 目录内的 .：指向自己

### 有子目录的硬链接数

```ash
mkdir dir/subdir
ls -ld dir
# drwxr-xr-x 3 user group 4096 Jan 1 12:00 dir
# 硬链接数为 3
```

**为什么是 3**：
- 目录本身：dir
- 目录内的 .：指向自己
- 子目录内的 ..：指向父目录

**规律**：
- 目录的硬链接数 = 2 + 子目录数

## 常见问题

**Q: 硬链接和软链接有什么区别？**  
A: 硬链接共享 inode，软链接是独立文件。硬链接不能跨文件系统和链接目录，软链接可以。

**Q: 删除源文件后，硬链接和软链接会怎样？**  
A: 硬链接仍然有效，软链接失效（悬空链接）。

**Q: 为什么硬链接不能跨文件系统？**  
A: 因为不同文件系统有不同的 inode 表，inode 号只在同一文件系统内唯一。

**Q: 为什么硬链接不能链接目录？**  
A: 防止循环引用，避免文件系统混乱。

**Q: 如何判断是硬链接还是软链接？**  
A: 用 \ls -l\，软链接第一个字符是 \l\，硬链接是普通文件 \-\。

**Q: 软链接的大小是什么？**  
A: 软链接的大小是目标路径的长度，不是目标文件的大小。

**Q: 如何找到一个文件的所有硬链接？**  
A: 使用 \
ind / -inum <inode号>\。

**Q: 什么时候用硬链接，什么时候用软链接？**  
A: 需要备份、不跨文件系统用硬链接。需要版本管理、跨文件系统、链接目录用软链接。

**Q: 目录的硬链接数为什么是 2？**  
A: 目录本身和目录内的 \. 都指向同一个 inode。

**Q: 如何删除软链接？**  
A: \
m link\（不要加 \/\，否则会删除目标目录的内容）。
