# 高频判断

面试前 5 分钟快速复习卡片。

## 核心易混点

### 1. 文件名 ≠ inode
**错误** 文件名就是 inode  
**正确** 文件名在目录 block 中，目录维护"文件名 → inode 号"映射

### 2. 删除 ≠ 立刻释放
**错误** rm 后数据立即消失  
**正确** 只标记空闲，数据还在磁盘，可恢复

### 3. 硬链接 = 共享 inode
**错误** 硬链接是复制文件  
**正确** 多个文件名指向同一 inode，修改一个全变

### 4. 僵尸进程 ≠ 还在执行
**错误** 僵尸进程还在运行  
**正确** 已结束，只占 PID，无法 kill

### 5. kill ≠ 立刻终止
**错误** kill 立即杀死进程  
**正确** kill 发 SIGTERM 可拒绝，kill -9 才强制

### 6. 端口监听 ≠ 服务可用
**错误** netstat 显示监听就可用  
**正确** 可能启动失败、初始化未完成

## 对比速查

| A | B | 区别 |
|---|---|------|
| 硬链接 | 软链接 | 共享 inode vs 独立文件 |
| inode | block | 元信息 vs 实际内容 |
| 僵尸 | 孤儿 | 已结束未回收 vs 父进程先结束 |
| kill | kill -9 | 可捕获 vs 强制 |
| `>` | `>>` | 覆盖 vs 追加 |
| 0.0.0.0 | 127.0.0.1 | 所有 IP vs 只本地 |

## 常见误区

**误区 1**：目录 x 权限不重要  
**真相**：没 x 连 cd 都进不去

**误区 2**：删除文件看文件权限  
**真相**：看目录权限（删除是修改目录）

**误区 3**：小文件不占空间  
**真相**：至少占 1 个 block（4KB）

**误区 4**：inode 用不完  
**真相**：inode 有限，用完不能创建文件

**误区 5**：`&` 后台不受终端影响  
**真相**：需要 `nohup`，否则终端关闭进程退出

**误区 6**：`2>&1 > file` 正确  
**真相**：顺序错了，应该 `> file 2>&1`

## 记忆口诀

- 文件名在目录，inode 在表中
- 删除标空闲，数据还在盘
- 硬链接共 inode，软链接存路径
- 僵尸已经死，孤儿被收养
- kill 可拒绝，kill -9 无法逃
- 端口在监听，服务未必通
- 目录没有 x，进都进不去
- 删除看目录，不看文件权限
