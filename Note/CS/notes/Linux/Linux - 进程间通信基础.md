## 为什么需要进程间通信

进程是独立的执行单元，拥有独立的地址空间。进程间需要通信来：
- 数据传输
- 资源共享
- 事件通知
- 进程控制

## 管道（Pipe）

**管道**：单向通信，数据只能从一端写入，另一端读出。

### 匿名管道

**特点**：
- 只能用于父子进程或兄弟进程
- 单向通信
- 数据在内存中，不经过磁盘

**使用**：
```bash
command1 | command2       # Shell 中的管道
ls | grep txt             # ls 的输出传给 grep
```

### 命名管道（FIFO）

**特点**：
- 可以用于任意进程
- 有文件名，存在于文件系统中
- 单向通信

**创建和使用**：
```bash
mkfifo mypipe             # 创建命名管道
echo "hello" > mypipe &   # 写入（后台）
cat < mypipe              # 读取
```

## 信号（Signal）

**信号**：异步通知机制，用于通知进程发生了某个事件。

- **SIGTERM（15）**：终止信号，可以被捕获
- **SIGKILL（9）**：杀死信号，不能被捕获
- **SIGINT（2）**：中断信号（Ctrl+C）
- **SIGSTOP（19）**：停止信号，不能被捕获
- **SIGCONT（18）**：继续信号
- **SIGHUP（1）**：挂断信号
- **SIGCHLD（17）**：子进程状态改变

### 发送信号

```bash
kill -TERM PID            # 发送 SIGTERM
kill -9 PID               # 发送 SIGKILL
kill -STOP PID            # 发送 SIGSTOP
```

## 共享内存

**共享内存**：多个进程共享同一块内存区域。

**特点**：
- 最快的 IPC 方式
- 需要同步机制（如信号量）
- 数据不需要复制

**使用场景**：
- 大量数据传输
- 高性能要求

## 消息队列

**消息队列**：消息的链表，存储在内核中。

**特点**：
- 异步通信
- 有消息边界
- 可以按类型接收消息

**使用场景**：
- 异步通信
- 解耦进程

## 套接字（Socket）

**套接字**：网络通信的接口，也可用于本地进程通信。

**特点**：
- 最通用的 IPC 方式
- 可以跨网络通信
- 支持多种协议（TCP、UDP）

**使用场景**：
- 网络通信
- 客户端-服务器模型

## IPC 方式对比

| 方式 | 速度 | 使用难度 | 适用场景 |
|------|------|---------|---------|
| 管道 | 中等 | 简单 | 父子进程 |
| 信号 | 快 | 简单 | 事件通知 |
| 共享内存 | 最快 | 复杂 | 大量数据 |
| 消息队列 | 中等 | 中等 | 异步通信 |
| 套接字 | 慢 | 复杂 | 网络通信 |

## 常见问题

**Q: 进程间通信有哪些方式？**  
A: 管道、信号、共享内存、消息队列、套接字。

**Q: 哪种 IPC 方式最快？**  
A: 共享内存，因为数据不需要复制。

**Q: 管道是单向还是双向？**  
A: 单向，数据只能从一端写入，另一端读出。

**Q: 信号可以传递数据吗？**  
A: 不能，信号只能通知事件，不能传递数据。

**Q: 什么时候用套接字？**  
A: 网络通信或需要跨机器的进程通信。
