## 寄存器 vs 内存 vs 磁盘

| 对比项 | 寄存器 | 内存(RAM) | 磁盘(HDD/SSD) |
|--------|--------|-----------|---------------|
| 位置 | CPU 内部 | CPU 外部 | 独立设备 |
| 速度 | 最快(0.3-1ns) | 快(50-100ns) | 慢(HDD:5-10ms, SSD:50-150μs) |
| 容量 | 最小(几十到几百字节) | 中等(4-64GB) | 最大(256GB-10TB) |
| 易失性 | 易失 | 易失 | 非易失 |
| 成本 | 最高 | 中等 | 最低 |
| CPU 访问 | 直接访问 | 直接访问 | 不能直接访问 |
| 用途 | 暂存运算数据 | 存储运行中的程序 | 长期存储文件 |
| ALU 能否直接运算 | 能 | 不能 | 不能 |

**常见误区:**

寄存器容量比内存大 → 错,寄存器极小(几百字节),内存大(GB 级)

CPU 可以直接对内存中的数据运算 → 错,**ALU 只能对寄存器中的数据运算**,内存数据必须先加载到寄存器

CPU 可以直接访问硬盘 → 错,必须先将数据加载到内存,再从内存加载到寄存器

内存容量越大,CPU 运算速度越快 → 不完全对,内存大可以运行更多程序,但不会直接提升 CPU 运算速度。运算速度主要看寄存器和 Cache

## Cache 为什么快

**物理距离近。**Cache 在 CPU 内部,与 ALU 直接相连。内存在 CPU 外部,需要通过总线传输,距离远延迟大。

**硬件实现快。**Cache 用 SRAM(静态随机存储器),不需要刷新,速度快。内存用 DRAM(动态随机存储器),电容会漏电,需要定期刷新,速度慢。

**访问简单。**Cache 访问无需复杂的地址转换。内存访问需要通过 MMU 进行虚拟地址到物理地址的转换,还可能触发缺页中断。

**容量小查找快。**Cache 容量小(几 MB),查找快。内存容量大(几 GB),查找相对慢。

| 存储单元 | 访问时间 | 相对速度 |
|----------|----------|----------|
| 寄存器 | 0.3-1 纳秒 | 1x |
| L1 Cache | 1-2 纳秒 | 2-4x |
| L2 Cache | 3-10 纳秒 | 10-20x |
| L3 Cache | 10-20 纳秒 | 20-40x |
| 内存 | 50-100 纳秒 | 100-200x |

**常见误区:**

Cache 容量比主存大 → 错,Cache 很小(几 MB),主存大(几 GB)

Cache 使用 DRAM 实现 → 错,Cache 用 SRAM,内存用 DRAM

所有 CPU 核心共享 L1 Cache → 错,L1 和 L2 每个核心独享,L3 所有核心共享

Cache 命中率越高,程序运行越快 → 对,Cache 命中意味着快速访问,未命中需要访问内存,慢很多

顺序访问数组比随机访问快 → 对,顺序访问利用空间局部性,Cache 命中率高

## 补码为什么重要

**统一加减运算,简化 CPU 设计。**减法可以转换为加法:a - b = a + (-b)。CPU 的 ALU 只需要一个加法器,不需要单独的减法器,硬件简化,成本降低,速度更快。

**消除负零,节省编码。**原码有 +0 和 -0,浪费一个编码。补码只有一个 0,多出来的编码用来表示 -128(8 位)。

**扩大表示范围。**n 位补码可以表示 -2^(n-1) 到 2^(n-1) - 1,比原码多表示一个负数。8 位补码范围是 -128 到 +127,不是 -127 到 +127。

**符号位参与运算。**补码的符号位和数值位一起参与运算,ALU 不需要特殊处理符号位,电路设计更简单。

**常见误区:**

8 位补码的范围是 -127 到 +127 → 错,是 -128 到 +127,比原码多表示一个负数

原码比补码更适合计算机运算 → 错,补码统一了加减运算,更适合计算机

补码加法需要特殊处理符号位 → 错,补码符号位和数值位一起参与运算

两个正数相加结果为负,说明溢出 → 对,这是判断溢出的常用方法

补码的补码是原码 → 对,对补码再次求补码,得到原码

-128 的补码是 10000000(8 位) → 对,-128 是 8 位补码能表示的最小负数

## 浮点数误差基础

**CPU 浮点寄存器位数有限。**单精度 32 位,尾数 23 位,约 7 位十进制精度。双精度 64 位,尾数 52 位,约 15-16 位十进制精度。CPU 的 FPU(浮点运算单元)只能在这个精度范围内计算。

**二进制无法精确表示某些十进制数。**比如 0.1、0.2、0.3。类似于十进制无法精确表示 1/3 = 0.333...。这是进制转换的本质问题,不是 CPU 的缺陷。

**多次运算后误差累积。**CPU 每次浮点运算都可能产生舍入误差,多次运算后误差累积。还有大数吃小数、减法消除等问题。

**经典问题:0.1 + 0.2 ≠ 0.3。**因为 0.1 和 0.2 在 CPU 浮点寄存器中都无法精确表示,存储时已经有舍入误差,FPU 执行加法后误差累积,结果不等于 0.3。

**常见误区:**

浮点数可以精确表示所有十进制小数 → 错,二进制无法精确表示某些十进制小数

浮点数的精度是无限的 → 错,单精度约 7 位,双精度约 15-16 位

浮点数比较应该使用 == 运算符 → 错,由于浮点数有误差,应该使用误差范围(epsilon)比较

浮点数运算比整数运算快 → 错,浮点数运算比整数运算慢,因为浮点运算更复杂

金融计算应该使用浮点数 → 错,金融计算要求精确,应该使用整数(以分为单位)

浮点数有 +0 和 -0 → 对,IEEE 754 标准中有 +0 和 -0,但比较时认为相等

除以零会产生无穷大 → 对,正数除以零得 +∞,负数除以零得 -∞

NaN 表示非数值 → 对,NaN(Not a Number)表示未定义的运算结果,如 0/0、∞-∞

double 比 float 精度更高 → 对,double 有 52 位尾数,float 只有 23 位

## 易混淆点速查

**PC 和 IR**

PC(程序计数器)存储**下一条**要执行的指令地址,自动递增或被跳转指令修改。IR(指令寄存器)存储**当前**正在执行的指令,供指令译码器使用。

**Cache 和内存**

Cache 在 CPU 内部,速度快,容量小,用 SRAM,不需要刷新,对程序员透明,硬件自动管理。内存在 CPU 外部,速度较慢,容量大,用 DRAM,需要定期刷新,操作系统管理。

**原码和补码**

原码:最高位为符号位,其余位表示绝对值,有 +0 和 -0,加减法需要特殊处理。补码:正数同原码,负数为反码加 1,只有一个 0,加减法统一,硬件简单。

**时间局部性和空间局部性**

时间局部性:最近访问的数据很可能再次被访问,例如循环变量。空间局部性:访问某个地址后附近的地址也可能被访问,例如顺序访问数组。

**外中断和内中断**

外中断(硬件中断):由 CPU 外部设备产生,异步发生,与当前程序无关,可屏蔽,例如 I/O 中断、时钟中断。内中断(异常):由 CPU 内部事件引起,同步发生,与当前指令相关,不可屏蔽,例如除零异常、缺页异常。

**SRAM 和 DRAM**

SRAM(静态随机存储器):不需要刷新,速度快,成本高,容量小,用于 Cache。DRAM(动态随机存储器):需要定期刷新,速度较慢,成本低,容量大,用于主存(内存)。

**单精度和双精度**

单精度(float):32 位,约 7 位十进制精度,范围 ±1.4×10⁻⁴⁵ 到 ±3.4×10³⁸。双精度(double):64 位,约 15-16 位十进制精度,范围 ±4.9×10⁻³²⁴ 到 ±1.8×10³⁰⁸。

**有符号和无符号**

无符号整数:只能表示非负整数,n 位范围 0 到 2ⁿ - 1,8 位是 0 到 255。有符号整数:可以表示负数、零和正数,n 位范围 -2ⁿ⁻¹ 到 2ⁿ⁻¹ - 1,8 位是 -128 到 127。

**L1/L2/L3 Cache**

L1 Cache:最快(1-2ns),最小(32-64KB),分指令 Cache 和数据 Cache,每个核心独享。L2 Cache:较快(3-10ns),较小(256-512KB),每个核心独享。L3 Cache:相对慢(10-20ns),较大(8-32MB),所有核心共享。

**易失性和非易失性**

易失性存储:断电后数据丢失,例如寄存器、Cache、内存。非易失性存储:断电后数据保留,例如硬盘、SSD、ROM。
