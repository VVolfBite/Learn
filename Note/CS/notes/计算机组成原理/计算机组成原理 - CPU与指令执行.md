## CPU 基础

CPU(中央处理器)是计算机的核心,负责执行程序指令。内部主要包括:

**运算器(ALU):**执行加减乘除、与或非等运算。ALU 只能操作寄存器中的数据,不能直接对内存运算。

**控制器:**程序计数器(PC)记录下一条指令在哪,指令寄存器(IR)存放当前指令,指令译码器解析指令,时序控制逻辑产生控制信号。

**寄存器组:**通用寄存器暂存数据和地址,专用寄存器(PC、IR、状态寄存器)有特定用途。

**CPU Cache:**L1 最快但最小(几十 KB),L2 较快较小(几百 KB),L3 相对慢但较大(几 MB)。Cache 缓存热点数据,避免频繁访问内存。

CPU 的主要工作就是不断重复"取指→译码→执行"循环。从内存取出指令,解析指令含义,执行运算,写回结果。主频越高、核心越多、Cache 越大,性能越好。

## 指令执行流程

### 1. 指令周期

CPU 执行一条指令的完整过程称为**指令周期**,包括三个基本阶段:

```
取指(Fetch) → 译码(Decode) → 执行(Execute)
```

这个循环不断重复,直到程序结束。

### 2. 取指阶段 — 从内存读取指令

PC(程序计数器)中存着下一条指令的地址,假设 PC = 0x1000。CPU 把地址 0x1000 发送到地址总线,同时发送"读"控制信号。内存收到信号后,从地址 0x1000 读取指令,通过数据总线传回 CPU。CPU 把指令存入 IR(指令寄存器),然后 PC 自增指向下一条(PC = 0x1004,假设指令长度 4 字节)。

### 3. 译码阶段 — 解析指令含义

指令译码器分析 IR 中的指令,提取操作码(OpCode)确定操作类型(加法、减法、加载等),提取操作数确定数据位置(寄存器、内存地址等)。然后生成控制信号,激活需要参与的部件。

比如指令 `ADD R1, R2, R3`,译码后得知:要执行 R1 = R2 + R3。

### 4. 执行阶段 — ALU 运算并写回

从 R2 和 R3 读取数据到 ALU,ALU 执行加法运算,产生结果并更新状态标志(进位、溢出、零标志等),最后把结果写入目标寄存器 R1。如果有异常就触发中断。

## 指令执行示例

### 1. 加法指令 — 为什么这么快

**指令:** `ADD R1, R2, R3`  (R1 = R2 + R3)

**执行过程:**

| 阶段 | 操作 | 说明 |
|------|------|------|
| 取指 | PC → 地址总线 → 内存 → 数据总线 → IR | 从内存取出 ADD 指令 |
| 译码 | 指令译码器解析 IR | 识别为加法操作,操作数为 R2、R3,目标为 R1 |
| 执行 | R2 → ALU, R3 → ALU, ALU 执行加法, 结果 → R1 | 完成加法运算并写回 |

**假设数据:**
- R2 = 10
- R3 = 20
- 执行后 R1 = 30

**为什么快?**因为操作数都在寄存器中,ALU 直接读取,1 个时钟周期完成。如果操作数在内存,需要 50-100 个时钟周期才能读取。这就是为什么编译器会尽量把频繁使用的变量分配到寄存器。

### 2. 加载指令

**指令:** `LOAD R1, [0x2000]`  (从内存地址 0x2000 加载数据到 R1)

**执行过程:**

| 阶段 | 操作 | 说明 |
|------|------|------|
| 取指 | 从内存取出 LOAD 指令 | 正常取指流程 |
| 译码 | 识别为加载操作 | 目标寄存器 R1,源地址 0x2000 |
| 执行 | 0x2000 → 地址总线 → 内存 → 数据总线 → R1 | 从内存读取数据到寄存器 |

**关键点:** 执行阶段需要再次访问内存,这会比纯寄存器操作慢很多。

### 3. 跳转指令

**指令:** `JMP 0x3000`  (跳转到地址 0x3000)

**执行过程:**

| 阶段 | 操作 | 说明 |
|------|------|------|
| 取指 | 从内存取出 JMP 指令 | 正常取指流程 |
| 译码 | 识别为跳转操作 | 目标地址 0x3000 |
| 执行 | PC = 0x3000 | 修改 PC,下一条指令从 0x3000 开始执行 |

**关键点:** 跳转指令改变了程序的执行流程,PC 不再是顺序递增。

## 流水线技术

### 基本思想

为了提高 CPU 效率,现代 CPU 使用**流水线技术**,让取指、译码、执行三个阶段并行工作。

**非流水线执行:**

```
指令1: [取指] [译码] [执行]
指令2:              [取指] [译码] [执行]
指令3:                           [取指] [译码] [执行]
```

**流水线执行:**

```
指令1: [取指] [译码] [执行]
指令2:       [取指] [译码] [执行]
指令3:             [取指] [译码] [执行]
```

### 流水线优势

- 提高 CPU 利用率
- 增加指令吞吐量
- 理想情况下,每个时钟周期完成一条指令

### 流水线冒险

**数据冒险:** 后续指令依赖前面指令的结果

```
ADD R1, R2, R3    # R1 = R2 + R3
SUB R4, R1, R5    # R4 = R1 - R5,需要等待 R1 的值
```

**控制冒险:** 遇到跳转指令,不确定下一条指令的地址

```
JMP 0x3000        # 跳转后,已经取出的后续指令作废
```

**解决方法:** 流水线停顿、数据前推、分支预测等。

## 高频判断

### 判断 1: CPU 执行指令的速度只取决于主频

**错误。** CPU 执行速度还受 Cache 大小、内存速度、指令集效率、流水线深度等多种因素影响。

### 判断 2: 指令周期包括取指、译码、执行三个阶段

**正确。** 这是指令执行的基本周期,也称为机器周期。

### 判断 3: PC 存储的是当前正在执行的指令地址

**错误。** PC 存储的是**下一条**要执行的指令地址,当前指令存储在 IR(指令寄存器)中。

### 判断 4: 所有指令的执行时间都相同

**错误。** 不同指令的复杂度不同,执行时间也不同。访问内存的指令比纯寄存器操作慢得多。

### 判断 5: 译码阶段会修改寄存器的值

**错误。** 译码阶段只是解析指令,不会修改数据。只有执行阶段才会修改寄存器或内存。

### 判断 6: 流水线技术可以提高指令执行效率

**正确。** 流水线让取指、译码、执行并行工作,提高了 CPU 利用率和指令吞吐量。

### 判断 7: CPU 可以直接执行高级语言代码

**错误。** CPU 只能执行机器指令(二进制代码)。高级语言需要编译或解释成机器指令才能执行。

### 判断 8: 多核 CPU 可以同时执行多条指令

**正确。** 每个核心可以独立执行指令,多核 CPU 可以真正并行执行多个任务。

### 判断 9: 跳转指令会破坏流水线效率

**正确。** 跳转指令改变 PC 的值,导致已经取出的后续指令作废,造成流水线停顿。

### 判断 10: ALU 只能执行算术运算

**错误。** ALU(算术逻辑单元)既能执行算术运算(加减乘除),也能执行逻辑运算(与或非、比较)。

### 判断 11: 指令执行过程中,PC 的值保持不变

**错误。** 在取指阶段结束后,PC 会自动指向下一条指令(PC = PC + 指令长度)。

### 判断 12: Cache 命中率越高,CPU 执行效率越高

**正确。** Cache 命中意味着数据在 Cache 中,访问速度快;Cache 未命中需要访问内存,速度慢很多。
