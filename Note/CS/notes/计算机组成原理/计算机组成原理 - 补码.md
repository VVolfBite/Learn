## 补码概述

### 1. 什么是补码

补码(Two's Complement)是计算机中表示有符号整数的编码方式。正数的补码就是其二进制表示,负数的补码是其绝对值按位取反后加 1。

**示例(8 位):**

```
+5 的补码: 00000101
-5 的补码: 11111011
  (5 的二进制: 00000101
   按位取反:   11111010
   加 1:       11111011)
```

**为什么 CPU 需要补码?**因为 CPU 内部的 ALU 只有加法器,没有独立的减法器。补码让减法可以转换成加法,硬件设计大大简化。

### 2. 为什么需要补码

**问题:** 计算机如何表示负数?

**方案 1: 原码(直观但有问题)**
- 最高位为符号位,0 表示正,1 表示负
- 其余位表示绝对值
- 问题: 加减运算复杂,存在 +0 和 -0

**方案 2: 补码(计算机实际使用)**
- 统一加减运算
- 只有一个 0
- 硬件实现简单

### 3. 补码的优势

这里真正要说明的是“3. 补码的优势”为什么值得单独拿出来讲。把它放到“计算机组成原理 - 补码”这张卡片的上下文里看，后面的要点和区别会更容易读顺。

**1. 加减法统一:**
- 减法可以转换为加法
- a - b = a + (-b)
- 只需要一个加法器

**2. 符号位参与运算:**
- 符号位和数值位一起参与运算
- 不需要特殊处理符号位

**3. 只有一个零:**
- 原码有 +0 和 -0
- 补码只有一个 0

**4. 表示范围更大:**
- n 位补码可以表示 -2ⁿ⁻¹ 到 2ⁿ⁻¹ - 1
- 比原码多表示一个负数

## 原码、反码、补码

### 1. 原码

**定义:** 最高位为符号位(0 正 1 负),其余位表示绝对值。

**示例(8 位):**

```
+5: 00000101
-5: 10000101

+0: 00000000
-0: 10000000  (存在两个零!)
```

**范围(n 位):**
- 正数: 0 到 2ⁿ⁻¹ - 1
- 负数: -(2ⁿ⁻¹ - 1) 到 -0
- 8 位: -127 到 +127

**问题:**

**问题 1: 两个零**

```
+0 = 00000000
-0 = 10000000
两者不同,但都表示 0,浪费一个编码
```

**问题 2: 加减法复杂**

```
计算 5 + (-3):
  00000101 (+5)
+ 10000011 (-3)
-----------
  10001000 (-8) 错误!

需要特殊处理:
  判断符号位
  如果符号不同,转换为减法
  比较绝对值大小
  用大的减小的
  结果符号取绝对值大的符号
```

### 2. 反码

这一节主要把“2. 反码”放回“计算机组成原理 - 补码”这条主线里看。先抓住它在整个计算机组成原理知识体系中的位置，再去看后面的分类和结论，会更容易把零散的判断串起来。

**定义:** 
- 正数的反码就是其原码
- 负数的反码是其原码除符号位外按位取反

**示例(8 位):**

```
+5: 00000101 (与原码相同)
-5: 11111010 (原码 10000101,除符号位外取反)

+0: 00000000
-0: 11111111  (仍然存在两个零!)
```

**计算方法:**

```
求 -5 的反码:
1. 写出 +5 的原码: 00000101
2. 保持符号位不变: 1_______
3. 其余位取反:      11111010
```

**问题:**
- 仍然存在 +0 和 -0
- 加法运算需要特殊处理(循环进位)

### 3. 补码

**定义:**
- 正数的补码就是其原码
- 负数的补码是其反码加 1

**示例(8 位):**

```
+5: 00000101 (与原码相同)
-5: 11111011
  (原码: 10000101
   反码: 11111010
   补码: 11111011)

+0: 00000000
-0: 00000000 (只有一个零!)
```

**计算方法:**

**方法 1: 反码加 1**

```
求 -5 的补码:
1. 写出 +5 的原码: 00000101
2. 求反码(符号位不变,其余取反): 11111010
3. 加 1: 11111011
```

**方法 2: 保留最右边的 1 及其右边的 0,左边取反**

```
求 -5 的补码:
+5: 00000101
    ↓↓↓↓↓↑↑↑ (从右往左,遇到第一个 1,它和右边的 0 保持不变,左边取反)
-5: 11111011
```

**方法 3: 用 2ⁿ 减去该数的绝对值**

```
求 -5 的补码(8 位):
2⁸ - 5 = 256 - 5 = 251 = 11111011
```

**补码转回原码:**

```
已知补码 11111011,求原数:
方法: 再次求补码(补码的补码是原码)
1. 减 1: 11111010
2. 取反: 10000101 (这是 -5 的原码)
3. 结果: -5
```

### 4. 三种表示法对比

**示例(8 位):**

| 十进制 | 原码 | 反码 | 补码 |
|--------|------|------|------|
| +127 | 01111111 | 01111111 | 01111111 |
| +5 | 00000101 | 00000101 | 00000101 |
| +1 | 00000001 | 00000001 | 00000001 |
| +0 | 00000000 | 00000000 | 00000000 |
| -0 | 10000000 | 11111111 | (不存在) |
| -1 | 10000001 | 11111110 | 11111111 |
| -5 | 10000101 | 11111010 | 11111011 |
| -127 | 11111111 | 10000000 | 10000001 |
| -128 | (不存在) | (不存在) | 10000000 |

**范围对比(8 位):**

| 表示法 | 范围 | 零的个数 |
|--------|------|----------|
| 原码 | -127 到 +127 | 2 个(+0, -0) |
| 反码 | -127 到 +127 | 2 个(+0, -0) |
| 补码 | -128 到 +127 | 1 个 |

## 为什么计算机使用补码

### 1. 统一加减运算 — CPU 只需要一个加法器

减法可以转换为加法:a - b = a + (-b)。这意味着 CPU 的 ALU 只需要实现加法器,不需要单独的减法器。

**示例 1: 5 - 3 = 5 + (-3)**

```
CPU 内部的 ALU 执行:
  00000101 (+5)
+ 11111101 (-3)
-----------
 100000010
  ↑ 溢出位丢弃
  00000010 (+2) 正确!
```

**示例 2: -5 + 3**

```
ALU 执行:
  11111011 (-5)
+ 00000011 (+3)
-----------
  11111110 (-2) 正确!
```

**示例 3: -5 - 3 = -5 + (-3)**

```
ALU 执行:
  11111011 (-5)
+ 11111101 (-3)
-----------
 111111000
  ↑ 溢出位丢弃
  11111000 (-8) 正确!
```

**硬件简化:**

```
使用原码:
  ALU 需要:
    - 加法器
    - 减法器
    - 符号判断电路
    - 绝对值比较电路

使用补码:
  ALU 只需要:
    - 加法器
  
减法转换为加法,硬件大大简化,成本降低,速度更快!
```

这就是为什么所有现代 CPU 都使用补码表示有符号整数。

### 2. 消除负零

**原码的问题:**

```
+0: 00000000
-0: 10000000

两个不同的编码表示同一个数,浪费!
```

**补码的解决:**

```
+0 的补码: 00000000
-0 的补码:
  原码: 10000000
  反码: 11111111
  补码: 11111111 + 1 = 100000000
       ↑ 溢出,丢弃
       = 00000000

-0 和 +0 的补码相同,只有一个零!
```

### 3. 扩大表示范围

这一节看起来往往容易变成一串结论，其实先抓住“3. 扩大表示范围”在计算机组成原理里的作用会更稳。后面的内容可以理解成围绕这个问题展开的几种常见判断和做法。

**原码和反码(8 位):**
- 范围: -127 到 +127
- 共 255 个数(+0 和 -0 占用两个编码)

**补码(8 位):**
- 范围: -128 到 +127
- 共 256 个数(只有一个 0)
- 多表示一个负数 -128

**-128 的补码:**

```
-128 的补码: 10000000

验证:
  10000000 的相反数:
  减 1: 01111111
  取反: 10000000 (回到自己)
  
所以 10000000 表示 -128
```

**为什么 -128 没有对应的正数:**
- n 位补码范围: -2ⁿ⁻¹ 到 2ⁿ⁻¹ - 1
- 负数比正数多一个
- 这是补码的特性,不是缺陷

## 补码的运算

### 1. 补码加法

**规则:** 直接相加,溢出位丢弃。

**示例 1: 正数 + 正数**

```
5 + 3:
  00000101
+ 00000011
-----------
  00001000 (+8) 正确
```

**示例 2: 正数 + 负数**

```
5 + (-3):
  00000101
+ 11111101
-----------
 100000010
  ↑ 丢弃
  00000010 (+2) 正确
```

**示例 3: 负数 + 负数**

```
-5 + (-3):
  11111011
+ 11111101
-----------
 111111000
  ↑ 丢弃
  11111000 (-8) 正确
```

### 2. 补码减法

**规则:** 转换为加法,a - b = a + (-b)

**步骤:**
1. 求 b 的补码(即 -b)
2. 执行加法 a + (-b)

**示例: 5 - 3**

```
1. 求 -3 的补码:
   +3: 00000011
   -3: 11111101

2. 计算 5 + (-3):
     00000101
   + 11111101
   -----------
    100000010
     ↑ 丢弃
     00000010 (+2) 正确
```

### 3. 溢出判断

**溢出:** 运算结果超出表示范围。

**溢出条件:**
- 正数 + 正数 = 负数 → 溢出
- 负数 + 负数 = 正数 → 溢出
- 正数 + 负数 → 不会溢出

**判断方法:**

**方法 1: 符号位判断**
- 两个正数相加,结果为负 → 溢出
- 两个负数相加,结果为正 → 溢出

**方法 2: 进位判断**
- 符号位的进位 ⊕ 最高位的进位 = 1 → 溢出

**示例 1: 正溢出**

```
127 + 1 (8 位):
  01111111
+ 00000001
-----------
  10000000 (-128) 错误!溢出!
```

**示例 2: 负溢出**

```
-128 + (-1) (8 位):
  10000000
+ 11111111
-----------
 101111111
  ↑ 丢弃
  01111111 (+127) 错误!溢出!
```

## 补码的应用

### 应用 1: 整数类型

**C/C++ 中的整数类型都使用补码:**

```c
char c = -128;        // 8 位补码: 10000000
short s = -32768;     // 16 位补码: 1000000000000000
int i = -2147483648;  // 32 位补码: 10000000000000000000000000000000
```

### 应用 2: 类型转换

**有符号到无符号:**

```c
int a = -1;           // 补码: 11111111111111111111111111111111
unsigned int b = a;   // 解释为无符号: 4294967295
```

**无符号到有符号:**

```c
unsigned int a = 4294967295;  // 11111111111111111111111111111111
int b = a;                    // 解释为补码: -1
```

### 应用 3: 位运算

**取相反数:**

```c
int negate(int x) {
    return ~x + 1;  // 按位取反加 1,即求补码
}
```

**判断符号:**

```c
bool isNegative(int x) {
    return x < 0;  // 等价于检查最高位
}
```

### 应用 4: 溢出检测

**检测加法溢出:**

```c
bool addOverflow(int a, int b) {
    int sum = a + b;
    // 正 + 正 = 负,或 负 + 负 = 正
    if (a > 0 && b > 0 && sum < 0) return true;
    if (a < 0 && b < 0 && sum > 0) return true;
    return false;
}
```

### 应用 5: 特殊值

**最小负数的特殊性:**

```c
int min = -2147483648;  // 32 位最小值
int abs_min = -min;     // 溢出!仍然是 -2147483648

// 因为 -(-2147483648) = +2147483648,超出范围
```

## 高频判断

### 判断 1: 计算机使用补码表示有符号整数

**正确。** 补码统一了加减运算,硬件实现简单。

### 判断 2: 正数的补码就是其二进制表示

**正确。** 正数的原码、反码、补码都相同。

### 判断 3: 负数的补码是其绝对值按位取反加 1

**正确。** 这是求负数补码的常用方法。

### 判断 4: 补码消除了 +0 和 -0 的问题

**正确。** 补码中只有一个 0,即 00000000。

### 判断 5: 8 位补码的范围是 -127 到 +127

**错误。** 8 位补码的范围是 -128 到 +127,比原码多表示一个负数。

### 判断 6: 补码加法可以直接相加,溢出位丢弃

**正确。** 这是补码的重要优势,简化了硬件设计。

### 判断 7: 补码减法需要转换为加法

**正确。** a - b = a + (-b),先求 -b 的补码,再相加。

### 判断 8: 两个正数相加结果为负,说明溢出

**正确。** 这是判断溢出的常用方法。

### 判断 9: 补码的补码是原码

**正确。** 对补码再次求补码,得到原码。

### 判断 10: -128 的补码是 10000000(8 位)

**正确。** -128 是 8 位补码能表示的最小负数。

### 判断 11: 补码中符号位参与运算

**正确。** 补码的符号位和数值位一起参与运算,不需要特殊处理。

### 判断 12: 原码比补码更适合计算机运算

**错误。** 补码统一了加减运算,更适合计算机。原码加减法需要特殊处理。

### 判断 13: 补码的最小负数没有对应的正数

**正确。** n 位补码的最小负数是 -2ⁿ⁻¹,没有对应的正数 2ⁿ⁻¹。

### 判断 14: 求补码的方法是按位取反加 1

**正确。** 这是求负数补码的快速方法。

### 判断 15: 补码加法不需要考虑符号位

**正确。** 补码加法直接相加即可,符号位自动处理。
