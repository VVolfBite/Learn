## 浮点数概述

### 1. 什么是浮点数

浮点数(Floating Point Number)是计算机中表示实数(包括小数和很大/很小的数)的方式。"浮点"指小数点的位置可以"浮动",类似于科学计数法,通过调整指数来移动小数点。

**示例:**

```
十进制科学计数法:
  123.45 = 1.2345 × 10²
  0.00012345 = 1.2345 × 10⁻⁴

小数点位置"浮动",通过指数调整
```

**为什么 CPU 需要浮点数?**因为整数只能表示整数,无法表示小数、很大的数(如 10²³)、很小的数(如 10⁻²³)。CPU 需要处理科学计算、图形渲染等任务,必须支持浮点运算。现代 CPU 都有专门的浮点运算单元(FPU)。

### 2. 为什么需要浮点数

这里真正要说明的是“2. 为什么需要浮点数”为什么值得单独拿出来讲。把它放到“计算机组成原理 - 浮点数基础”这张卡片的上下文里看，后面的要点和区别会更容易读顺。

**整数的局限:**
- 只能表示整数
- 无法表示小数: 3.14, 0.5
- 无法表示很大的数: 10²³
- 无法表示很小的数: 10⁻²³

**浮点数的优势:**
- 可以表示小数
- 可以表示很大的数(通过大指数)
- 可以表示很小的数(通过负指数)
- 表示范围广

**示例:**

```c
int a = 3.14;        // 错误!整数无法存储小数
float b = 3.14;      // 正确,浮点数可以表示小数

int c = 1000000000000;  // 可能溢出
float d = 1e12;         // 可以表示,1 × 10¹²
```

### 3. 浮点数的特点

这一节看起来往往容易变成一串结论，其实先抓住“3. 浮点数的特点”在计算机组成原理里的作用会更稳。后面的内容可以理解成围绕这个问题展开的几种常见判断和做法。

**优点:**
- 表示范围广
- 可以表示小数
- 适合科学计算

**缺点:**
- 精度有限
- 存在舍入误差
- 运算速度比整数慢
- 不能精确表示所有十进制小数

## 浮点数表示的基本思想

### 1. 科学计数法

**十进制科学计数法:**

```
格式: ±M × 10^E
  M: 尾数(Mantissa),1 ≤ |M| < 10
  E: 指数(Exponent)

示例:
  123.45 = 1.2345 × 10²
  -0.00012345 = -1.2345 × 10⁻⁴
  6.02 × 10²³ (阿伏伽德罗常数)
```

**优势:**
- 可以表示很大的数(大指数)
- 可以表示很小的数(负指数)
- 统一的表示格式

### 2. 二进制科学计数法

**格式:** ±M × 2^E

```
示例:
  5.5 (十进制) = 101.1 (二进制)
              = 1.011 × 2²

  0.375 (十进制) = 0.011 (二进制)
                 = 1.1 × 2⁻²
```

**规格化:**
- 尾数 M 的整数部分只有一位,且为 1
- 即 1.xxx × 2^E 的形式
- 这样可以节省存储空间(隐藏整数部分的 1)

### 3. IEEE 754 标准

**定义:** IEEE 754 是国际标准,定义了浮点数的表示格式和运算规则。

**组成部分:**
1. **符号位(Sign)**: 0 表示正,1 表示负
2. **指数(Exponent)**: 用移码表示
3. **尾数(Mantissa/Fraction)**: 小数部分

**格式:**

```
浮点数 = (-1)^符号位 × 1.尾数 × 2^(指数-偏移量)
```

## IEEE 754 浮点数格式

### 1. 单精度(32 位)

**格式:**

```
| 符号位 | 指数(8位) | 尾数(23位) |
|   1    |     8     |     23     |
```

**参数:**
- 总位数: 32 位
- 指数位数: 8 位
- 尾数位数: 23 位
- 指数偏移量: 127
- 表示范围: 约 ±1.4 × 10⁻⁴⁵ 到 ±3.4 × 10³⁸
- 精度: 约 7 位十进制数字

**示例: 表示 5.5**

```
1. 转换为二进制: 5.5 = 101.1
2. 规格化: 1.011 × 2²
3. 提取各部分:
   - 符号位: 0 (正数)
   - 指数: 2 + 127 = 129 = 10000001
   - 尾数: 011 (后面补 0 到 23 位)

4. 最终表示:
   0 10000001 01100000000000000000000
   ↑    ↑              ↑
   符号 指数           尾数
```

**示例: 表示 -0.375**

```
1. 转换为二进制: 0.375 = 0.011
2. 规格化: 1.1 × 2⁻²
3. 提取各部分:
   - 符号位: 1 (负数)
   - 指数: -2 + 127 = 125 = 01111101
   - 尾数: 1 (后面补 0 到 23 位)

4. 最终表示:
   1 01111101 10000000000000000000000
```

### 2. 双精度(64 位)

**格式:**

```
| 符号位 | 指数(11位) | 尾数(52位) |
|   1    |     11     |     52     |
```

**参数:**
- 总位数: 64 位
- 指数位数: 11 位
- 尾数位数: 52 位
- 指数偏移量: 1023
- 表示范围: 约 ±4.9 × 10⁻³²⁴ 到 ±1.8 × 10³⁰⁸
- 精度: 约 15-16 位十进制数字

**对比:**

| 类型 | 位数 | 指数位 | 尾数位 | 偏移量 | 精度 |
|------|------|--------|--------|--------|------|
| float | 32 | 8 | 23 | 127 | 约 7 位 |
| double | 64 | 11 | 52 | 1023 | 约 15-16 位 |

### 3. 特殊值

**零:**

```
+0: 0 00000000 00000000000000000000000
-0: 1 00000000 00000000000000000000000

注意: 浮点数有 +0 和 -0,但在比较时认为相等
```

**无穷大:**

```
+∞: 0 11111111 00000000000000000000000
-∞: 1 11111111 00000000000000000000000

产生原因: 除以零,溢出
```

**NaN(Not a Number):**

```
NaN: _ 11111111 非零尾数

产生原因:
  - 0 / 0
  - ∞ - ∞
  - ∞ / ∞
  - 负数开平方
```

**示例:**

```c
float a = 1.0 / 0.0;    // +∞
float b = -1.0 / 0.0;   // -∞
float c = 0.0 / 0.0;    // NaN
float d = sqrt(-1.0);   // NaN

printf("%f\n", a);      // inf
printf("%f\n", c);      // nan
```

## 为什么浮点数可能有误差

### 1. 精度有限 — CPU 的浮点寄存器位数有限

CPU 的浮点寄存器(如 x87 FPU 的 ST 寄存器、SSE 的 XMM 寄存器)位数有限。单精度 32 位,尾数只有 23 位,约 7 位十进制精度。双精度 64 位,尾数 52 位,约 15-16 位十进制精度。

**示例:**

```c
float a = 1.0f / 3.0f;  // 0.333333...
// CPU 浮点寄存器只能存储: 0.33333334 (约 7 位精度)
// 无法精确表示无限循环小数
```

**单精度 vs 双精度:**

```c
float f = 1.23456789f;
printf("%.10f\n", f);   // 1.2345678806 (精度丢失)

double d = 1.23456789;
printf("%.10f\n", d);   // 1.2345678900 (精度更高)
```

这就是为什么科学计算通常使用 double,而游戏图形可以用 float(速度更快,精度够用)。

### 2. 无法精确表示某些十进制数

**核心问题:** 二进制无法精确表示某些十进制小数。

**类比:**
- 十进制无法精确表示 1/3 = 0.333...
- 二进制无法精确表示 0.1, 0.2, 0.3 等

**示例: 0.1 的二进制表示**

```
0.1 (十进制) = 0.0001100110011001100... (二进制,无限循环)

转换过程:
0.1 × 2 = 0.2 → 0
0.2 × 2 = 0.4 → 0
0.4 × 2 = 0.8 → 0
0.8 × 2 = 1.6 → 1
0.6 × 2 = 1.2 → 1
0.2 × 2 = 0.4 → 0 (开始循环)
...

结果: 0.0(0011)循环
```

**经典问题: 0.1 + 0.2 ≠ 0.3**

```c
float a = 0.1f;
float b = 0.2f;
float c = a + b;

printf("%.20f\n", a);  // 0.10000000149011611938
printf("%.20f\n", b);  // 0.20000000298023223877
printf("%.20f\n", c);  // 0.30000001192092895508

if (c == 0.3f) {
    printf("相等\n");
} else {
    printf("不相等\n");  // 输出这个!
}
```

**原因:**
- 0.1 和 0.2 都无法精确表示
- 存储时已经有舍入误差
- 相加后误差累积
- 结果不等于 0.3

### 3. 运算误差累积

**问题:** 多次运算后,误差会累积。

**示例 1: 大数吃小数**

```c
float a = 1e20f;
float b = 1.0f;
float c = a + b;

printf("%.0f\n", c);  // 100000000000000000000 (b 被"吃掉"了)

// 原因: a 太大,b 太小,b 的精度在 a 的误差范围内
```

**示例 2: 减法消除**

```c
float a = 1.0f;
float b = 1e-10f;
float c = a + b;
float d = c - a;

printf("%.20f\n", d);  // 不等于 1e-10,误差很大
```

**示例 3: 循环累积**

```c
float sum = 0.0f;
for (int i = 0; i < 10000000; i++) {
    sum += 0.1f;
}
printf("%.10f\n", sum);  // 不等于 1000000.0,有明显误差
```

**示例 4: 比较大小的问题**

```c
float a = 0.1f + 0.2f;
float b = 0.3f;

if (a == b) {
    printf("相等\n");
} else {
    printf("不相等\n");  // 输出这个!
}
```

## 浮点数的使用注意事项

### 注意事项 1: 不要直接比较相等

**错误做法:**

```c
float a = 0.1f + 0.2f;
if (a == 0.3f) {  // 错误!
    // ...
}
```

**正确做法:** 使用误差范围(epsilon)

```c
float a = 0.1f + 0.2f;
float epsilon = 1e-6f;

if (fabs(a - 0.3f) < epsilon) {  // 正确
    // 认为相等
}
```

### 注意事项 2: 避免大数和小数运算

**问题:**

```c
float a = 1e20f;
float b = 1.0f;
float c = a + b - a;  // 期望得到 1.0

printf("%.10f\n", c);  // 0.0 (b 被"吃掉"了)
```

**改进:** 调整运算顺序

```c
float c = (a - a) + b;  // 1.0 (正确)
```

### 注意事项 3: 避免减去相近的数

**问题:**

```c
float a = 1.0000001f;
float b = 1.0000000f;
float c = a - b;  // 精度损失严重
```

**改进:** 使用数学变换避免减法

```
例如: (a² - b²) / (a + b) 而不是 (a - b) × (a + b)
```

### 注意事项 4: 循环累加要小心

**问题:**

```c
float sum = 0.0f;
for (int i = 0; i < 1000000; i++) {
    sum += 0.1f;  // 误差累积
}
```

**改进:** 使用 double 或整数运算

```c
double sum = 0.0;
for (int i = 0; i < 1000000; i++) {
    sum += 0.1;
}

// 或者
int sum_int = 0;
for (int i = 0; i < 1000000; i++) {
    sum_int += 1;  // 整数运算,无误差
}
float sum = sum_int * 0.1f;
```

### 注意事项 5: 金融计算不要用浮点数

**问题:** 金融计算要求精确,浮点数有误差。

**错误做法:**

```c
float price = 0.1f;
float total = price * 10;  // 可能不等于 1.0
```

**正确做法:** 使用整数(以分为单位)

```c
int price_cents = 10;  // 0.1 元 = 10 分
int total_cents = price_cents * 10;  // 100 分 = 1.0 元
```

### 注意事项 6: 选择合适的精度

**单精度 vs 双精度:**

```c
float f = 1.23456789f;   // 约 7 位精度
double d = 1.23456789;   // 约 15-16 位精度

// 如果需要高精度,使用 double
```

## 高频判断

### 判断 1: 浮点数使用科学计数法表示

**正确。** 浮点数的本质是二进制科学计数法,格式为 ±M × 2^E。

### 判断 2: 浮点数可以精确表示所有十进制小数

**错误。** 二进制无法精确表示某些十进制小数,如 0.1, 0.2, 0.3 等。

### 判断 3: 0.1 + 0.2 在计算机中可能不等于 0.3

**正确。** 由于 0.1 和 0.2 无法精确表示,相加后有误差。

### 判断 4: 单精度浮点数占 32 位

**正确。** 单精度(float)占 32 位,双精度(double)占 64 位。

### 判断 5: 浮点数的精度是无限的

**错误。** 浮点数精度有限,单精度约 7 位,双精度约 15-16 位十进制数字。

### 判断 6: 浮点数有 +0 和 -0

**正确。** IEEE 754 标准中,浮点数有 +0 和 -0,但在比较时认为相等。

### 判断 7: 除以零会产生无穷大

**正确。** 正数除以零得 +∞,负数除以零得 -∞。

### 判断 8: NaN 表示非数值

**正确。** NaN(Not a Number)表示未定义的运算结果,如 0/0, ∞-∞ 等。

### 判断 9: 浮点数比较应该使用 == 运算符

**错误。** 由于浮点数有误差,应该使用误差范围(epsilon)比较。

### 判断 10: double 比 float 精度更高

**正确。** double 有 52 位尾数,float 只有 23 位,double 精度更高。

### 判断 11: 浮点数运算比整数运算快

**错误。** 浮点数运算比整数运算慢,因为浮点运算更复杂。

### 判断 12: 浮点数误差会累积

**正确。** 多次运算后,舍入误差会累积,导致结果偏差越来越大。

### 判断 13: 金融计算应该使用浮点数

**错误。** 金融计算要求精确,应该使用整数(以分或厘为单位)或专门的十进制类型。

### 判断 14: 浮点数的表示范围比整数大

**正确。** 浮点数可以表示很大或很小的数,范围远大于整数。

### 判断 15: IEEE 754 是浮点数的国际标准

**正确。** IEEE 754 定义了浮点数的表示格式和运算规则,被广泛采用。
