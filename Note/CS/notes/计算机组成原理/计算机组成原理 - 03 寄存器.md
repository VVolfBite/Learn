## 寄存器

寄存器(Register)是 CPU 内部的高速存储单元,用于暂存指令、数据和地址。位于 CPU 内部,是速度最快的存储单元,但容量最小(通常几十到几百字节),直接参与运算。

```
CPU 内部:
  ┌─────────────────────────────┐
  │  ┌──────────┐  ┌──────────┐ │
  │  │ 寄存器组 │  │   ALU    │ │
  │  └──────────┘  └──────────┘ │
  │  ┌──────────┐  ┌──────────┐ │
  │  │  Cache   │  │  控制器  │ │
  │  └──────────┘  └──────────┘ │
  └─────────────────────────────┘
           ↕
        内存(RAM)
           ↕
        硬盘(SSD/HDD)
```

| 特性 | 说明 |
|------|------|
| 位置 | CPU 内部 |
| 速度 | 最快(0.3-1 纳秒) |
| 容量 | 最小(x86-64 约 128 字节) |
| 成本 | 最高 |
| 访问方式 | 直接访问,无需地址 |
| 数量 | 有限,通常几十个 |

ALU 只能对寄存器中的数据进行运算，不能直接对内存中的数据运算，因此需要寄存器做临时存储器。

### 作用

* **暂存地址**：存储内存地址,用于访问内存。比如访问数组元素

```c
int arr[100];
int x = arr[10];
```

​	**寄存器操作:**基址寄存器存储数组起始地址，索引寄存器存储偏移量(10 * sizeof(int))，计算得到实际地址 = 基址 + 偏移量，然后访问该地址的内存

* **暂存指令**：指令寄存器(IR)存储当前正在执行的指令。

​	**寄存器操作:**从内存取出指令，存入 IR，指令译码器解析 IR 中的指令并执行指令

* **控制程序执行**：程序计数器(PC)控制程序的执行顺序。

​	**寄存器操作**:PC 存储下一条指令的地址，每执行一条指令,PC 自动递增，遇到跳转指令,PC 被修改为跳转目标地址



由于**ALU 直接连接寄存器,不连接内存。**计算 `c = a + b` 时,CPU 必须这样做:

1. 从内存加载变量 a 到寄存器 R1
2. 从内存加载变量 b 到寄存器 R2
3. ALU 执行 R1 + R2,结果存入 R3
4. 把 R3 写回内存中的 c

因为内存太慢(50-100ns),ALU 太快(1ns)。如果 ALU 直接连接内存,每次运算都要等内存,CPU 大部分时间在空等。寄存器作为缓冲,让 ALU 能高速运算。寄存器还暂存地址。访问数组 `arr[i]` 时,基地址存在一个寄存器,索引 i 存在另一个寄存器,计算出 `基地址 + i × 元素大小` 得到实际地址。指令寄存器(IR)存放当前正在执行的指令,供译码器解析。程序计数器(PC)存放下一条指令的地址,控制程序执行流程。

### 常见类型

**通用寄存器**可以存储数据或地址,用途灵活。x86-64 有 RAX(累加器,算术运算和函数返回值)、RBX(基址寄存器)、RCX(计数器,循环计数)、RDX(数据寄存器)、RSI(源索引)、RDI(目的索引)、RSP(栈指针)、RBP(基址指针)等。

函数调用时,参数通过寄存器传递。比如 `add(3, 5)`,参数 3 存入 RDI,参数 5 存入 RSI,函数内部 RAX = RDI + RSI,返回值通过 RAX 传递。

**专用寄存器**有特定用途。程序计数器(PC/IP)存储下一条指令地址,自动递增或被跳转指令修改。指令寄存器(IR)存储当前正在执行的指令。状态寄存器(FLAGS)存储 CPU 状态标志:ZF(零标志,运算结果为 0)、CF(进位标志)、OF(溢出标志)、SF(符号标志,运算结果为负)。条件跳转依赖状态寄存器。`CMP R1, R2` 比较两个寄存器,结果影响 FLAGS。`JE LABEL` 如果相等(ZF=1)就跳转。

### 快速访问

寄存器在 CPU 内部,与 ALU 直接相连,无需总线传输。内存在 CPU 外部,需要通过总线,距离远延迟大。寄存器通过寄存器编号直接访问,无需地址计算和地址转换。内存访问需要计算地址,通过 MMU 进行虚拟地址到物理地址转换,还可能触发缺页中断。寄存器用触发器实现,直接集成在 CPU 芯片上,与 ALU 共享同一时钟周期。内存用电容存储(DRAM),独立芯片通过总线连接,需要刷新,访问延迟大。

| 存储单元 | 访问时间 | 相对速度 |
|----------|----------|----------|
| 寄存器 | 0.3-1 纳秒 | 1x |
| L1 Cache | 1-2 纳秒 | 2-4x |
| L2 Cache | 3-10 纳秒 | 10-20x |
| 内存 | 50-100 纳秒 | 100-200x |

## 高频判断

**寄存器是 CPU 内部的存储单元** → 对,寄存器位于 CPU 内部,是 CPU 的组成部分。

**寄存器的容量比内存大** → 错,寄存器极小(几百字节),内存大(GB 级)。

**寄存器是计算机中速度最快的存储单元** → 对,寄存器访问速度最快,通常 1 个时钟周期。

**CPU 可以直接对内存中的数据进行运算** → 错,ALU 只能对寄存器中的数据运算,内存数据必须先加载到寄存器。

**所有寄存器都可以用于任意用途** → 错,专用寄存器(如 PC、IR)有特定用途,不能随意使用。

**寄存器数量越多,程序运行越快** → 基本对,更多寄存器可以减少内存访问,但实际性能还受其他因素影响。

**程序计数器(PC)存储当前正在执行的指令** → 错,PC 存储**下一条**要执行的指令地址,当前指令在 IR 中。

**寄存器访问需要地址** → 错,寄存器通过寄存器编号直接访问,不需要地址。

**64 位 CPU 的寄存器宽度是 64 位** → 对,64 位 CPU 的通用寄存器宽度为 64 位。

**寄存器的内容断电后会丢失** → 对,寄存器是易失性存储,断电后数据丢失。

**函数的返回值通常通过寄存器传递** → 对,大多数调用约定中,返回值通过特定寄存器(如 x86-64 的 RAX)传递。

**寄存器比 L1 Cache 快** → 对,寄存器约 0.3-1 纳秒,L1 Cache 约 1-2 纳秒。

**编译器会自动优化寄存器的使用** → 对,编译器会进行寄存器分配优化,让频繁使用的变量保持在寄存器中。

**状态寄存器存储运算结果** → 错,状态寄存器存储运算的**状态标志**(如零标志、进位标志),运算结果存在通用寄存器中。
