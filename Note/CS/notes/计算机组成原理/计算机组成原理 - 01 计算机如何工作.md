## 计算机基本组成

冯·诺依曼体系结构把计算机分成五大部件:**运算器、控制器、存储器、输入设备、输出设备**。

运算器的核心是 ALU(算术逻辑单元),负责加减乘除、与或非等运算。运算时需要暂存数据,所以配有累加器、状态寄存器等。控制器负责指挥协调。程序计数器(PC)记录下一条指令在哪,指令寄存器(IR)存放当前正在执行的指令,指令译码器解析指令,时序控制逻辑产生控制信号。**运算器和控制器合在一起就是 CPU**。存储器分两类：内存(主存)直接和 CPU 打交道,速度快但容量小,断电丢数据，而外存(硬盘、SSD)容量大但速度慢,断电数据保留。程序运行时必须先从外存加载到内存,CPU 才能访问。输入设备把外部信息送进来(键盘、鼠标、网卡),输出设备把处理结果送出去(显示器、打印机、音箱)。

## 结构关系

CPU 是计算机的"大脑",内部有寄存器(最快)、Cache(高速缓存)、ALU(运算单元)和控制单元。CPU 只能直接操作寄存器和 Cache,访问内存需要通过总线。

内存是 CPU 和硬盘之间的桥梁。程序运行时必须先从硬盘加载到内存,CPU 才能读取指令和数据。因为硬盘读取速度太慢(毫秒级)；而寄存器太小(几百字节)，又无法放下完整程序。而内存速度适中(纳秒级),容量适中(几 GB),正适合做桥梁。

硬盘(HDD/SSD)负责长期保存。容量最大(TB 级),速度最慢,但断电后数据还在。CPU 不能直接访问硬盘,必须先把数据搬到内存。

总线是连接各部件的公共通道。数据总线传输数据(双向),地址总线传输地址(单向,CPU 发出),控制总线传输控制信号(读/写等)。

## 程序执行过程

程序存在硬盘上，运行时，操作系统把程序从硬盘加载到内存，分配好代码段、数据段、堆、栈等空间，然后把程序入口地址写入 PC（程序计数器）。

**CPU 开始不断重复"取指→译码→执行"循环：**

**取指阶段**：根据 PC 中的地址从内存取出指令，放入指令寄存器（IR），PC 自动指向下一条。

**译码阶段**：指令译码器解析指令要做什么，操作数在哪。

**执行阶段**：如果需要访问内存就计算地址，从内存或寄存器读取操作数到寄存器，ALU 对寄存器中的数据执行运算，结果写回寄存器或内存。

**具体示例：执行 `c = a + b`**

```
1. 取指：从内存读取 LOAD 指令到 IR
2. 译码：识别为加载指令，目标是变量 a
3. 执行：从内存地址 0x1000 加载 a 的值到寄存器 R1

4. 取指：从内存读取 LOAD 指令到 IR
5. 译码：识别为加载指令，目标是变量 b
6. 执行：从内存地址 0x1004 加载 b 的值到寄存器 R2

7. 取指：从内存读取 ADD 指令到 IR
8. 译码：识别为加法指令，操作数 R1 和 R2
9. 执行：ALU 计算 R1 + R2，结果存入 R3

10. 取指：从内存读取 STORE 指令到 IR
11. 译码：识别为存储指令，源是 R3
12. 执行：把 R3 的值写回内存地址 0x1008（变量 c）
```

## 存储层次

CPU 执行指令时，数据在多层存储之间流动。

**寄存器—CPU 内部的临时存储**

寄存器是 CPU 内部的存储单元，速度最快（1 个时钟周期），容量最小（几十到几百字节）。ALU 只能对寄存器中的数据进行运算，不能直接对内存数据运算。这就是为什么上面的例子中，必须先把 a 和 b 从内存加载到寄存器 R1 和 R2，ALU 才能执行加法。常见寄存器有通用寄存器（R0-R15，存放操作数和运算结果）、程序计数器（PC，存放下一条指令地址）、指令寄存器（IR，存放当前指令）、状态寄存器（标志位，如进位、溢出）。

**Cache—CPU 和内存之间的缓冲**

内存虽然比硬盘快，但和 CPU 相比还是太慢。CPU 主频 3GHz，1 个时钟周期 0.33 纳秒；内存访问需要 50-100 纳秒，相当于 CPU 等待 150-300 个时钟周期。为了缓解这个速度差距，CPU 内部有 Cache（高速缓存）Cache 分三级：L1 Cache（最快最小，几十 KB，1-2 个时钟周期）、L2 Cache（中等，几百 KB，10-20 个时钟周期）、L3 Cache（最慢最大，几 MB，30-50 个时钟周期）。CPU 访问数据时，先查 L1，没有再查 L2，再没有查 L3，最后才访问内存。因为程序访问具有局部性——时间局部性：刚访问的数据很可能再次访问（循环变量）和空间局部性：相邻的数据很可能被访问（数组元素）。Cache 利用局部性实现有效性，把热点数据缓存起来，命中率通常在 90%-99%。

**内存—运行中程序的家**

内存（RAM）是 CPU 和硬盘之间的桥梁。容量几 GB，速度 50-100 纳秒，易失性（断电丢失）。程序运行时必须在内存中，CPU 才能访问。内存分为代码段（存放指令）、数据段（存放全局变量）、堆（动态分配）、栈（函数调用）。

**数据流动路径：**

```
硬盘（TB 级，毫秒级）
  ↓ 程序加载
内存（GB 级，纳秒级）
  ↓ CPU 读取
L3 Cache（MB 级，30-50 周期）
  ↓ 命中
L2 Cache（KB 级，10-20 周期）
  ↓ 命中
L1 Cache（KB 级，1-2 周期）
  ↓ 加载到
寄存器（字节级，1 周期）
  ↓ 送入
ALU 执行运算
  ↓ 结果写回
寄存器 → Cache → 内存 → 硬盘
```

**关键点：**
- CPU 只能直接操作寄存器，不能直接对内存数据运算
- 内存数据必须先加载到寄存器，ALU 才能运算
- Cache 在中间透明地加速数据访问，程序员通常不需要显式管理
- 离 CPU 越近越快越小越贵

## CPU 中断

CPU 执行程序时，可能被中断打断。中断是一种机制，让 CPU 暂停当前程序，处理紧急事件，然后恢复执行。

如果没有中断，CPU 只能轮询（不断检查）外设是否有事件。比如等待键盘输入，CPU 就得一直循环检查键盘，浪费大量时间。有了中断，CPU 可以继续执行其他程序，键盘有输入时发出中断信号，CPU 再处理。

**中断的类型：**

* **外中断（异步）**：来自 CPU 外部，与当前执行的指令无关。I/O 中断（硬盘读写完成、网卡收到数据）、时钟中断（定时器到期，用于进程调度）。

* **内中断（同步）**：来自 CPU 内部，与当前执行的指令有关。异常（除零错误、非法指令、缺页异常）、陷阱（系统调用，用户程序主动请求操作系统服务）。

**中断处理过程：**

1. **中断发生**：外设或 CPU 内部产生中断信号
2. **保存现场**：把当前 PC、寄存器、状态寄存器等保存到栈
3. **查找中断向量**：根据中断类型找到对应的中断处理程序地址
4. **执行中断处理程序**：处理中断事件（如读取键盘输入）
5. **恢复现场**：从栈恢复 PC、寄存器等
6. **继续执行**：CPU 回到被中断的地方继续执行

**中断与程序执行的关系：**

中断让 CPU 可以同时"处理"多个任务。实际上 CPU 同一时刻只能执行一条指令，但通过快速切换（时钟中断触发进程调度），给人一种"同时运行"的错觉。这就是多任务操作系统的基础。

**面试高频问题：中断和轮询的区别？**

轮询：CPU 主动不断检查外设，浪费 CPU 时间，但实现简单。  
中断：外设主动通知 CPU，CPU 可以做其他事，效率高，但实现复杂。

现代计算机主要用中断，只有少数实时性要求极高的场景用轮询。

## 一次完整的程序执行

现在把所有概念串起来，看一次完整的程序执行过程：

1. **程序装入**：操作系统把程序从硬盘加载到内存，分配代码段、数据段、堆、栈
2. **CPU 取指**：根据 PC 从内存读取指令到 IR（可能经过 Cache 加速）
3. **指令译码**：控制器解析指令，确定操作类型和操作数位置
4. **数据加载**：如果需要内存数据，从内存加载到寄存器（可能经过 Cache）
5. **ALU 运算**：对寄存器中的数据执行运算，结果存入寄存器
6. **结果写回**：如果需要持久化，把寄存器数据写回内存（可能经过 Cache）
7. **PC 更新**：PC 指向下一条指令，继续循环
8. **中断处理**：如果有中断（I/O 完成、时钟到期），保存现场，处理中断，恢复现场
9. **程序结束**：执行完毕，操作系统回收内存，程序退出

## 高频判断

**CPU 可以直接访问硬盘数据** → 错。CPU 不能直接访问硬盘,必须先把数据从硬盘加载到内存,再从内存加载到寄存器。

**程序运行时必须全部加载到内存** → 错。现代操作系统用虚拟内存技术,程序可以部分加载,需要时再从硬盘加载缺失部分。

**内存容量越大,CPU 运算速度越快** → 不完全对。内存大可以运行更多程序,但不会直接提升 CPU 运算速度。CPU 速度主要看主频、核心数、Cache 大小。

**冯·诺依曼体系的核心思想是"程序存储"** → 对。程序和数据都以二进制存在存储器中,程序可以像数据一样被修改。

**总线宽度越大,系统性能越好** → 基本对。总线宽度大,一次能传更多数据,但系统性能还受 CPU 主频、内存速度、硬盘速度等影响。

**CPU 执行指令的基本步骤是取指、译码、执行** → 对。这是指令周期的三个基本阶段。

**程序计数器(PC)存储的是当前正在执行的指令** → 错。PC 存的是**下一条**要执行的指令地址,当前指令在指令寄存器(IR)中。

**内存和硬盘都是存储器,可以互相替代** → 错。内存易失、快、小;硬盘非易失、慢、大。功能不同,不能替代。

**地址总线是双向的** → 错。地址总线单向,只能从 CPU 发出地址。数据总线才是双向的。

**运算器和控制器合称为 CPU** → 对。CPU = 运算器 + 控制器。
