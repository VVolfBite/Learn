## 主线

```
事务 ACID
    ↓
并发环境 → 脏读/不可重复读/幻读/丢失更新
    ↓
解决方案分支：
    ├─ 锁（共享锁/排他锁/行锁/表锁/封锁协议）
    ├─ 隔离级别（RU/RC/RR/Serializable）
    ├─ MVCC（版本链 + ReadView）
    └─ Next-Key Lock（间隙锁 + 记录锁）
    ↓
持久性保证 → redo log / undo log / WAL
    ↓
性能优化 → 索引（B+树/聚簇/非聚簇/覆盖/回表）
    ↓
查询优化 → 扫描/排序/临时表/explain
```

## 高频面试点分布

- **事务**：ACID 含义、ACID 之间关系、为什么需要事务
- **并发问题**：脏读/不可重复读/幻读的区别、发生场景
- **锁**：共享锁 vs 排他锁、行锁 vs 表锁、乐观锁 vs 悲观锁、意向锁作用、死锁处理
- **隔离级别**：四种级别分别解决什么问题、MySQL 默认隔离级别、不可重复读 vs 幻读
- **MVCC**：MVCC 原理、ReadView 机制、快照读 vs 当前读、MVCC 能否解决幻读
- **Next-Key Lock**：间隙锁原理、如何防止幻读、RR 下的加锁规则
- **日志**：redo log vs undo log、WAL 机制、crash recovery
- **索引**：B+树特点、聚簇索引 vs 非聚簇索引、回表、覆盖索引、最左前缀、索引失效场景
- **查询优化**：慢查询原因、explain 关键字段、如何优化

## 易混点提示

- 不可重复读 vs 幻读：前者是修改，后者是插入/删除
- MVCC vs 锁：不是对立关系，MVCC 用于读，锁用于写
- 快照读 vs 当前读：普通 SELECT 是快照读，UPDATE/DELETE/SELECT FOR UPDATE 是当前读
- 间隙锁 vs 临键锁：间隙锁锁间隙，临键锁 = 记录锁 + 间隙锁
- redo log vs undo log：redo 保证持久性，undo 保证原子性
- 聚簇索引 vs 非聚簇索引：聚簇索引叶子节点存数据，非聚簇存主键值
