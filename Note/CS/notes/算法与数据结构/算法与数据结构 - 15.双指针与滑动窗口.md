## 双指针

双指针用两个指针遍历数组或链表，根据问题特点选择不同的移动策略。

### 对撞指针

两个指针分别从数组两端向中间移动，常用于有序数组。

**两数之和（有序数组）**

```python
def two_sum(nums, target):
    left, right = 0, len(nums) - 1
    
    while left < right:
        curr_sum = nums[left] + nums[right]
        if curr_sum == target:
            return [left, right]
        elif curr_sum < target:
            left += 1
        else:
            right -= 1
    
    return []
```

因为数组有序，如果和太小就移动左指针，太大就移动右指针。

**反转字符串**

```python
def reverse_string(s):
    left, right = 0, len(s) - 1
    
    while left < right:
        s[left], s[right] = s[right], s[left]
        left += 1
        right -= 1
```

**判断回文串**

```python
def is_palindrome(s):
    left, right = 0, len(s) - 1
    
    while left < right:
        if s[left] != s[right]:
            return False
        left += 1
        right -= 1
    
    return True
```

### 快慢指针

两个指针以不同速度移动，常用于链表。

**链表中点**

```python
def find_middle(head):
    slow = fast = head
    
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
    
    return slow
```

快指针每次走两步，慢指针每次走一步。当快指针到达末尾时，慢指针正好在中点。

**检测链表环**

```python
def has_cycle(head):
    slow = fast = head
    
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            return True
    
    return False
```

如果有环，快指针最终会追上慢指针。

**找环的入口**

```python
def detect_cycle(head):
    slow = fast = head
    
    # 先找到相遇点
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            break
    else:
        return None
    
    # 一个指针从头开始，一个从相遇点开始
    # 它们会在环入口相遇
    slow = head
    while slow != fast:
        slow = slow.next
        fast = fast.next
    
    return slow
```

### 同向双指针

两个指针同向移动，通常用于原地修改数组。

**移除元素**

```python
def remove_element(nums, val):
    slow = 0
    
    for fast in range(len(nums)):
        if nums[fast] != val:
            nums[slow] = nums[fast]
            slow += 1
    
    return slow
```

快指针遍历数组，慢指针指向下一个要填入的位置。

**移动零**

```python
def move_zeroes(nums):
    slow = 0
    
    for fast in range(len(nums)):
        if nums[fast] != 0:
            nums[slow], nums[fast] = nums[fast], nums[slow]
            slow += 1
```

将所有非零元素移到前面，零自然就到后面了。

## 滑动窗口

滑动窗口是双指针的特殊应用，维护一个窗口在数组或字符串上滑动。窗口的左右边界都只向右移动，不回退。

### 固定窗口大小

**最大平均值**

```python
def find_max_average(nums, k):
    # 先计算第一个窗口的和
    window_sum = sum(nums[:k])
    max_sum = window_sum
    
    # 滑动窗口
    for i in range(k, len(nums)):
        window_sum = window_sum - nums[i - k] + nums[i]
        max_sum = max(max_sum, window_sum)
    
    return max_sum / k
```

### 可变窗口大小

**最长无重复子串**

```python
def length_of_longest_substring(s):
    char_set = set()
    left = 0
    max_len = 0
    
    for right in range(len(s)):
        # 如果有重复，收缩左边界
        while s[right] in char_set:
            char_set.remove(s[left])
            left += 1
        
        char_set.add(s[right])
        max_len = max(max_len, right - left + 1)
    
    return max_len
```

右指针不断扩展窗口，遇到重复字符时收缩左边界。

**最小覆盖子串**

给定字符串 s 和 t，找出 s 中包含 t 所有字符的最小子串。

```python
def min_window(s, t):
    from collections import Counter
    
    need = Counter(t)
    window = {}
    left = right = 0
    valid = 0
    start, length = 0, float('inf')
    
    while right < len(s):
        c = s[right]
        right += 1
        
        # 更新窗口
        if c in need:
            window[c] = window.get(c, 0) + 1
            if window[c] == need[c]:
                valid += 1
        
        # 收缩窗口
        while valid == len(need):
            # 更新结果
            if right - left < length:
                start = left
                length = right - left
            
            d = s[left]
            left += 1
            
            if d in need:
                if window[d] == need[d]:
                    valid -= 1
                window[d] -= 1
        
    return s[start:start + length] if length != float('inf') else ""
```

这是滑动窗口的经典应用，需要维护窗口内字符的计数。

**字符串的排列**

判断 s2 是否包含 s1 的排列。

```python
def check_inclusion(s1, s2):
    from collections import Counter
    
    need = Counter(s1)
    window = {}
    left = right = 0
    valid = 0
    
    while right < len(s2):
        c = s2[right]
        right += 1
        
        if c in need:
            window[c] = window.get(c, 0) + 1
            if window[c] == need[c]:
                valid += 1
        
        # 窗口大小固定为 len(s1)
        while right - left >= len(s1):
            if valid == len(need):
                return True
            
            d = s2[left]
            left += 1
            
            if d in need:
                if window[d] == need[d]:
                    valid -= 1
                window[d] -= 1
    
    return False
```

## 滑动窗口模板

滑动窗口的问题通常遵循这个模板：

```python
def sliding_window(s):
    window = {}
    left = right = 0
    
    while right < len(s):
        # 扩大窗口
        c = s[right]
        right += 1
        # 更新窗口数据
        window[c] = window.get(c, 0) + 1
        
        # 判断是否需要收缩窗口
        while 需要收缩:
            # 缩小窗口
            d = s[left]
            left += 1
            # 更新窗口数据
            window[d] -= 1
```

关键是确定：
1. 什么时候扩大窗口（通常是每次迭代）
2. 什么时候收缩窗口（根据具体条件）
3. 什么时候更新结果（扩大时或收缩时）

## 什么时候用双指针和滑动窗口

读到“什么时候用双指针和滑动窗口”时，先别急着记结论。更有效的方式是先弄清它在“算法与数据结构 - 15.双指针与滑动窗口”这一张卡片里到底解决什么问题，这样后面的分点和对比才不至于显得太散。

**双指针适合**：
- 有序数组的查找问题
- 链表问题（快慢指针）
- 原地修改数组
- 需要从两端向中间处理

**滑动窗口适合**：
- 连续子数组或子串问题
- 求最长/最短满足条件的子串
- 求固定大小窗口的最值

**不适合的场景**：
- 需要回溯的问题
- 需要考虑所有子集的问题
- 数据无序且不能排序

## 常见面试问题

**Q1：双指针为什么能优化时间复杂度？**

暴力方法需要两层循环，时间复杂度 O(N²)。双指针利用问题的特性（如有序性），让两个指针协同移动，每个元素最多访问一次，时间复杂度降到 O(N)。

**Q2：滑动窗口的本质是什么？**

滑动窗口是双指针的特殊应用，维护一个满足条件的区间。关键是理解窗口的扩大和收缩时机。

**Q3：如何判断是否需要收缩窗口？**

根据具体问题。常见的条件有：
- 窗口内出现重复元素
- 窗口大小达到限制
- 窗口内元素满足某个条件

**Q4：快慢指针为什么能检测环？**

如果有环，快指针会在环内循环，最终追上慢指针。如果没有环，快指针会先到达末尾。
