## 并查集

并查集（Union-Find）用于处理不相交集合的合并和查询问题。它支持两种操作：
- **查找（Find）**：判断两个元素是否属于同一集合
- **合并（Union）**：将两个集合合并为一个

### 基本实现

```python
class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n
    
    def find(self, x):
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])  # 路径压缩
        return self.parent[x]
    
    def union(self, x, y):
        px, py = self.find(x), self.find(y)
        if px == py:
            return False
        
        # 按秩合并
        if self.rank[px] < self.rank[py]:
            px, py = py, px
        self.parent[py] = px
        if self.rank[px] == self.rank[py]:
            self.rank[px] += 1
        
        return True
    
    def connected(self, x, y):
        return self.find(x) == self.find(y)
```

### 路径压缩

在查找过程中，将路径上的所有节点直接连接到根节点，使树变得扁平。

```python
def find(self, x):
    if self.parent[x] != x:
        self.parent[x] = self.find(self.parent[x])
    return self.parent[x]
```

### 按秩合并

合并时，将秩小的树连接到秩大的树上，避免树退化成链表。

```python
def union(self, x, y):
    px, py = self.find(x), self.find(y)
    if px == py:
        return False
    
    if self.rank[px] < self.rank[py]:
        px, py = py, px
    self.parent[py] = px
    if self.rank[px] == self.rank[py]:
        self.rank[px] += 1
    
    return True
```

有了这两个优化，并查集的时间复杂度接近 O(1)（准确地说是 O(α(N))，α 是反阿克曼函数，增长极慢）。

### 应用场景

**判断图的连通性**

```python
def count_components(n, edges):
    uf = UnionFind(n)
    
    for u, v in edges:
        uf.union(u, v)
    
    # 统计有多少个不同的根节点
    return len(set(uf.find(i) for i in range(n)))
```

**检测无向图的环**

```python
def has_cycle(n, edges):
    uf = UnionFind(n)
    
    for u, v in edges:
        if uf.connected(u, v):
            return True  # 已经连通，再加边就有环
        uf.union(u, v)
    
    return False
```

**朋友圈问题**

```python
def find_circle_num(is_connected):
    n = len(is_connected)
    uf = UnionFind(n)
    
    for i in range(n):
        for j in range(i + 1, n):
            if is_connected[i][j]:
                uf.union(i, j)
    
    return len(set(uf.find(i) for i in range(n)))
```

## Trie（前缀树）

Trie 是一种树形结构，用于高效存储和检索字符串。每个节点代表一个字符，从根到某个节点的路径表示一个字符串的前缀。

### 基本实现

```python
class TrieNode:
    def __init__(self):
        self.children = {}
        self.is_end = False

class Trie:
    def __init__(self):
        self.root = TrieNode()
    
    def insert(self, word):
        node = self.root
        for char in word:
            if char not in node.children:
                node.children[char] = TrieNode()
            node = node.children[char]
        node.is_end = True
    
    def search(self, word):
        node = self.root
        for char in word:
            if char not in node.children:
                return False
            node = node.children[char]
        return node.is_end
    
    def starts_with(self, prefix):
        node = self.root
        for char in prefix:
            if char not in node.children:
                return False
            node = node.children[char]
        return True
```

### 时间复杂度

这一节主要把“时间复杂度”放回“算法与数据结构 - 16.并查集与Trie”这条主线里看。先抓住它在整个算法与数据结构知识体系中的位置，再去看后面的分类和结论，会更容易把零散的判断串起来。

- 插入：O(L)，L 是字符串长度
- 查找：O(L)
- 前缀匹配：O(L)

相比哈希表，Trie 的优势是支持前缀查询和字典序遍历。

### 应用场景

**自动补全**

```python
def autocomplete(trie, prefix):
    node = trie.root
    for char in prefix:
        if char not in node.children:
            return []
        node = node.children[char]
    
    # 从当前节点开始，收集所有单词
    results = []
    def dfs(node, path):
        if node.is_end:
            results.append(prefix + path)
        for char, child in node.children.items():
            dfs(child, path + char)
    
    dfs(node, "")
    return results
```

**单词搜索 II**

在二维网格中搜索多个单词，用 Trie 可以避免重复搜索。

```python
def find_words(board, words):
    # 构建 Trie
    trie = Trie()
    for word in words:
        trie.insert(word)
    
    result = set()
    m, n = len(board), len(board[0])
    
    def dfs(i, j, node, path):
        if node.is_end:
            result.add(path)
        
        if i < 0 or i >= m or j < 0 or j >= n:
            return
        
        char = board[i][j]
        if char not in node.children:
            return
        
        board[i][j] = '#'  # 标记已访问
        for di, dj in [(0,1), (1,0), (0,-1), (-1,0)]:
            dfs(i+di, j+dj, node.children[char], path+char)
        board[i][j] = char  # 恢复
    
    for i in range(m):
        for j in range(n):
            dfs(i, j, trie.root, "")
    
    return list(result)
```

## 并查集 vs 其他数据结构

这里真正要说明的是“并查集 vs 其他数据结构”为什么值得单独拿出来讲。把它放到“算法与数据结构 - 16.并查集与Trie”这张卡片的上下文里看，后面的要点和区别会更容易读顺。

**并查集 vs DFS/BFS**：
- 并查集：适合动态连通性问题，支持在线查询
- DFS/BFS：适合静态图的遍历

**并查集 vs 哈希表**：
- 并查集：支持集合合并，查询两个元素是否在同一集合
- 哈希表：只能查询单个元素

## Trie vs 其他数据结构

读到“Trie vs 其他数据结构”时，先别急着记结论。更有效的方式是先弄清它在“算法与数据结构 - 16.并查集与Trie”这一张卡片里到底解决什么问题，这样后面的分点和对比才不至于显得太散。

**Trie vs 哈希表**：
- Trie：支持前缀查询，空间可能更大
- 哈希表：查询更快，但不支持前缀查询

**Trie vs 二叉搜索树**：
- Trie：按字符查找，适合字符串
- BST：按大小关系查找，适合数值

## 常见面试问题

**Q1：并查集的时间复杂度？**

有路径压缩和按秩合并的并查集，单次操作的时间复杂度是 O(α(N))，α 是反阿克曼函数，实际应用中可以认为是常数。

**Q2：什么时候用并查集？**

动态连通性问题，需要频繁判断两个元素是否连通，或合并两个集合。

**Q3：Trie 的空间复杂度？**

最坏情况 O(N × L × C)，N 是单词数，L 是平均长度，C 是字符集大小。实际应用中，由于前缀共享，空间会小很多。

**Q4：什么时候用 Trie？**

需要前缀匹配、自动补全、字典序遍历等场景。如果只是查找单词是否存在，哈希表更简单。
