
# 算法与数据结构 - 动态规划

动态规划是一种通过保存子问题的解来避免重复计算的算法思想，本质是"空间换时间"。它适用于具有最优子结构和重叠子问题的问题，核心是定义状态和找到状态转移方程。

动态规划通常从递归开始，通过记忆化搜索优化，最后改写为自底向上的动态规划。相比递归，动态规划避免了重复计算，时间复杂度通常从指数级降到多项式级。

面试中动态规划的考点主要是：如何判断能用动态规划、如何定义状态和写出状态转移方程、背包问题的各种变形、以及动态规划和贪心的区别。

## 1. 动态规划基础

判断一个问题能否用动态规划，需要检查两个必要条件：最优子结构和重叠子问题。

**最优子结构**：问题的最优解包含子问题的最优解，可以通过子问题的最优解推导出原问题的最优解。

**重叠子问题**：子问题会被重复计算多次，可以通过保存子问题的解来避免重复计算。

动态规划问题通常有这些特征：求最优解（最大、最小、最长、最短）、求方案数、求可行性。

1. **定义状态**：`dp[i]` 或 `dp[i][j]` 表示什么
2. **找状态转移方程**：`dp[i]` 如何从之前的状态推导
3. **确定边界条件**：`dp[0]` 或 `dp[0][0]` 的初始值
4. **确定计算顺序**：从小到大还是从大到小
5. **优化空间复杂度**：能否用滚动数组

## 2. 经典问题

### 2.1 斐波那契数列

**问题**：F(0) = 0, F(1) = 1, F(n) = F(n-1) + F(n-2)

**方法 1：递归（指数级）**
```python
def fib_recursive(n):
    if n <= 1:
        return n
    return fib_recursive(n - 1) + fib_recursive(n - 2)
# 时间复杂度：O(2^n)
```

**方法 2：记忆化搜索（自顶向下）**
```python
def fib_memo(n, memo={}):
    if n <= 1:
        return n
    if n in memo:
        return memo[n]
    memo[n] = fib_memo(n - 1, memo) + fib_memo(n - 2, memo)
    return memo[n]
# 时间复杂度：O(n)
```

**方法 3：动态规划（自底向上）**
```python
def fib_dp(n):
    if n <= 1:
        return n
    dp = [0] * (n + 1)
    dp[1] = 1
    for i in range(2, n + 1):
        dp[i] = dp[i - 1] + dp[i - 2]
    return dp[n]
# 时间复杂度：O(n)，空间复杂度：O(n)
```

**方法 4：空间优化**
```python
def fib_optimized(n):
    if n <= 1:
        return n
    prev, curr = 0, 1
    for _ in range(2, n + 1):
        prev, curr = curr, prev + curr
    return curr
# 时间复杂度：O(n)，空间复杂度：O(1)
```

### 2.2 爬楼梯

**问题**：每次可以爬 1 或 2 个台阶，爬到第 n 阶有多少种方法

**分析**：
- 状态定义：`dp[i]` = 爬到第 i 阶的方法数
- 状态转移：`dp[i] = dp[i-1] + dp[i-2]`
- 边界条件：`dp[0] = 1, dp[1] = 1`

**代码**：
```python
def climb_stairs(n):
    if n <= 1:
        return 1
    dp = [0] * (n + 1)
    dp[0] = dp[1] = 1
    for i in range(2, n + 1):
        dp[i] = dp[i - 1] + dp[i - 2]
    return dp[n]
```

**本质**：斐波那契数列

### 2.3 打家劫舍

**问题**：不能偷相邻的房子，求最大金额

**分析**：
- 状态定义：`dp[i]` = 前 i 个房子能偷的最大金额
- 状态转移：`dp[i] = max(dp[i-1], dp[i-2] + nums[i])`
  - 不偷第 i 个：`dp[i-1]`
  - 偷第 i 个：`dp[i-2] + nums[i]`
- 边界条件：`dp[0] = nums[0], dp[1] = max(nums[0], nums[1])`

**代码**：
```python
def rob(nums):
    if not nums:
        return 0
    if len(nums) == 1:
        return nums[0]
    
    dp = [0] * len(nums)
    dp[0] = nums[0]
    dp[1] = max(nums[0], nums[1])
    
    for i in range(2, len(nums)):
        dp[i] = max(dp[i - 1], dp[i - 2] + nums[i])
    
    return dp[-1]
```

**空间优化**：
```python
def rob_optimized(nums):
    if not nums:
        return 0
    prev, curr = 0, 0
    for num in nums:
        prev, curr = curr, max(curr, prev + num)
    return curr
```

## 3. 背包问题

背包问题是动态规划的经典问题，有多种变形。核心思路是：对于每个物品，决定选还是不选。

### 3.1 0-1 背包

0-1 背包问题是：有 n 个物品，每个物品有重量 w[i] 和价值 v[i]，背包容量为 W，每个物品只能选一次，求最大价值。

状态定义：`dp[i][j]` 表示前 i 个物品、容量为 j 时的最大价值。

状态转移：对于第 i 个物品，可以选或不选：
- 不选：`dp[i][j] = dp[i-1][j]`
- 选（如果放得下）：`dp[i][j] = dp[i-1][j-w[i]] + v[i]`

取两者的最大值。

**代码**：
```python
def knapsack_01(W, weights, values):
    n = len(weights)
    dp = [[0] * (W + 1) for _ in range(n + 1)]
    
    for i in range(1, n + 1):
        for j in range(W + 1):
            # 不选第 i 个物品
            dp[i][j] = dp[i - 1][j]
            # 选第 i 个物品（如果放得下）
            if j >= weights[i - 1]:
                dp[i][j] = max(dp[i][j], dp[i - 1][j - weights[i - 1]] + values[i - 1])
    
    return dp[n][W]
```

**空间优化（滚动数组）**：
```python
def knapsack_01_optimized(W, weights, values):
    dp = [0] * (W + 1)
    
    for i in range(len(weights)):
        # 从后往前遍历，避免重复使用
        for j in range(W, weights[i] - 1, -1):
            dp[j] = max(dp[j], dp[j - weights[i]] + values[i])
    
    return dp[W]
```

### 3.2 完全背包

完全背包和 0-1 背包的区别是：每个物品可以选无限次。

状态转移的关键变化：选第 i 个物品时，不是从 `dp[i-1][j-w[i]]` 转移，而是从 `dp[i][j-w[i]]` 转移。因为 `dp[i]` 表示可以重复选第 i 个物品。

在空间优化的一维数组实现中，0-1 背包从后往前遍历（避免重复使用），完全背包从前往后遍历（允许重复使用）。这是两者最直观的区别。

**代码**：
```python
def knapsack_complete(W, weights, values):
    dp = [0] * (W + 1)
    
    for i in range(len(weights)):
        # 从前往后遍历，允许重复使用
        for j in range(weights[i], W + 1):
            dp[j] = max(dp[j], dp[j - weights[i]] + values[i])
    
    return dp[W]
```

**0-1 背包 vs 完全背包**：
- 0-1 背包：从后往前遍历
- 完全背包：从前往后遍历

### 3.3 多重背包

**问题**：每个物品有数量限制 count[i]

**方法 1：转化为 0-1 背包**
```python
def knapsack_multiple(W, weights, values, counts):
    # 展开成 0-1 背包
    new_weights = []
    new_values = []
    for i in range(len(weights)):
        for _ in range(counts[i]):
            new_weights.append(weights[i])
            new_values.append(values[i])
    
    return knapsack_01_optimized(W, new_weights, new_values)
```

**方法 2：二进制优化**
```python
def knapsack_multiple_binary(W, weights, values, counts):
    new_weights = []
    new_values = []
    
    for i in range(len(weights)):
        count = counts[i]
        k = 1
        while k <= count:
            new_weights.append(k * weights[i])
            new_values.append(k * values[i])
            count -= k
            k *= 2
        if count > 0:
            new_weights.append(count * weights[i])
            new_values.append(count * values[i])
    
    return knapsack_01_optimized(W, new_weights, new_values)
```

## 4. 线性 DP

### 4.1 最长递增子序列（LIS）

**问题**：找到数组中最长的严格递增子序列的长度

**方法 1：DP（O(N²)）**
```python
def length_of_lis(nums):
    if not nums:
        return 0
    
    n = len(nums)
    dp = [1] * n  # dp[i] = 以 nums[i] 结尾的 LIS 长度
    
    for i in range(1, n):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)
    
    return max(dp)
```

**方法 2：贪心 + 二分（O(N log N)）**
```python
def length_of_lis_binary(nums):
    if not nums:
        return 0
    
    tails = []  # tails[i] = 长度为 i+1 的 LIS 的最小尾部元素
    
    for num in nums:
        # 二分查找插入位置
        left, right = 0, len(tails)
        while left < right:
            mid = (left + right) // 2
            if tails[mid] < num:
                left = mid + 1
            else:
                right = mid
        
        if left == len(tails):
            tails.append(num)
        else:
            tails[left] = num
    
    return len(tails)
```

### 4.2 最长公共子序列（LCS）

**问题**：找到两个字符串的最长公共子序列

**分析**：
- 状态定义：`dp[i][j]` = text1[0:i] 和 text2[0:j] 的 LCS 长度
- 状态转移：
  ```
  if text1[i-1] == text2[j-1]:
      dp[i][j] = dp[i-1][j-1] + 1
  else:
      dp[i][j] = max(dp[i-1][j], dp[i][j-1])
  ```

**代码**：
```python
def longest_common_subsequence(text1, text2):
    m, n = len(text1), len(text2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if text1[i - 1] == text2[j - 1]:
                dp[i][j] = dp[i - 1][j - 1] + 1
            else:
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])
    
    return dp[m][n]
```

### 4.3 编辑距离

**问题**：将 word1 转换为 word2 的最少操作次数（插入、删除、替换）

**分析**：
- 状态定义：`dp[i][j]` = word1[0:i] 转换为 word2[0:j] 的最少操作数
- 状态转移：
  ```
  if word1[i-1] == word2[j-1]:
      dp[i][j] = dp[i-1][j-1]
  else:
      dp[i][j] = 1 + min(
          dp[i-1][j],    # 删除
          dp[i][j-1],    # 插入
          dp[i-1][j-1]   # 替换
      )
  ```

**代码**：
```python
def min_distance(word1, word2):
    m, n = len(word1), len(word2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    
    # 边界条件
    for i in range(m + 1):
        dp[i][0] = i
    for j in range(n + 1):
        dp[0][j] = j
    
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if word1[i - 1] == word2[j - 1]:
                dp[i][j] = dp[i - 1][j - 1]
            else:
                dp[i][j] = 1 + min(dp[i - 1][j], dp[i][j - 1], dp[i - 1][j - 1])
    
    return dp[m][n]
```

## 5. 区间 DP

**特点**：状态定义为区间 `dp[i][j]`，通常枚举区间长度

### 5.1 最长回文子串

**问题**：找到字符串中最长的回文子串

**分析**：
- 状态定义：`dp[i][j]` = s[i:j+1] 是否是回文
- 状态转移：
  ```
  if s[i] == s[j]:
      dp[i][j] = dp[i+1][j-1]  (if j - i > 1)
      dp[i][j] = True          (if j - i <= 1)
  else:
      dp[i][j] = False
  ```

**代码**：
```python
def longest_palindrome(s):
    n = len(s)
    if n < 2:
        return s
    
    dp = [[False] * n for _ in range(n)]
    start, max_len = 0, 1
    
    # 单个字符都是回文
    for i in range(n):
        dp[i][i] = True
    
    # 枚举区间长度
    for length in range(2, n + 1):
        for i in range(n - length + 1):
            j = i + length - 1
            if s[i] == s[j]:
                if length == 2:
                    dp[i][j] = True
                else:
                    dp[i][j] = dp[i + 1][j - 1]
            
            if dp[i][j] and length > max_len:
                start = i
                max_len = length
    
    return s[start:start + max_len]
```

## 6. 高频判断

### 判断 1：如何判断能用动态规划？

**检查清单**：
1. 是否求最优解（最大、最小、最长、最短）
2. 是否有重叠子问题（递归会重复计算）
3. 是否有最优子结构（子问题的最优解能推导出原问题的最优解）
4. 是否可以定义状态和写出状态转移方程

### 判断 2：动态规划 vs 贪心？

| 对比项 | 动态规划 | 贪心 |
|--------|---------|------|
| **适用条件** | 最优子结构 + 重叠子问题 | 贪心选择性质 + 最优子结构 |
| **决策** | 考虑所有可能 | 只考虑当前最优 |
| **时间复杂度** | 通常 O(N²) 或 O(N³) | 通常 O(N) 或 O(N log N) |
| **正确性** | 一定正确 | 需要证明 |

**例子**：
- 背包问题：动态规划（贪心不对）
- 活动选择：贪心（动态规划也可以但没必要）

### 判断 3：如何定义状态？

这里真正要说明的是“判断 3：如何定义状态？”为什么值得单独拿出来讲。把它放到“算法与数据结构 - 11.动态规划”这张卡片的上下文里看，后面的要点和区别会更容易读顺。

**常见状态定义**：
- 一维：`dp[i]` = 前 i 个元素的最优解
- 二维：`dp[i][j]` = 前 i 个元素，容量为 j 的最优解
- 区间：`dp[i][j]` = 区间 [i, j] 的最优解

**技巧**：
- 状态要能表示子问题
- 状态要能推导出原问题
- 状态数量要合理（不能太多）

### 判断 4：如何优化空间复杂度？

这里真正要说明的是“判断 4：如何优化空间复杂度？”为什么值得单独拿出来讲。把它放到“算法与数据结构 - 11.动态规划”这张卡片的上下文里看，后面的要点和区别会更容易读顺。

**滚动数组**：
- 如果 `dp[i]` 只依赖 `dp[i-1]`，可以用两个变量
- 如果 `dp[i][j]` 只依赖 `dp[i-1][j]`，可以用一维数组

**注意**：
- 0-1 背包：从后往前遍历
- 完全背包：从前往后遍历

### 判断 5：常见面试问题

**Q1：动态规划和递归的区别？**
- 递归是自顶向下，动态规划是自底向上
- 递归有重复计算，动态规划避免重复计算
- 递归可能栈溢出，动态规划不会

**Q2：什么时候用记忆化搜索？**
- 状态转移不好写时
- 不是所有状态都会被访问时
- 递归更直观时

**Q3：如何打印动态规划的路径？**
- 记录每个状态的来源
- 从终点回溯到起点

**Q4：动态规划一定比递归快吗？**
- 不一定，如果状态很多但只访问少数状态，记忆化搜索可能更快
- 但动态规划空间局部性更好，缓存友好

## 8. 面试技巧

### 口头回答模板

**问：什么是动态规划？**

答：动态规划是一种通过保存子问题的解来避免重复计算的算法思想。它适用于具有最优子结构和重叠子问题的问题。核心是定义状态和找到状态转移方程。动态规划通常从递归开始，通过记忆化搜索优化，最后改写为自底向上的动态规划。相比递归，动态规划避免了重复计算，时间复杂度通常从指数级降到多项式级。

**问：如何解决动态规划问题？**

答：解决动态规划问题通常分五步：
1. 定义状态：确定 dp[i] 或 dp[i][j] 表示什么
2. 找状态转移方程：dp[i] 如何从之前的状态推导
3. 确定边界条件：dp[0] 的初始值
4. 确定计算顺序：从小到大还是从大到小
5. 优化空间复杂度：能否用滚动数组

**Q：0-1 背包和完全背包的区别？**
- 0-1 背包每个物品只能选一次，从后往前遍历
- 完全背包每个物品可以选无限次，从前往后遍历
- 核心区别在于状态转移方程中是 dp[i-1] 还是 dp[i]

**Q：如何优化动态规划的空间？**
- 如果 dp[i] 只依赖 dp[i-1]，可以用滚动数组
- 如果 dp[i][j] 只依赖 dp[i-1][j]，可以用一维数组
- 注意遍历顺序，避免覆盖还需要用的状态

**Q：动态规划的时间复杂度如何分析？**
- 状态数量 × 每个状态的计算时间
- 例如：二维 DP 通常是 O(N²)，三维 DP 通常是 O(N³)
