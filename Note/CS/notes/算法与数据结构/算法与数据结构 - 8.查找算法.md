## 顺序查找

从头到尾遍历数组，逐个比较元素。时间复杂度 O(N)，适用于无序数组或数据量很小的情况。

```python
def linear_search(arr, target):
    for i in range(len(arr)):
        if arr[i] == target:
            return i
    return -1
```

顺序查找简单直接，但效率低。如果数据有序，可以用二分查找将时间复杂度降到 O(log N)。

## 二分查找

二分查找要求数组有序。每次比较中间元素，根据大小关系排除一半的搜索空间，直到找到目标或搜索空间为空。

### 基本模板

```python
def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    
    while left <= right:
        mid = left + (right - left) // 2
        
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    
    return -1
```

**关键点**：
- `mid = left + (right - left) // 2` 避免整数溢出
- 循环条件是 `left <= right`，注意等号
- 更新边界时是 `mid + 1` 或 `mid - 1`，不包含 mid

### 边界问题

二分查找最容易出错的地方是边界处理。根据不同需求，有几种变体：

**1. 查找第一个等于目标的位置**

```python
def binary_search_first(arr, target):
    left, right = 0, len(arr) - 1
    result = -1
    
    while left <= right:
        mid = left + (right - left) // 2
        
        if arr[mid] == target:
            result = mid
            right = mid - 1  # 继续向左找
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    
    return result
```

**2. 查找最后一个等于目标的位置**

```python
def binary_search_last(arr, target):
    left, right = 0, len(arr) - 1
    result = -1
    
    while left <= right:
        mid = left + (right - left) // 2
        
        if arr[mid] == target:
            result = mid
            left = mid + 1  # 继续向右找
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    
    return result
```

**3. 查找第一个大于等于目标的位置**

```python
def binary_search_lower_bound(arr, target):
    left, right = 0, len(arr)
    
    while left < right:
        mid = left + (right - left) // 2
        
        if arr[mid] < target:
            left = mid + 1
        else:
            right = mid
    
    return left
```

**4. 查找第一个大于目标的位置**

```python
def binary_search_upper_bound(arr, target):
    left, right = 0, len(arr)
    
    while left < right:
        mid = left + (right - left) // 2
        
        if arr[mid] <= target:
            left = mid + 1
        else:
            right = mid
    
    return left
```

### 边界模板的选择

这一节看起来往往容易变成一串结论，其实先抓住“边界模板的选择”在算法与数据结构里的作用会更稳。后面的内容可以理解成围绕这个问题展开的几种常见判断和做法。

**模板 1**：`left <= right`，更新时 `left = mid + 1` 或 `right = mid - 1`
- 适合查找确定存在的元素
- 循环结束时 `left > right`

**模板 2**：`left < right`，更新时 `left = mid + 1` 或 `right = mid`
- 适合查找边界位置
- 循环结束时 `left == right`

选择哪个模板取决于具体问题。关键是保持一致性，避免死循环和遗漏元素。

## 二分查找的应用

### 旋转数组查找

数组原本有序，但在某个位置旋转了。例如 `[4,5,6,7,0,1,2]` 是 `[0,1,2,4,5,6,7]` 旋转后的结果。

```python
def search_rotated(arr, target):
    left, right = 0, len(arr) - 1
    
    while left <= right:
        mid = left + (right - left) // 2
        
        if arr[mid] == target:
            return mid
        
        # 判断哪一半是有序的
        if arr[left] <= arr[mid]:
            # 左半部分有序
            if arr[left] <= target < arr[mid]:
                right = mid - 1
            else:
                left = mid + 1
        else:
            # 右半部分有序
            if arr[mid] < target <= arr[right]:
                left = mid + 1
            else:
                right = mid - 1
    
    return -1
```

### 查找峰值

峰值元素是指其值大于左右相邻值的元素。数组可能包含多个峰值，返回任何一个即可。

```python
def find_peak_element(arr):
    left, right = 0, len(arr) - 1
    
    while left < right:
        mid = left + (right - left) // 2
        
        if arr[mid] > arr[mid + 1]:
            # 峰值在左侧（包括 mid）
            right = mid
        else:
            # 峰值在右侧
            left = mid + 1
    
    return left
```

### 在二维矩阵中查找

如果矩阵每行从左到右递增，每列从上到下递增，可以从右上角或左下角开始查找。

```python
def search_matrix(matrix, target):
    if not matrix or not matrix[0]:
        return False
    
    row, col = 0, len(matrix[0]) - 1
    
    while row < len(matrix) and col >= 0:
        if matrix[row][col] == target:
            return True
        elif matrix[row][col] > target:
            col -= 1
        else:
            row += 1
    
    return False
```

时间复杂度 O(M + N)，M 和 N 分别是行数和列数。

## 什么时候用二分查找

这一节主要把“什么时候用二分查找”放回“算法与数据结构 - 8.查找算法”这条主线里看。先抓住它在整个算法与数据结构知识体系中的位置，再去看后面的分类和结论，会更容易把零散的判断串起来。

**必须满足的条件**：
- 数据有序（或部分有序）
- 可以随机访问（通常是数组）

**适合的场景**：
- 在有序数组中查找元素
- 查找边界位置
- 在答案空间中搜索（二分答案）

**不适合的场景**：
- 数据无序且不能排序
- 链表等不支持随机访问的结构
- 数据量很小（顺序查找更简单）

## 二分答案

有些问题的答案在某个范围内，且具有单调性。可以用二分查找在答案空间中搜索。

**例子**：在 D 天内送达包裹的最低运载能力

```python
def ship_within_days(weights, days):
    def can_ship(capacity):
        # 判断给定运载能力能否在 days 天内送完
        need_days = 1
        curr_weight = 0
        for w in weights:
            if curr_weight + w > capacity:
                need_days += 1
                curr_weight = w
            else:
                curr_weight += w
        return need_days <= days
    
    # 二分查找最小运载能力
    left, right = max(weights), sum(weights)
    
    while left < right:
        mid = left + (right - left) // 2
        if can_ship(mid):
            right = mid
        else:
            left = mid + 1
    
    return left
```

这类问题的关键是：
1. 确定答案的范围（left 和 right）
2. 写出判断函数（给定答案是否可行）
3. 根据判断结果调整搜索范围

## 常见面试问题

**Q1：二分查找的时间复杂度为什么是 O(log N)？**

每次比较后搜索空间减半，最多需要 log₂N 次比较。例如 1024 个元素最多需要 10 次比较（2¹⁰ = 1024）。

**Q2：为什么 mid = left + (right - left) // 2？**

直接写 `(left + right) // 2` 在某些语言中可能整数溢出。`left + (right - left) // 2` 等价但更安全。

**Q3：循环条件用 `left <= right` 还是 `left < right`？**

取决于具体问题。`left <= right` 适合查找确定存在的元素，`left < right` 适合查找边界。关键是保持边界更新的一致性。

**Q4：二分查找能用于链表吗？**

理论上可以，但效率很低。链表不支持随机访问，找到中间节点需要 O(N) 时间，总时间复杂度退化为 O(N log N)，不如顺序查找。

**Q5：如何处理重复元素？**

根据需求选择合适的边界模板。查找第一个等于目标的位置时，找到后继续向左搜索；查找最后一个时，继续向右搜索。
