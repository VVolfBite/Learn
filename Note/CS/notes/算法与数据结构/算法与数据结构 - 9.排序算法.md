
# 算法与数据结构 - 排序算法

排序是将一组数据按特定顺序重新排列的过程，是最基础也最常用的算法之一。不同排序算法在时间复杂度、空间复杂度、稳定性上各有特点，没有"最好"的排序算法，只有"最适合"的排序算法。

面试中排序的考点主要是：快速排序和归并排序的原理、两者的区别、稳定性的概念、以及如何根据场景选择合适的排序算法。实际工程中最常用的是快速排序（数组）、归并排序（链表、需要稳定性）、堆排序（内存受限）。

## 1. 排序算法总览

### 1.1 十大排序算法

| 排序算法 | 平均时间 | 最坏时间 | 最好时间 | 空间复杂度 | 稳定性 |
|---------|---------|---------|---------|-----------|--------|
| **冒泡排序** | O(N²) | O(N²) | O(N) | O(1) | 稳定 |
| **选择排序** | O(N²) | O(N²) | O(N²) | O(1) | 不稳定 |
| **插入排序** | O(N²) | O(N²) | O(N) | O(1) | 稳定 |
| **希尔排序** | O(N^1.3) | O(N²) | O(N) | O(1) | 不稳定 |
| **快速排序** | O(N log N) | O(N²) | O(N log N) | O(log N) | 不稳定 |
| **归并排序** | O(N log N) | O(N log N) | O(N log N) | O(N) | 稳定 |
| **堆排序** | O(N log N) | O(N log N) | O(N log N) | O(1) | 不稳定 |
| **计数排序** | O(N+K) | O(N+K) | O(N+K) | O(K) | 稳定 |
| **桶排序** | O(N+K) | O(N²) | O(N) | O(N+K) | 稳定 |
| **基数排序** | O(N×K) | O(N×K) | O(N×K) | O(N+K) | 稳定 |

**K**：数据范围或位数

### 1.2 稳定性

**定义**：相等元素的相对顺序在排序前后保持不变

**示例**：
```
原始数据：(3, a), (2, b), (3, c), (1, d)
稳定排序：(1, d), (2, b), (3, a), (3, c)  // (3,a) 仍在 (3,c) 前面
不稳定排序：(1, d), (2, b), (3, c), (3, a)  // (3,a) 和 (3,c) 顺序可能改变
```

**为什么重要**：
- 多关键字排序时需要稳定性
- 例如：先按年龄排序，再按姓名排序，需要保持年龄相同时姓名的顺序

## 2. 简单排序（O(N²)）

### 2.1 冒泡排序

**思想**：相邻元素两两比较，大的往后移，每轮确定一个最大值

**代码**：
```python
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        swapped = False
        for j in range(n - 1 - i):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                swapped = True
        if not swapped:  # 优化：如果没有交换，说明已经有序
            break
    return arr
```

**特点**：
- 稳定
- 原地排序
- 最好情况 O(N)（已经有序）
- 实际很少使用

### 2.2 选择排序

**思想**：每轮选择最小值，放到已排序部分的末尾

**代码**：
```python
def selection_sort(arr):
    n = len(arr)
    for i in range(n):
        min_idx = i
        for j in range(i + 1, n):
            if arr[j] < arr[min_idx]:
                min_idx = j
        arr[i], arr[min_idx] = arr[min_idx], arr[i]
    return arr
```

**特点**：
- 不稳定（交换可能破坏相对顺序）
- 原地排序
- 交换次数少（最多 N 次）
- 无论什么情况都是 O(N²)

### 2.3 插入排序

**思想**：将元素插入到已排序部分的合适位置

**代码**：
```python
def insertion_sort(arr):
    n = len(arr)
    for i in range(1, n):
        key = arr[i]
        j = i - 1
        # 将大于 key 的元素后移
        while j >= 0 and arr[j] > key:
            arr[j + 1] = arr[j]
            j -= 1
        arr[j + 1] = key
    return arr
```

**特点**：
- 稳定
- 原地排序
- 最好情况 O(N)（已经有序）
- 对小规模数据或基本有序的数据效率高
- 实际应用：Timsort（Python 的内置排序）在小规模数据时使用插入排序

### 2.4 希尔排序

**思想**：插入排序的改进，先进行大间隔的插入排序，逐步缩小间隔

**代码**：
```python
def shell_sort(arr):
    n = len(arr)
    gap = n // 2
    
    while gap > 0:
        for i in range(gap, n):
            temp = arr[i]
            j = i
            while j >= gap and arr[j - gap] > temp:
                arr[j] = arr[j - gap]
                j -= gap
            arr[j] = temp
        gap //= 2
    
    return arr
```

**特点**：
- 不稳定
- 原地排序
- 性能取决于间隔序列的选择
- 实际应用较少

## 3. 高效排序（O(N log N)）

### 3.1 快速排序

快速排序是实际应用最广泛的排序算法。它的核心思想是"分治"：选择一个基准元素（pivot），将数组分为小于基准和大于基准的两部分，然后递归地对这两部分排序。

快速排序的关键在于分区（partition）操作。一次分区后，基准元素会被放到正确的位置，左边都比它小，右边都比它大。这样每次分区都能确定一个元素的最终位置。

**代码**：
```python
def quick_sort(arr):
    if len(arr) <= 1:
        return arr
    
    pivot = arr[len(arr) // 2]
    left = [x for x in arr if x < pivot]
    middle = [x for x in arr if x == pivot]
    right = [x for x in arr if x > pivot]
    
    return quick_sort(left) + middle + quick_sort(right)
```

**原地快排**（更常用）：
```python
def quick_sort_inplace(arr, low, high):
    if low < high:
        pi = partition(arr, low, high)
        quick_sort_inplace(arr, low, pi - 1)
        quick_sort_inplace(arr, pi + 1, high)

def partition(arr, low, high):
    pivot = arr[high]
    i = low - 1
    
    for j in range(low, high):
        if arr[j] <= pivot:
            i += 1
            arr[i], arr[j] = arr[j], arr[i]
    
    arr[i + 1], arr[high] = arr[high], arr[i + 1]
    return i + 1
```

**特点**：
- 不稳定
- 原地排序（递归栈 O(log N)）
- 平均 O(N log N)，最坏 O(N²)（已排序或逆序）
- 实际应用最广泛（C++ sort、Java Arrays.sort 对基本类型）

快速排序为什么快？主要有三个原因：一是缓存友好（顺序访问数组），二是原地排序（不需要额外空间），三是内层循环简单（常数因子小）。虽然最坏情况是 O(N²)，但通过随机化或三数取中可以有效避免。

**常见优化**：
1. **三数取中**：选择首、中、尾三个元素的中位数作为基准
2. **随机化**：随机选择基准，避免最坏情况
3. **小数组优化**：数组小于阈值时改用插入排序
4. **三路快排**：处理大量重复元素

### 3.2 归并排序

归并排序是典型的分治算法。它的思路是：将数组分成两半，递归地对每一半排序，然后将两个有序数组合并成一个有序数组。

归并排序的关键在于合并（merge）操作。合并两个有序数组很简单：用两个指针分别指向两个数组的开头，每次选择较小的元素放入结果数组，直到所有元素都被处理。

**代码**：
```python
def merge_sort(arr):
    if len(arr) <= 1:
        return arr
    
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    
    return merge(left, right)

def merge(left, right):
    result = []
    i = j = 0
    
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    
    result.extend(left[i:])
    result.extend(right[j:])
    return result
```

**特点**：
- 稳定
- 非原地排序（需要 O(N) 额外空间）
- 任何情况都是 O(N log N)
- 适合链表排序（不需要随机访问）
- 实际应用：Java Arrays.sort 对对象类型、Python Timsort 的基础

归并排序和快速排序都是 O(N log N) 的分治算法，但策略相反：归并排序是"先分后治"（先递归分解，再合并），快速排序是"先治后分"（先分区，再递归）。归并排序的优势是稳定且最坏情况也是 O(N log N)，劣势是需要额外空间。

**归并排序 vs 快速排序**：

| 对比项 | 归并排序 | 快速排序 |
|--------|---------|---------|
| **稳定性** | 稳定 | 不稳定 |
| **空间复杂度** | O(N) | O(log N) |
| **最坏时间** | O(N log N) | O(N²) |
| **平均性能** | 稍慢 | 更快 |
| **适用场景** | 需要稳定性、链表 | 数组、不需要稳定性 |

### 3.3 堆排序

堆排序利用堆这种数据结构来排序。它的思路是：先将数组构建成大根堆（父节点大于子节点），然后每次取出堆顶元素（最大值）放到数组末尾，再调整剩余元素为堆。重复这个过程直到所有元素都被取出。

堆排序的优势是原地排序且任何情况都是 O(N log N)，不会像快速排序那样退化。但它的劣势是不稳定，且因为跳跃访问内存（父子节点在数组中不相邻），缓存不友好，实际运行速度比快速排序慢。

**代码**：
```python
def heap_sort(arr):
    n = len(arr)
    
    # 构建大根堆
    for i in range(n // 2 - 1, -1, -1):
        heapify(arr, n, i)
    
    # 依次取出堆顶
    for i in range(n - 1, 0, -1):
        arr[0], arr[i] = arr[i], arr[0]
        heapify(arr, i, 0)
    
    return arr

def heapify(arr, n, i):
    largest = i
    left = 2 * i + 1
    right = 2 * i + 2
    
    if left < n and arr[left] > arr[largest]:
        largest = left
    if right < n and arr[right] > arr[largest]:
        largest = right
    
    if largest != i:
        arr[i], arr[largest] = arr[largest], arr[i]
        heapify(arr, n, largest)
```

**特点**：
- 不稳定
- 原地排序
- 任何情况都是 O(N log N)
- 不需要递归栈
- 实际应用：优先队列、Top K 问题

## 4. 线性排序（O(N)）

### 4.1 计数排序

**思想**：统计每个值出现的次数，按顺序输出

**代码**：
```python
def counting_sort(arr):
    if not arr:
        return arr
    
    max_val = max(arr)
    min_val = min(arr)
    range_size = max_val - min_val + 1
    
    # 计数
    count = [0] * range_size
    for num in arr:
        count[num - min_val] += 1
    
    # 累加
    for i in range(1, range_size):
        count[i] += count[i - 1]
    
    # 输出
    output = [0] * len(arr)
    for num in reversed(arr):  # 从后往前保证稳定性
        index = count[num - min_val] - 1
        output[index] = num
        count[num - min_val] -= 1
    
    return output
```

**特点**：
- 稳定
- 非原地排序
- 时间复杂度 O(N+K)，K 是数据范围
- 适用于数据范围不大的整数排序

### 4.2 桶排序

**思想**：将数据分到若干个桶中，每个桶内部排序，最后合并

**代码**：
```python
def bucket_sort(arr):
    if not arr:
        return arr
    
    # 确定桶的数量
    bucket_count = len(arr)
    max_val = max(arr)
    min_val = min(arr)
    
    # 创建桶
    buckets = [[] for _ in range(bucket_count)]
    
    # 分配到桶
    for num in arr:
        index = int((num - min_val) / (max_val - min_val + 1) * bucket_count)
        if index == bucket_count:
            index -= 1
        buckets[index].append(num)
    
    # 每个桶内部排序
    for bucket in buckets:
        bucket.sort()  # 可以用插入排序
    
    # 合并桶
    result = []
    for bucket in buckets:
        result.extend(bucket)
    
    return result
```

**特点**：
- 稳定
- 非原地排序
- 平均 O(N+K)，最坏 O(N²)
- 适用于数据均匀分布的情况

### 4.3 基数排序

**思想**：按位排序，从低位到高位依次排序

**代码**：
```python
def radix_sort(arr):
    if not arr:
        return arr
    
    max_val = max(arr)
    exp = 1
    
    while max_val // exp > 0:
        counting_sort_by_digit(arr, exp)
        exp *= 10
    
    return arr

def counting_sort_by_digit(arr, exp):
    n = len(arr)
    output = [0] * n
    count = [0] * 10
    
    # 计数
    for num in arr:
        index = (num // exp) % 10
        count[index] += 1
    
    # 累加
    for i in range(1, 10):
        count[i] += count[i - 1]
    
    # 输出
    for i in range(n - 1, -1, -1):
        index = (arr[i] // exp) % 10
        output[count[index] - 1] = arr[i]
        count[index] -= 1
    
    for i in range(n):
        arr[i] = output[i]
```

**特点**：
- 稳定
- 非原地排序
- 时间复杂度 O(N×K)，K 是位数
- 适用于整数或字符串排序

## 5. 高频判断

### 判断 1：如何选择排序算法？

| 场景 | 推荐算法 | 原因 |
|------|---------|------|
| **小规模数据（N < 50）** | 插入排序 | 简单高效 |
| **基本有序** | 插入排序 | O(N) |
| **需要稳定性** | 归并排序 | 稳定且 O(N log N) |
| **不需要稳定性** | 快速排序 | 平均最快 |
| **内存受限** | 堆排序 | 原地排序 |
| **数据范围小** | 计数排序 | O(N+K) |
| **链表排序** | 归并排序 | 不需要随机访问 |

### 判断 2：稳定排序有哪些？

**稳定**：冒泡、插入、归并、计数、桶、基数
**不稳定**：选择、希尔、快速、堆

**记忆技巧**：
- 简单排序中，只有选择不稳定（因为交换可能跨越多个元素）
- 高效排序中，只有归并稳定
- 线性排序都稳定

### 判断 3：原地排序有哪些？

**原地**（O(1) 额外空间）：冒泡、选择、插入、希尔、快速、堆
**非原地**：归并、计数、桶、基数

**注意**：快速排序的递归栈需要 O(log N) 空间，但通常认为是原地排序

### 判断 4：快速排序为什么快？

这一节看起来往往容易变成一串结论，其实先抓住“判断 4：快速排序为什么快？”在算法与数据结构里的作用会更稳。后面的内容可以理解成围绕这个问题展开的几种常见判断和做法。

1. **缓存友好**：顺序访问数组，局部性好
2. **原地排序**：不需要额外空间
3. **常数因子小**：内层循环简单
4. **平均情况好**：虽然最坏 O(N²)，但平均 O(N log N) 且常数小

### 判断 5：什么时候快速排序会退化？

**退化情况**：
- 数组已经有序或逆序
- 每次选择的基准都是最小或最大值

**解决方法**：
- 随机选择基准
- 三数取中
- 三路快排（处理大量重复元素）

### 判断 6：归并排序和快速排序的区别？

| 对比项 | 归并排序 | 快速排序 |
|--------|---------|---------|
| **分治策略** | 先分后治 | 先治后分 |
| **稳定性** | 稳定 | 不稳定 |
| **空间** | O(N) | O(log N) |
| **最坏时间** | O(N log N) | O(N²) |
| **适用场景** | 链表、需要稳定性 | 数组、不需要稳定性 |

### 判断 7：堆排序的优缺点？

这里真正要说明的是“判断 7：堆排序的优缺点？”为什么值得单独拿出来讲。把它放到“算法与数据结构 - 9.排序算法”这张卡片的上下文里看，后面的要点和区别会更容易读顺。

**优点**：
- 原地排序
- 任何情况都是 O(N log N)
- 不需要递归栈

**缺点**：
- 不稳定
- 缓存不友好（跳跃访问）
- 常数因子大，实际比快排慢

### 判断 8：什么时候用线性排序？

这一节看起来往往容易变成一串结论，其实先抓住“判断 8：什么时候用线性排序？”在算法与数据结构里的作用会更稳。后面的内容可以理解成围绕这个问题展开的几种常见判断和做法。

**计数排序**：
- 数据范围小（K 不大）
- 整数排序
- 例如：年龄排序（0-150）

**桶排序**：
- 数据均匀分布
- 浮点数排序
- 例如：0-1 之间的随机数

**基数排序**：
- 整数或字符串
- 位数不多
- 例如：电话号码、身份证号

### 判断 9：常见面试问题

这一节看起来往往容易变成一串结论，其实先抓住“判断 9：常见面试问题”在算法与数据结构里的作用会更稳。后面的内容可以理解成围绕这个问题展开的几种常见判断和做法。

**Q1：如何对链表排序？**
- 归并排序（不需要随机访问）
- 时间 O(N log N)，空间 O(log N)（递归栈）

**Q2：如何找第 K 大的元素？**
- 快速选择算法（快排的变形）
- 平均 O(N)，最坏 O(N²)
- 或者用堆，O(N log K)

**Q3：如何判断数组是否有序？**
- 遍历一遍，检查相邻元素
- O(N)

**Q4：如何对几乎有序的数组排序？**
- 插入排序，O(N)
- 或者堆排序，O(N log K)，K 是最大位移

**Q5：如何实现稳定的快速排序？**
- 不能原地排序，需要额外空间
- 或者改用归并排序

## 7. 面试技巧

### 口头回答模板

**问：快速排序的原理是什么？**

答：快速排序是一种分治算法。它的核心思想是选择一个基准元素，将数组分为小于基准和大于基准的两部分，然后递归地对这两部分进行排序。平均时间复杂度是 O(N log N)，但最坏情况下会退化为 O(N²)，比如数组已经有序时。为了避免最坏情况，可以随机选择基准或使用三数取中法。快速排序是原地排序，空间复杂度是 O(log N)（递归栈），但它不是稳定排序。

**问：归并排序和快速排序有什么区别？**

答：两者都是分治算法，时间复杂度都是 O(N log N)，但有几个关键区别：
1. 归并排序是"先分后治"，快速排序是"先治后分"
2. 归并排序是稳定排序，快速排序不稳定
3. 归并排序需要 O(N) 额外空间，快速排序只需要 O(log N)
4. 归并排序任何情况都是 O(N log N)，快速排序最坏是 O(N²)
5. 实际应用中，快速排序通常更快，因为缓存友好且常数因子小

### 常见追问

这一节看起来往往容易变成一串结论，其实先抓住“常见追问”在算法与数据结构里的作用会更稳。后面的内容可以理解成围绕这个问题展开的几种常见判断和做法。

**Q：为什么 Python 的 sort 用 Timsort？**
- Timsort 是归并排序和插入排序的混合
- 稳定排序，适合对象排序
- 对部分有序的数据效率很高
- 最坏 O(N log N)，最好 O(N)

**Q：为什么 C++ 的 sort 用快速排序？**
- 实际上是 Introsort（快排 + 堆排序 + 插入排序）
- 快排平均最快
- 当递归深度过大时切换到堆排序，避免 O(N²)
- 小数组时用插入排序

**Q：如何优化排序算法？**
- 小数组用插入排序
- 三数取中或随机化基准
- 三路快排处理重复元素
- 尾递归优化
- 并行化（多线程）

## 8. 代码模板

### 快速排序模板

```python
def quick_sort(arr, low, high):
    if low < high:
        pi = partition(arr, low, high)
        quick_sort(arr, low, pi - 1)
        quick_sort(arr, pi + 1, high)

def partition(arr, low, high):
    pivot = arr[high]
    i = low - 1
    for j in range(low, high):
        if arr[j] <= pivot:
            i += 1
            arr[i], arr[j] = arr[j], arr[i]
    arr[i + 1], arr[high] = arr[high], arr[i + 1]
    return i + 1
```

### 归并排序模板

```python
def merge_sort(arr):
    if len(arr) <= 1:
        return arr
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    return merge(left, right)

def merge(left, right):
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    result.extend(left[i:])
    result.extend(right[j:])
    return result
```

### 堆排序模板

```python
def heap_sort(arr):
    n = len(arr)
    for i in range(n // 2 - 1, -1, -1):
        heapify(arr, n, i)
    for i in range(n - 1, 0, -1):
        arr[0], arr[i] = arr[i], arr[0]
        heapify(arr, i, 0)
    return arr

def heapify(arr, n, i):
    largest = i
    left = 2 * i + 1
    right = 2 * i + 2
    if left < n and arr[left] > arr[largest]:
        largest = left
    if right < n and arr[right] > arr[largest]:
        largest = right
    if largest != i:
        arr[i], arr[largest] = arr[largest], arr[i]
        heapify(arr, n, largest)
```
