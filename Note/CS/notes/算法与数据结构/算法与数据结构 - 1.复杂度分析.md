
# 算法与数据结构 - 复杂度分析

## 1. 时间复杂度

### 1.1 定义

时间复杂度是算法执行时间随数据规模增长的变化趋势，用于衡量算法的执行效率。

### 1.2 渐进表示法

使用 `~f(N)` 来表示所有随着 N 的增大除以 f(N) 的结果趋近于 1 的函数。

例如：N³/6 - N²/2 + N/3 ~ N³/6

### 1.3 Big O 表示法

**Big O (大O表示法)**：表示算法时间复杂度的上界，即最坏情况下的时间复杂度。

- 定义：如果存在常数 c 和 n₀，使得对所有 n ≥ n₀，都有 T(n) ≤ c·f(n)，则 T(n) = O(f(n))
- 含义：算法的运行时间不会超过某个上限
- 例子：N³/6 - N²/2 + N/3 的增长数量级为 O(N³)

### 1.4 Big Omega 表示法

**Big Omega (大Ω表示法)**：表示算法时间复杂度的下界，即最好情况下的时间复杂度。

- 定义：如果存在常数 c 和 n₀，使得对所有 n ≥ n₀，都有 T(n) ≥ c·f(n)，则 T(n) = Ω(f(n))
- 含义：算法的运行时间至少需要某个下限
- 例子：快速排序的最好情况是 Ω(N log N)

### 1.5 Big Theta 表示法

**Big Theta (大Θ表示法)**：表示算法时间复杂度的确界，即平均情况下的时间复杂度。

- 定义：如果 T(n) = O(f(n)) 且 T(n) = Ω(f(n))，则 T(n) = Θ(f(n))
- 含义：算法的运行时间紧密地被某个函数限定
- 例子：归并排序在所有情况下都是 Θ(N log N)

### 1.6 常见复杂度量级对比

按照从低到高的顺序：

| 复杂度 | 名称 | 示例 |
| :---: | :---: | :---: |
| O(1) | 常数级 | 数组访问、哈希表查找 |
| O(log N) | 对数级 | 二分查找 |
| O(N) | 线性级 | 遍历数组 |
| O(N log N) | 线性对数级 | 快速排序、归并排序 |
| O(N²) | 平方级 | 冒泡排序、选择排序 |
| O(N³) | 立方级 | 三重循环 |
| O(2ᴺ) | 指数级 | 递归求斐波那契数列 |
| O(N!) | 阶乘级 | 全排列 |

**性能对比**（N = 1000 时）：

- O(1) = 1
- O(log N) ≈ 10
- O(N) = 1,000
- O(N log N) ≈ 10,000
- O(N²) = 1,000,000
- O(N³) = 1,000,000,000
- O(2ᴺ) ≈ 10³⁰⁰（不可计算）

### 1.7 最好、最坏、平均情况

这一节看起来往往容易变成一串结论，其实先抓住“1.7 最好、最坏、平均情况”在算法与数据结构里的作用会更稳。后面的内容可以理解成围绕这个问题展开的几种常见判断和做法。

**最好情况（Best Case）**：
- 算法在最理想输入下的时间复杂度
- 例：插入排序对已排序数组的时间复杂度为 O(N)

**最坏情况（Worst Case）**：
- 算法在最不利输入下的时间复杂度
- 例：快速排序对已排序数组的时间复杂度为 O(N²)

**平均情况（Average Case）**：
- 算法在所有可能输入下的期望时间复杂度
- 例：快速排序的平均时间复杂度为 O(N log N)

### 1.8 递归复杂度基础

递归算法的时间复杂度通常通过递推关系式求解。

**主定理（Master Theorem）**：

对于递推式 T(n) = aT(n/b) + f(n)，其中 a ≥ 1, b > 1：

1. 如果 f(n) = O(n^(log_b(a) - ε))，则 T(n) = Θ(n^log_b(a))
2. 如果 f(n) = Θ(n^log_b(a))，则 T(n) = Θ(n^log_b(a) · log n)
3. 如果 f(n) = Ω(n^(log_b(a) + ε))，则 T(n) = Θ(f(n))

**常见递归复杂度**：

```python
# 二分查找：T(n) = T(n/2) + O(1) = O(log N)
def binary_search(arr, target, left, right):
    if left > right:
        return -1
    mid = (left + right) // 2
    if arr[mid] == target:
        return mid
    elif arr[mid] > target:
        return binary_search(arr, target, left, mid - 1)
    else:
        return binary_search(arr, target, mid + 1, right)

# 归并排序：T(n) = 2T(n/2) + O(n) = O(N log N)
def merge_sort(arr):
    if len(arr) <= 1:
        return arr
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    return merge(left, right)

# 斐波那契（朴素递归）：T(n) = T(n-1) + T(n-2) + O(1) = O(2^N)
def fibonacci(n):
    if n <= 1:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)
```

## 2. 空间复杂度

### 2.1 定义

空间复杂度是算法执行过程中所需的额外存储空间随数据规模增长的变化趋势。

### 2.2 空间复杂度分类

这一节看起来往往容易变成一串结论，其实先抓住“2.2 空间复杂度分类”在算法与数据结构里的作用会更稳。后面的内容可以理解成围绕这个问题展开的几种常见判断和做法。

**固定空间**：
- 代码本身占用的空间
- 常量、简单变量占用的空间

**可变空间**：
- 递归调用栈的深度
- 动态分配的内存（数组、对象等）

### 2.3 常见空间复杂度

| 复杂度 | 说明 | 示例 |
| :---: | :---: | :---: |
| O(1) | 常数空间 | 只使用有限个变量 |
| O(log N) | 对数空间 | 递归深度为 log N |
| O(N) | 线性空间 | 需要额外的 N 大小数组 |
| O(N²) | 平方空间 | 需要 N×N 的二维数组 |

**示例**：

```python
# O(1) 空间复杂度
def sum_array(arr):
    total = 0  # 只使用一个变量
    for num in arr:
        total += num
    return total

# O(N) 空间复杂度
def copy_array(arr):
    new_arr = []  # 创建新数组
    for num in arr:
        new_arr.append(num)
    return new_arr

# O(log N) 空间复杂度（递归栈）
def binary_search_recursive(arr, target, left, right):
    if left > right:
        return -1
    mid = (left + right) // 2
    if arr[mid] == target:
        return mid
    elif arr[mid] > target:
        return binary_search_recursive(arr, target, left, mid - 1)
    else:
        return binary_search_recursive(arr, target, mid + 1, right)
```

## 3. 算法分析方法

### 3.1 数学模型

#### 近似

在分析算法时，可以忽略低阶项和常数系数，只关注最高阶项。

例如：N³/6 - N²/2 + N/3 ~ N³/6

#### 增长数量级

增长数量级将算法与它的具体实现隔离开来，一个算法的增长数量级为 O(N³) 与它是否用 Java 实现、是否运行于特定计算机上无关。

#### 内循环

执行最频繁的指令决定了程序执行的总时间，把这些指令称为程序的内循环。

#### 成本模型

使用成本模型来评估算法，例如数组的访问次数就是一种成本模型。

### 3.2 倍率实验

如果 T(N) ~ aN^b logN，那么 T(2N)/T(N) ~ 2^b。

**ThreeSum 示例**：

```java
public class ThreeSumSlow {
    public int count(int[] nums) {
        int N = nums.length;
        int cnt = 0;
        for (int i = 0; i < N; i++) {
            for (int j = i + 1; j < N; j++) {
                for (int k = j + 1; k < N; k++) {
                    if (nums[i] + nums[j] + nums[k] == 0) {
                        cnt++;
                    }
                }
            }
        }
        return cnt;
    }
}
```

该算法的内循环为 `if (nums[i] + nums[j] + nums[k] == 0)` 语句，总共执行的次数为 N(N-1)(N-2)/6 = N³/6 - N²/2 + N/3，因此它的近似执行次数为 ~N³/6，增长数量级为 O(N³)。

**倍率实验结果**：

| N | Time(ms) | Ratio |
| :---: | :---: | :---: |
| 500 | 48 | / |
| 1000 | 320 | 6.7 |
| 2000 | 555 | 1.7 |
| 4000 | 4105 | 7.4 |
| 8000 | 33575 | 8.2 |
| 16000 | 268909 | 8.0 |

可以看到，T(2N)/T(N) ~ 2³ = 8，因此可以确定 T(N) ~ aN³。

### 3.3 注意事项

#### 大常数

在求近似时，如果低级项的常数系数很大，那么近似的结果可能在较小的 N 值下是错误的。

#### 缓存

计算机系统会使用缓存技术来组织内存，访问数组相邻的元素会比访问不相邻的元素快很多。

#### 对最坏情况下的性能的保证

在核反应堆、心脏起搏器或者刹车控制器中的软件，最坏情况下的性能是十分重要的。

#### 随机化算法

通过打乱输入，去除算法对输入的依赖。

#### 均摊分析

将所有操作的总成本除以操作总数来将成本均摊。

例如对一个空栈进行 N 次连续的 push() 调用需要访问数组的次数为 N+4+8+16+...+2N=5N-4（N 是向数组写入元素的次数，其余都是调整数组大小时进行复制需要的访问数组次数），均摊后访问数组的平均次数为常数。

## 4. 高频判断

### 4.1 如何快速判断时间复杂度？

这一节看起来往往容易变成一串结论，其实先抓住“4.1 如何快速判断时间复杂度？”在算法与数据结构里的作用会更稳。后面的内容可以理解成围绕这个问题展开的几种常见判断和做法。

**看循环层数**：
- 单层循环：O(N)
- 双层循环：O(N²)
- 三层循环：O(N³)

**看递归深度和每层工作量**：
- 二分递归 + 常数工作：O(log N)
- 二分递归 + 线性工作：O(N log N)
- 每层分裂成两个 + 常数工作：O(2^N)

**看数据结构操作**：
- 数组访问：O(1)
- 链表遍历：O(N)
- 二叉树遍历：O(N)
- 哈希表查找：平均 O(1)

### 4.2 常见面试问题

**Q1：为什么快速排序比归并排序快？**

A：虽然两者时间复杂度都是 O(N log N)，但快速排序的常数因子更小，且是原地排序（空间复杂度 O(log N)），而归并排序需要 O(N) 的额外空间。

**Q2：递归一定比循环慢吗？**

A：不一定。递归有函数调用开销和栈空间开销，但在某些情况下（如树的遍历），递归代码更简洁且性能相当。现代编译器可能会进行尾递归优化。

**Q3：O(N) 一定比 O(N log N) 快吗？**

A：不一定。大O表示法忽略了常数因子，如果 O(N) 算法的常数因子很大，在实际数据规模下可能比 O(N log N) 慢。

**Q4：如何优化算法的时间复杂度？**

A：常见方法：
- 使用哈希表将 O(N) 查找优化为 O(1)
- 使用排序 + 双指针将 O(N²) 优化为 O(N log N)
- 使用动态规划避免重复计算
- 使用空间换时间（记忆化）

**Q5：空间复杂度和时间复杂度如何权衡？**

A：
- 如果内存充足，可以用空间换时间（如哈希表、动态规划）
- 如果内存受限，优先考虑空间复杂度（如原地排序）
- 实际应用中需要根据具体场景权衡

## 5. 典型题

### 5.1 ThreeSum 问题

**问题**：统计一个数组中和为 0 的三元组数量。

**解法1：暴力枚举 - O(N³)**

```python
def three_sum_slow(nums):
    n = len(nums)
    count = 0
    for i in range(n):
        for j in range(i + 1, n):
            for k in range(j + 1, n):
                if nums[i] + nums[j] + nums[k] == 0:
                    count += 1
    return count
```

**解法2：排序 + 二分查找 - O(N² log N)**

```python
def three_sum_binary_search(nums):
    nums.sort()
    n = len(nums)
    count = 0
    for i in range(n):
        for j in range(i + 1, n):
            target = -(nums[i] + nums[j])
            # 二分查找 target
            left, right = j + 1, n - 1
            while left <= right:
                mid = (left + right) // 2
                if nums[mid] == target:
                    count += 1
                    break
                elif nums[mid] < target:
                    left = mid + 1
                else:
                    right = mid - 1
    return count
```

**解法3：排序 + 双指针 - O(N²)**

```python
def three_sum_two_pointer(nums):
    nums.sort()
    n = len(nums)
    count = 0
    for i in range(n - 2):
        left, right = i + 1, n - 1
        target = -nums[i]
        while left < right:
            sum_val = nums[left] + nums[right]
            if sum_val == target:
                count += 1
                left += 1
                right -= 1
            elif sum_val < target:
                left += 1
            else:
                right -= 1
    return count
```

### 5.2 复杂度分析练习

**练习1**：分析以下代码的时间复杂度

```python
def mystery(n):
    count = 0
    i = 1
    while i < n:
        j = 0
        while j < n:
            count += 1
            j += 1
        i *= 2
    return count
```

**答案**：O(N log N)
- 外层循环：i 每次乘以 2，执行 log N 次
- 内层循环：执行 N 次
- 总复杂度：O(N log N)

**练习2**：分析以下递归的时间复杂度

```python
def mystery(n):
    if n <= 1:
        return 1
    return mystery(n // 2) + mystery(n // 2)
```

**答案**：O(N)
- 递推式：T(n) = 2T(n/2) + O(1)
- 根据主定理：T(n) = Θ(n)
