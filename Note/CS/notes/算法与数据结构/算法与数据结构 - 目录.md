## 使用方法

这份笔记的核心是帮你快速建立算法知识体系，而不是从零学习。适合：
- 学过算法但遗忘较多，需要系统复习
- 准备面试，需要快速唤醒记忆
- 刷题遇到瓶颈，需要补充理论基础

**不适合**：完全没学过算法的初学者（建议先看教材或视频课程）

## 核心主线：从基础到进阶

算法面试的本质是考察你对数据结构和算法的理解深度。这份笔记按照"数据结构 → 基础算法 → 优化算法 → 技巧专题"的顺序组织，每个阶段都是后续内容的基础。

**第一层：理解数据结构的本质**  
数组、链表、栈、队列、哈希表是所有算法的基础。理解它们的增删改查复杂度，知道什么场景用什么结构。

**第二层：掌握树和图的遍历**  
树和图是最常考的数据结构。二叉树的前中后序遍历、BFS/DFS、最短路径算法是面试高频点。

**第三层：熟练基础算法**  
排序和查找是算法的基本功。快排、归并、二分查找必须能手写，理解时间复杂度的推导。

**第四层：理解优化思想**  
递归、回溯、动态规划、贪心是解决复杂问题的核心思想。重点不是背模板，而是理解状态转移和最优子结构。

**第五层：掌握常见技巧**  
双指针、滑动窗口、位运算、并查集是面试加分项。这些技巧能显著降低时间复杂度。

```
复杂度分析（基础）
    ↓
基础数据结构：数组/链表/栈/队列/哈希表
    ↓
树形结构：二叉树/BST/堆
    ↓
图结构：BFS/DFS/最短路径/最小生成树
    ↓
基础算法：查找/排序
    ↓
递归与分治：递归/回溯
    ↓
优化算法：动态规划/贪心
    ↓
技巧专题：位运算/双指针/滑动窗口
    ↓
进阶结构：并查集/Trie
    ↓
高频判断（面试冲刺）
```

## 面试复习策略

### 第一轮：建立知识体系（3-5 天）

**目标**：理解每个数据结构和算法的本质

1. 基础数据结构（1-2 天）
   - 数组/链表/栈/队列/哈希表
   - 理解各自的特点和适用场景

2. 树和图（1-2 天）
   - 二叉树遍历、BST、堆
   - BFS/DFS、最短路径

3. 基础算法（1 天）
   - 排序算法对比
   - 二分查找

### 第二轮：掌握核心算法（3-5 天）

**目标**：能独立实现和应用

1. 递归与回溯（1 天）
   - 理解递归思想
   - 掌握回溯框架

2. 动态规划（2-3 天）
   - 理解状态转移
   - 掌握常见 DP 类型

3. 贪心算法（1 天）
   - 理解贪心思想
   - 识别贪心题型

### 第三轮：技巧与刷题（5-7 天）

**目标**：提高解题速度和准确率

1. 双指针/滑动窗口（1-2 天）
2. 位运算技巧（1 天）
3. 并查集/Trie（1 天）
4. 刷 Leetcode 题解（2-3 天）

### 第四轮：面试冲刺（1-2 天）

**目标**：快速回忆和查漏补缺

1. 看"高频判断"（1 小时）
2. 回顾薄弱点（2-3 小时）
3. 模拟面试（1-2 小时）

## 高频面试考点

### 数据结构选型（必考）
- 什么时候用数组？什么时候用链表？
- 什么时候用栈？什么时候用队列？
- 什么时候用哈希表？
- 什么时候用堆？

### 算法复杂度（必考）
- 常见排序算法的时间复杂度
- 二分查找的时间复杂度
- BFS/DFS 的时间复杂度
- 动态规划的时间复杂度

### 经典算法（高频）
- 二叉树遍历（前中后序、层次）
- 链表反转
- 二分查找及其变形
- 快速排序
- 归并排序
- 最短路径（Dijkstra）
- 动态规划（背包、最长子序列）

### 技巧题型（加分）
- 双指针（快慢指针、左右指针）
- 滑动窗口
- 单调栈
- 位运算技巧

## 学习建议

### 初学者路线
1. 先学习基础数据结构：数组、链表、栈、队列
2. 掌握基本算法：排序、查找
3. 理解复杂度分析
4. 练习 Leetcode Easy 题目

### 进阶路线
1. 学习树、图、堆等高级数据结构
2. 掌握动态规划、贪心、回溯等算法思想
3. 理解各种算法的应用场景
4. 练习 Leetcode Medium 题目

### 面试准备路线
1. 系统复习所有知识点
2. 重点掌握高频考点
3. 练习典型题和变形题
4. 模拟面试，提高代码速度
5. 练习 Leetcode Hard 题目
