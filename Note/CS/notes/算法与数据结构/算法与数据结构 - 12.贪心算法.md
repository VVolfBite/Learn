
# 算法与数据结构 - 贪心算法

贪心算法每一步都选择当前看起来最优的选择，期望通过局部最优达到全局最优。它的优点是简单高效，缺点是不一定能得到正确答案，需要证明贪心选择的正确性。

## 贪心算法的特点

**贪心选择性质**：每一步做出的选择是当前状态下的最优选择，且这个选择不会影响后续问题的求解。

**最优子结构**：问题的最优解包含子问题的最优解。

贪心算法和动态规划都要求最优子结构，但贪心算法还需要贪心选择性质。动态规划会考虑所有可能的选择，而贪心算法只考虑当前最优的选择。

## 贪心 vs 动态规划

读到“贪心 vs 动态规划”时，先别急着记结论。更有效的方式是先弄清它在“算法与数据结构 - 12.贪心算法”这一张卡片里到底解决什么问题，这样后面的分点和对比才不至于显得太散。

**贪心算法**：
- 每步做局部最优选择
- 不回溯，不考虑其他可能
- 时间复杂度通常是 O(N) 或 O(N log N)
- 不一定能得到正确答案

**动态规划**：
- 考虑所有可能的选择
- 保存子问题的解
- 时间复杂度通常是 O(N²) 或更高
- 一定能得到正确答案

**例子**：背包问题
- 0-1 背包：必须用动态规划，贪心不对
- 分数背包：可以用贪心，按性价比排序

## 经典问题

### 活动选择问题

有 n 个活动，每个活动有开始时间和结束时间。选择最多的活动，使得它们不重叠。

**贪心策略**：每次选择结束时间最早的活动。

```python
def activity_selection(activities):
    # activities: [(start, end), ...]
    # 按结束时间排序
    activities.sort(key=lambda x: x[1])
    
    result = [activities[0]]
    last_end = activities[0][1]
    
    for start, end in activities[1:]:
        if start >= last_end:
            result.append((start, end))
            last_end = end
    
    return result
```

为什么选择结束时间最早的？因为这样能为后续活动留出更多时间。

### 分数背包问题

与 0-1 背包不同，物品可以取一部分。贪心策略是按性价比（价值/重量）排序，优先取性价比高的。

```python
def fractional_knapsack(items, capacity):
    # items: [(value, weight), ...]
    # 按性价比排序
    items.sort(key=lambda x: x[0] / x[1], reverse=True)
    
    total_value = 0
    
    for value, weight in items:
        if capacity >= weight:
            # 全部取走
            total_value += value
            capacity -= weight
        else:
            # 取一部分
            total_value += value * (capacity / weight)
            break
    
    return total_value
```

### 跳跃游戏

给定数组，每个元素表示在该位置可以跳跃的最大长度。判断能否到达最后一个位置。

**贪心策略**：维护当前能到达的最远位置。

```python
def can_jump(nums):
    max_reach = 0
    
    for i in range(len(nums)):
        if i > max_reach:
            return False
        max_reach = max(max_reach, i + nums[i])
        if max_reach >= len(nums) - 1:
            return True
    
    return True
```

### 跳跃游戏 II

找到到达最后一个位置的最少跳跃次数。

**贪心策略**：在当前跳跃范围内，选择能跳得最远的位置。

```python
def jump(nums):
    if len(nums) <= 1:
        return 0
    
    jumps = 0
    current_end = 0
    farthest = 0
    
    for i in range(len(nums) - 1):
        farthest = max(farthest, i + nums[i])
        
        if i == current_end:
            jumps += 1
            current_end = farthest
            
            if current_end >= len(nums) - 1:
                break
    
    return jumps
```

### 买卖股票的最佳时机 II

可以多次买卖股票，但同一时间只能持有一股。求最大利润。

**贪心策略**：只要第二天价格比今天高，就今天买明天卖。

```python
def max_profit(prices):
    profit = 0
    
    for i in range(1, len(prices)):
        if prices[i] > prices[i - 1]:
            profit += prices[i] - prices[i - 1]
    
    return profit
```

这个策略等价于抓住所有上涨区间。

### 分发饼干

有一群孩子和一些饼干，每个孩子有胃口值，每块饼干有大小。饼干大小大于等于孩子胃口值才能满足。求最多能满足多少孩子。

**贪心策略**：将孩子和饼干都排序，用最小的饼干满足胃口最小的孩子。

```python
def find_content_children(children, cookies):
    children.sort()
    cookies.sort()
    
    child = cookie = 0
    
    while child < len(children) and cookie < len(cookies):
        if cookies[cookie] >= children[child]:
            child += 1
        cookie += 1
    
    return child
```

### 无重叠区间

给定多个区间，找到需要移除的最少区间数，使剩余区间不重叠。

**贪心策略**：按结束时间排序，保留结束时间最早的区间。

```python
def erase_overlap_intervals(intervals):
    if not intervals:
        return 0
    
    intervals.sort(key=lambda x: x[1])
    
    count = 0
    end = intervals[0][1]
    
    for i in range(1, len(intervals)):
        if intervals[i][0] < end:
            # 重叠，移除当前区间
            count += 1
        else:
            # 不重叠，更新结束时间
            end = intervals[i][1]
    
    return count
```

## 如何判断能用贪心

贪心算法的难点在于判断问题是否适合用贪心，以及找到正确的贪心策略。

**适合贪心的特征**：
1. 问题可以分解为子问题
2. 子问题的最优解能推导出原问题的最优解
3. 局部最优选择不会影响后续选择

**常见的贪心策略**：
- 排序后选择（活动选择、区间问题）
- 维护最值（跳跃游戏）
- 贪心地选择当前最优（分数背包）

**如何验证贪心策略**：
1. 尝试构造反例
2. 用数学归纳法证明
3. 与动态规划对比（如果 DP 也能做，贪心通常更快）

## 贪心算法的局限

贪心算法不是万能的。很多问题看起来可以用贪心，但实际上贪心得不到正确答案。

**例子**：0-1 背包问题

如果用贪心（按性价比排序），可能得不到最优解：
- 背包容量 10
- 物品 A：价值 60，重量 10
- 物品 B：价值 100，重量 20
- 物品 C：价值 120，重量 30

按性价比排序：A(6) > B(5) > C(4)，贪心选 A，总价值 60。
但最优解是不选任何物品（因为 B 和 C 都放不下），或者如果容量够大，选 C 更优。

这个例子说明，贪心算法需要仔细分析问题的性质，不能盲目使用。

## 常见面试问题

**Q1：贪心算法和动态规划的区别？**

贪心算法每步做局部最优选择，不回溯；动态规划考虑所有可能，保存子问题的解。贪心更快但不一定正确，动态规划一定正确但更慢。

**Q2：如何判断问题能用贪心？**

检查是否满足贪心选择性质和最优子结构。尝试构造反例，如果找不到反例，可能可以用贪心。但严格来说需要数学证明。

**Q3：贪心算法的时间复杂度？**

通常是 O(N) 或 O(N log N)（如果需要排序）。比动态规划的 O(N²) 或 O(N³) 快很多。

**Q4：什么时候贪心比动态规划好？**

当问题满足贪心选择性质时，贪心更简单高效。但如果不确定，用动态规划更保险。
