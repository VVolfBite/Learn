
# 算法与数据结构 - 树

树是一种层次结构的数据结构，由节点和边组成，没有环路。每个节点最多有一个父节点，可以有多个子节点。树的递归性质使得很多问题可以用递归优雅地解决。

面试中树的考点主要集中在：二叉树的遍历（前中后序、层次）、二叉搜索树的性质和操作、平衡树的原理、以及各种递归问题（深度、路径、翻转等）。理解树的关键是掌握递归思维和不同遍历方式的应用场景。

## 1. 树的基本概念

### 1.1 术语

| 术语 | 定义 |
|------|------|
| **根节点** | 没有父节点的节点 |
| **叶子节点** | 没有子节点的节点 |
| **节点的度** | 子节点的个数 |
| **树的度** | 所有节点度的最大值 |
| **节点的深度** | 从根到该节点的边数 |
| **节点的高度** | 从该节点到叶子节点的最长路径 |
| **树的高度** | 根节点的高度 |

### 1.2 二叉树

**定义**：每个节点最多有两个子节点（左子节点和右子节点）

**节点结构**：
```python
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right
```

### 1.3 二叉树的类型

读到“1.3 二叉树的类型”时，先别急着记结论。更有效的方式是先弄清它在“算法与数据结构 - 5.树”这一张卡片里到底解决什么问题，这样后面的分点和对比才不至于显得太散。

**满二叉树**：
- 每层节点都达到最大值
- 深度为 k 的满二叉树有 2^k - 1 个节点

**完全二叉树**：
- 除最后一层外，其他层都是满的
- 最后一层节点从左到右连续
- 适合用数组存储（堆就是完全二叉树）

**平衡二叉树**：
- 任意节点的左右子树高度差不超过 1
- 保证 O(log N) 的查找效率

## 2. 二叉树的遍历

### 2.1 深度优先遍历（DFS）

#### 前序遍历（根-左-右）

**递归实现**：
```python
def preorder(root):
    if not root:
        return []
    return [root.val] + preorder(root.left) + preorder(root.right)
```

**迭代实现**（用栈）：
```python
def preorder_iterative(root):
    if not root:
        return []
    
    result = []
    stack = [root]
    
    while stack:
        node = stack.pop()
        result.append(node.val)
        # 先压右子树，再压左子树（栈是后进先出）
        if node.right:
            stack.append(node.right)
        if node.left:
            stack.append(node.left)
    
    return result
```

#### 中序遍历（左-根-右）

**递归实现**：
```python
def inorder(root):
    if not root:
        return []
    return inorder(root.left) + [root.val] + inorder(root.right)
```

**迭代实现**：
```python
def inorder_iterative(root):
    result = []
    stack = []
    curr = root
    
    while curr or stack:
        # 一直往左走，把左子树全部入栈
        while curr:
            stack.append(curr)
            curr = curr.left
        
        # 弹出栈顶（最左节点）
        curr = stack.pop()
        result.append(curr.val)
        
        # 转向右子树
        curr = curr.right
    
    return result
```

**重要性质**：
- **二叉搜索树的中序遍历是有序的**
- 这是判断 BST 的常用方法

#### 后序遍历（左-右-根）

**递归实现**：
```python
def postorder(root):
    if not root:
        return []
    return postorder(root.left) + postorder(root.right) + [root.val]
```

**迭代实现**（前序遍历的变形）：
```python
def postorder_iterative(root):
    if not root:
        return []
    
    result = []
    stack = [root]
    
    while stack:
        node = stack.pop()
        result.append(node.val)
        # 先压左子树，再压右子树
        if node.left:
            stack.append(node.left)
        if node.right:
            stack.append(node.right)
    
    # 反转结果（根-右-左 → 左-右-根）
    return result[::-1]
```

### 2.2 广度优先遍历（BFS）

#### 层次遍历

**用队列实现**：
```python
from collections import deque

def level_order(root):
    if not root:
        return []
    
    result = []
    queue = deque([root])
    
    while queue:
        level_size = len(queue)
        level = []
        
        for _ in range(level_size):
            node = queue.popleft()
            level.append(node.val)
            
            if node.left:
                queue.append(node.left)
            if node.right:
                queue.append(node.right)
        
        result.append(level)
    
    return result
```

**应用场景**：
- 按层打印二叉树
- 求二叉树的最小深度
- 求二叉树的最大宽度

### 2.3 遍历方式对比

| 遍历方式 | 顺序 | 应用场景 |
|---------|------|---------|
| **前序** | 根-左-右 | 复制树、序列化 |
| **中序** | 左-根-右 | BST 排序、验证 BST |
| **后序** | 左-右-根 | 删除树、计算树的属性 |
| **层次** | 逐层 | 最短路径、按层处理 |

## 3. 二叉搜索树（BST）

二叉搜索树是一种特殊的二叉树，它的核心性质是：左子树所有节点的值小于根节点，右子树所有节点的值大于根节点。这个性质递归地适用于所有子树。

BST 最重要的特点是：中序遍历可以得到有序序列。这既是验证 BST 的常用方法，也是 BST 能高效支持范围查询的原因。

在平衡的情况下，BST 的查找、插入、删除操作时间复杂度都是 O(log N)。但如果树退化成链表（比如按顺序插入），时间复杂度会退化为 O(N)。这就是为什么实际应用中需要平衡树（AVL、红黑树）。

### 3.1 查找

```python
def search_bst(root, val):
    if not root:
        return None
    
    if val == root.val:
        return root
    elif val < root.val:
        return search_bst(root.left, val)
    else:
        return search_bst(root.right, val)
```

### 3.2 插入

```python
def insert_bst(root, val):
    if not root:
        return TreeNode(val)
    
    if val < root.val:
        root.left = insert_bst(root.left, val)
    elif val > root.val:
        root.right = insert_bst(root.right, val)
    
    return root
```

### 3.3 删除

删除节点是 BST 中最复杂的操作，需要分三种情况处理：

1. **叶子节点**：直接删除
2. **只有一个子节点**：用子节点替换
3. **有两个子节点**：用右子树的最小节点（或左子树的最大节点）替换

```python
def delete_bst(root, val):
    if not root:
        return None
    
    if val < root.val:
        root.left = delete_bst(root.left, val)
    elif val > root.val:
        root.right = delete_bst(root.right, val)
    else:
        # 找到要删除的节点
        if not root.left:
            return root.right
        if not root.right:
            return root.left
        
        # 有两个子节点：找右子树的最小节点
        min_node = find_min(root.right)
        root.val = min_node.val
        root.right = delete_bst(root.right, min_node.val)
    
    return root

def find_min(root):
    while root.left:
        root = root.left
    return root
```

## 4. 平衡二叉树

BST 的性能依赖于树的高度。如果按顺序插入数据，BST 会退化成链表，查找效率从 O(log N) 降到 O(N)。平衡树通过旋转操作保持树的高度在 O(log N)，从而保证操作效率。

### 4.1 AVL 树

AVL 树是最早的自平衡二叉搜索树，它要求任意节点的左右子树高度差不超过 1。这是一个很严格的平衡条件，保证了树的高度始终是 O(log N)。

当插入或删除节点导致不平衡时，AVL 树通过旋转来恢复平衡。根据不平衡的类型，有四种旋转方式：LL（右旋）、RR（左旋）、LR（先左旋后右旋）、RL（先右旋后左旋）。

**右旋示例**：
```python
def right_rotate(y):
    x = y.left
    T2 = x.right
    
    # 旋转
    x.right = y
    y.left = T2
    
    # 更新高度
    y.height = 1 + max(get_height(y.left), get_height(y.right))
    x.height = 1 + max(get_height(x.left), get_height(x.right))
    
    return x
```

AVL 树的优点是严格平衡，查找效率最高；缺点是插入删除时需要频繁旋转，维护成本较高。

### 4.2 红黑树

这里真正要说明的是“4.2 红黑树”为什么值得单独拿出来讲。把它放到“算法与数据结构 - 5.树”这张卡片的上下文里看，后面的要点和区别会更容易读顺。

红黑树是一种近似平衡的二叉搜索树，它通过给节点着色（红或黑）并维护一些性质来保持平衡：
- 每个节点是红色或黑色
- 根节点是黑色
- 叶子节点（NIL）是黑色
- 红色节点的子节点必须是黑色
- 从根到叶子的所有路径包含相同数量的黑色节点

这些性质保证了红黑树的最长路径不超过最短路径的 2 倍，从而保证了 O(log N) 的性能。

相比 AVL 树，红黑树的平衡条件更宽松，插入删除时旋转次数更少，维护成本更低。虽然查找效率略低于 AVL，但在插入删除密集的场景下整体性能更好。这就是为什么 Java 的 TreeMap、C++ 的 map 都选择红黑树而不是 AVL 树。

**AVL vs 红黑树**：

| 特性 | AVL 树 | 红黑树 |
|------|--------|--------|
| **平衡性** | 严格平衡 | 近似平衡 |
| **查找效率** | 更快 | 稍慢 |
| **插入删除** | 旋转多，慢 | 旋转少，快 |
| **应用场景** | 查找密集 | 插入删除密集 |

## 5. 树的常见问题

### 5.1 树的深度

**最大深度**：
```python
def max_depth(root):
    if not root:
        return 0
    return 1 + max(max_depth(root.left), max_depth(root.right))
```

**最小深度**：
```python
def min_depth(root):
    if not root:
        return 0
    if not root.left:
        return 1 + min_depth(root.right)
    if not root.right:
        return 1 + min_depth(root.left)
    return 1 + min(min_depth(root.left), min_depth(root.right))
```

### 5.2 判断平衡二叉树

```python
def is_balanced(root):
    def height(node):
        if not node:
            return 0
        
        left_height = height(node.left)
        if left_height == -1:
            return -1
        
        right_height = height(node.right)
        if right_height == -1:
            return -1
        
        if abs(left_height - right_height) > 1:
            return -1
        
        return 1 + max(left_height, right_height)
    
    return height(root) != -1
```

### 5.3 翻转二叉树

```python
def invert_tree(root):
    if not root:
        return None
    
    # 交换左右子树
    root.left, root.right = root.right, root.left
    
    # 递归翻转左右子树
    invert_tree(root.left)
    invert_tree(root.right)
    
    return root
```

### 5.4 对称二叉树

```python
def is_symmetric(root):
    def is_mirror(left, right):
        if not left and not right:
            return True
        if not left or not right:
            return False
        
        return (left.val == right.val and
                is_mirror(left.left, right.right) and
                is_mirror(left.right, right.left))
    
    return is_mirror(root, root)
```

### 5.5 路径总和

```python
def has_path_sum(root, target_sum):
    if not root:
        return False
    
    # 叶子节点
    if not root.left and not root.right:
        return root.val == target_sum
    
    # 递归检查左右子树
    return (has_path_sum(root.left, target_sum - root.val) or
            has_path_sum(root.right, target_sum - root.val))
```

### 5.6 最近公共祖先（LCA）

**二叉树**：
```python
def lowest_common_ancestor(root, p, q):
    if not root or root == p or root == q:
        return root
    
    left = lowest_common_ancestor(root.left, p, q)
    right = lowest_common_ancestor(root.right, p, q)
    
    if left and right:
        return root
    return left if left else right
```

**二叉搜索树**（利用 BST 性质）：
```python
def lowest_common_ancestor_bst(root, p, q):
    if p.val < root.val and q.val < root.val:
        return lowest_common_ancestor_bst(root.left, p, q)
    if p.val > root.val and q.val > root.val:
        return lowest_common_ancestor_bst(root.right, p, q)
    return root
```

## 6. 高频判断

### 判断 1：什么时候用树？

这里真正要说明的是“判断 1：什么时候用树？”为什么值得单独拿出来讲。把它放到“算法与数据结构 - 5.树”这张卡片的上下文里看，后面的要点和区别会更容易读顺。

**适合场景**：
- 层次结构数据（文件系统、组织架构）
- 需要快速查找、插入、删除（BST）
- 需要范围查询（BST）
- 需要维护有序数据（BST）

**不适合场景**：
- 需要频繁随机访问（用数组）
- 数据量小且无序（用数组或链表）

### 判断 2：遍历方式如何选择？

| 场景 | 遍历方式 |
|------|---------|
| 复制树 | 前序 |
| 验证 BST | 中序 |
| 删除树 | 后序 |
| 计算树的属性（高度、节点数） | 后序 |
| 最短路径、按层处理 | 层次 |

### 判断 3：BST vs 哈希表？

| 对比项 | BST | 哈希表 |
|--------|-----|--------|
| **查找** | O(log N) | O(1) |
| **插入** | O(log N) | O(1) |
| **删除** | O(log N) | O(1) |
| **有序性** | 支持 | 不支持 |
| **范围查询** | 支持 | 不支持 |
| **空间** | 较小 | 较大 |

**选择建议**：
- 需要有序或范围查询：BST
- 只需要快速查找：哈希表

### 判断 4：AVL vs 红黑树？

| 场景 | 选择 |
|------|------|
| 查找密集 | AVL |
| 插入删除密集 | 红黑树 |
| 需要严格平衡 | AVL |
| 工程实践 | 红黑树（Java TreeMap、C++ map） |

### 判断 5：递归 vs 迭代？

这里真正要说明的是“判断 5：递归 vs 迭代？”为什么值得单独拿出来讲。把它放到“算法与数据结构 - 5.树”这张卡片的上下文里看，后面的要点和区别会更容易读顺。

**递归**：
- 优点：代码简洁，易理解
- 缺点：栈空间 O(h)，可能栈溢出

**迭代**：
- 优点：空间可控，不会栈溢出
- 缺点：代码复杂，需要手动维护栈

**选择建议**：
- 树的高度不大：递归
- 树可能很深：迭代
- 面试时：先写递归，再优化为迭代

### 判断 6：如何判断是否是 BST？

**方法 1**：中序遍历，检查是否有序
```python
def is_valid_bst(root):
    def inorder(node):
        if not node:
            return []
        return inorder(node.left) + [node.val] + inorder(node.right)
    
    vals = inorder(root)
    return vals == sorted(vals) and len(vals) == len(set(vals))
```

**方法 2**：递归，传递上下界
```python
def is_valid_bst(root):
    def validate(node, low=float('-inf'), high=float('inf')):
        if not node:
            return True
        
        if node.val <= low or node.val >= high:
            return False
        
        return (validate(node.left, low, node.val) and
                validate(node.right, node.val, high))
    
    return validate(root)
```

## 8. 面试技巧

### 口头回答模板

**问：什么是二叉搜索树？**

答：二叉搜索树是一种特殊的二叉树，它的每个节点都满足：左子树所有节点的值小于根节点，右子树所有节点的值大于根节点。这个性质保证了中序遍历 BST 可以得到有序序列，也使得查找、插入、删除操作的平均时间复杂度为 O(log N)。但如果树不平衡，最坏情况会退化为 O(N)，所以实际应用中会使用 AVL 树或红黑树来保持平衡。

**问：如何遍历二叉树？**

答：二叉树有四种常见遍历方式：前序（根-左-右）、中序（左-根-右）、后序（左-右-根）和层次遍历。前三种是深度优先遍历，可以用递归或栈实现；层次遍历是广度优先遍历，用队列实现。不同遍历方式适用于不同场景，比如中序遍历用于验证 BST，后序遍历用于删除树，层次遍历用于按层处理。

### 常见追问

这里真正要说明的是“常见追问”为什么值得单独拿出来讲。把它放到“算法与数据结构 - 5.树”这张卡片的上下文里看，后面的要点和区别会更容易读顺。

**Q：为什么红黑树比 AVL 树更常用？**
- 红黑树的平衡条件更宽松，插入删除时旋转次数少，维护成本低
- 虽然查找效率略低于 AVL，但在插入删除密集的场景下整体性能更好
- 工程实践中（如 Java TreeMap）更倾向于使用红黑树

**Q：如何避免递归栈溢出？**
- 改用迭代实现，手动维护栈
- 使用尾递归优化（某些语言支持）
- 限制递归深度，超过阈值时报错

**Q：树的空间复杂度是多少？**
- 存储空间：O(N)
- 递归调用栈：O(h)，h 是树的高度
- 平衡树：O(log N)
- 非平衡树：O(N)
