
# 算法与数据结构 - 递归与回溯

递归是函数调用自己的编程技巧，核心思想是将大问题分解为相同结构的小问题。回溯是在递归基础上的一种搜索策略，通过尝试所有可能的解，遇到不满足条件时回退，本质是"深度优先搜索 + 剪枝"。

递归的优点是代码简洁，缺点是可能栈溢出或重复计算。回溯适用于排列、组合、子集等需要遍历所有可能解的问题，虽然时间复杂度通常很高（指数级），但通过剪枝可以显著提升性能。

面试中递归和回溯的考点主要是：递归三要素、如何避免栈溢出、排列/组合/子集问题的区别、以及如何剪枝优化。

## 1. 递归基础

写递归函数需要明确三个要素：终止条件、递归公式、返回值。终止条件决定什么时候停止递归，递归公式定义如何将大问题分解为小问题，返回值决定如何组合子问题的结果。

**递归三要素**：

1. **递归终止条件（Base Case）**：什么时候停止递归，防止无限递归
2. **递归公式（Recurrence Relation）**：如何将大问题分解为小问题，当前层和下一层的关系
3. **返回值**：每层递归返回什么，如何组合子问题的结果

**递归模板**：

```python
def recursion(level, param1, param2, ...):
    # 1. 递归终止条件
    if level > MAX_LEVEL:
        # 处理结果
        return
    
    # 2. 处理当前层逻辑
    process(level, data)
    
    # 3. 下探到下一层
    recursion(level + 1, new_param1, new_param2, ...)
    
    # 4. 清理当前层（如果需要）
    cleanup(level)
```

递归最常见的两个问题是栈溢出和重复计算。栈溢出是因为递归深度过大，超过栈空间限制（Python 默认 1000 层）；重复计算是因为子问题被多次计算。解决方法分别是改用迭代或增加栈深度，以及使用记忆化搜索或动态规划。

### 1.1 递归的问题

**示例：斐波那契数列**
```python
# 朴素递归：O(2^n)
def fib(n):
    if n <= 1:
        return n
    return fib(n - 1) + fib(n - 2)

# 记忆化搜索：O(n)
def fib_memo(n, memo={}):
    if n <= 1:
        return n
    if n in memo:
        return memo[n]
    memo[n] = fib_memo(n - 1, memo) + fib_memo(n - 2, memo)
    return memo[n]
```

## 2. 回溯算法

回溯算法的核心思想是"试错"：尝试所有可能的解，遇到不满足条件的解时回退（撤销选择），继续尝试其他路径。回溯适用于排列、组合、子集、棋盘问题（N皇后、数独）、路径问题等需要遍历所有可能解的场景。

回溯和 DFS 很相似，都是深度优先搜索，都用递归实现。区别在于：DFS 是遍历所有节点，回溯是寻找所有解，需要"做选择"和"撤销选择"这两个步骤。

### 2.1 回溯模板

```python
def backtrack(path, choices):
    # 1. 递归终止条件
    if 满足结束条件:
        result.append(path[:])  # 注意：要复制
        return
    
    # 2. 遍历所有选择
    for choice in choices:
        # 3. 做选择
        path.append(choice)
        
        # 4. 递归
        backtrack(path, new_choices)
        
        # 5. 撤销选择（回溯）
        path.pop()
```

回溯的关键是"做选择"和"撤销选择"这两步。做选择时将当前选项加入路径，递归处理；递归返回后撤销选择，恢复状态，继续尝试其他选项。这个过程就像走迷宫，走到死路就退回来换条路试试。

## 3. 排列问题

### 3.1 全排列

全排列问题是：给定 n 个不重复的数，返回所有可能的排列。排列考虑顺序，[1,2] 和 [2,1] 是不同的排列。

解题思路：每个位置都可以选择剩余的任意数字，选择后递归处理剩余位置，回溯时撤销选择。为了标记哪些数字已经使用，可以用 visited 数组。

**代码**：
```python
def permute(nums):
    result = []
    
    def backtrack(path, remaining):
        # 终止条件
        if not remaining:
            result.append(path[:])
            return
        
        # 遍历所有选择
        for i in range(len(remaining)):
            # 做选择
            path.append(remaining[i])
            # 递归
            backtrack(path, remaining[:i] + remaining[i+1:])
            # 撤销选择
            path.pop()
    
    backtrack([], nums)
    return result
```

**优化：使用 visited 数组**
```python
def permute_optimized(nums):
    result = []
    visited = [False] * len(nums)
    
    def backtrack(path):
        if len(path) == len(nums):
            result.append(path[:])
            return
        
        for i in range(len(nums)):
            if visited[i]:
                continue
            
            visited[i] = True
            path.append(nums[i])
            backtrack(path)
            path.pop()
            visited[i] = False
    
    backtrack([])
    return result
```

### 3.2 全排列 II（含重复元素）

当数组中包含重复元素时，需要去重。关键技巧是先排序，然后在回溯过程中跳过重复元素。判断条件是：`if i > 0 and nums[i] == nums[i-1] and not visited[i-1]`，意思是如果当前元素和前一个元素相同，且前一个元素还没被使用，就跳过当前元素。

**代码**：
```python
def permute_unique(nums):
    result = []
    nums.sort()  # 排序
    visited = [False] * len(nums)
    
    def backtrack(path):
        if len(path) == len(nums):
            result.append(path[:])
            return
        
        for i in range(len(nums)):
            if visited[i]:
                continue
            
            # 剪枝：跳过重复元素
            if i > 0 and nums[i] == nums[i-1] and not visited[i-1]:
                continue
            
            visited[i] = True
            path.append(nums[i])
            backtrack(path)
            path.pop()
            visited[i] = False
    
    backtrack([])
    return result
```

## 4. 组合问题

### 4.1 组合

组合问题是：从 n 个数中选 k 个数的所有组合。组合不考虑顺序，[1,2] 和 [2,1] 是同一个组合。

为了避免重复，每次只选择当前位置之后的数字。这通过 start 参数实现：递归时传入 `i+1` 而不是 `start+1`，保证每个数字只被考虑一次。

**代码**：
```python
def combine(n, k):
    result = []
    
    def backtrack(start, path):
        # 终止条件
        if len(path) == k:
            result.append(path[:])
            return
        
        # 遍历选择（从 start 开始，避免重复）
        for i in range(start, n + 1):
            path.append(i)
            backtrack(i + 1, path)  # 注意：i+1，不是 start+1
            path.pop()
    
    backtrack(1, [])
    return result
```

**剪枝优化**：
```python
def combine_pruned(n, k):
    result = []
    
    def backtrack(start, path):
        if len(path) == k:
            result.append(path[:])
            return
        
        # 剪枝：剩余数字不够了
        need = k - len(path)  # 还需要选几个
        remain = n - start + 1  # 还剩几个可选
        if remain < need:
            return
        
        for i in range(start, n + 1):
            path.append(i)
            backtrack(i + 1, path)
            path.pop()
    
    backtrack(1, [])
    return result
```

### 4.2 组合总和

**问题**：给定数组和目标值，找出所有和为目标值的组合（数字可重复使用）

**代码**：
```python
def combination_sum(candidates, target):
    result = []
    
    def backtrack(start, path, remain):
        if remain == 0:
            result.append(path[:])
            return
        if remain < 0:
            return
        
        for i in range(start, len(candidates)):
            path.append(candidates[i])
            # 注意：i 不是 i+1，因为可以重复使用
            backtrack(i, path, remain - candidates[i])
            path.pop()
    
    backtrack(0, [], target)
    return result
```

## 5. 子集问题

### 5.1 子集

**问题**：返回数组的所有子集（幂集）

**分析**：
- 每个元素都有"选"和"不选"两种状态
- 子集数量：2^n

**方法 1：回溯**
```python
def subsets(nums):
    result = []
    
    def backtrack(start, path):
        # 每个状态都是一个子集
        result.append(path[:])
        
        for i in range(start, len(nums)):
            path.append(nums[i])
            backtrack(i + 1, path)
            path.pop()
    
    backtrack(0, [])
    return result
```

**方法 2：位运算**
```python
def subsets_bit(nums):
    n = len(nums)
    result = []
    
    # 遍历 0 到 2^n - 1
    for i in range(1 << n):
        subset = []
        for j in range(n):
            # 检查第 j 位是否为 1
            if i & (1 << j):
                subset.append(nums[j])
        result.append(subset)
    
    return result
```

### 5.2 子集 II（含重复元素）

**问题**：数组可能包含重复元素，返回所有不重复的子集

**关键**：
- 先排序
- 跳过重复元素

**代码**：
```python
def subsets_with_dup(nums):
    result = []
    nums.sort()
    
    def backtrack(start, path):
        result.append(path[:])
        
        for i in range(start, len(nums)):
            # 剪枝：跳过重复元素
            if i > start and nums[i] == nums[i-1]:
                continue
            
            path.append(nums[i])
            backtrack(i + 1, path)
            path.pop()
    
    backtrack(0, [])
    return result
```

## 6. 经典问题

### 6.1 N 皇后

**问题**：在 N×N 的棋盘上放置 N 个皇后，使得它们不能互相攻击

**分析**：
- 每行只能放一个皇后
- 每列只能放一个皇后
- 每条对角线只能放一个皇后

**代码**：
```python
def solve_n_queens(n):
    result = []
    board = [['.'] * n for _ in range(n)]
    cols = set()
    diag1 = set()  # 主对角线：row - col
    diag2 = set()  # 副对角线：row + col
    
    def backtrack(row):
        if row == n:
            result.append([''.join(row) for row in board])
            return
        
        for col in range(n):
            # 检查是否可以放置
            if col in cols or (row - col) in diag1 or (row + col) in diag2:
                continue
            
            # 放置皇后
            board[row][col] = 'Q'
            cols.add(col)
            diag1.add(row - col)
            diag2.add(row + col)
            
            # 递归
            backtrack(row + 1)
            
            # 撤销
            board[row][col] = '.'
            cols.remove(col)
            diag1.remove(row - col)
            diag2.remove(row + col)
    
    backtrack(0)
    return result
```

### 6.2 括号生成

**问题**：生成所有有效的括号组合

**分析**：
- 左括号数量 <= n
- 右括号数量 <= 左括号数量

**代码**：
```python
def generate_parenthesis(n):
    result = []
    
    def backtrack(path, left, right):
        # 终止条件
        if len(path) == 2 * n:
            result.append(path)
            return
        
        # 添加左括号
        if left < n:
            backtrack(path + '(', left + 1, right)
        
        # 添加右括号
        if right < left:
            backtrack(path + ')', left, right + 1)
    
    backtrack('', 0, 0)
    return result
```

### 6.3 单词搜索

**问题**：在二维网格中搜索单词

**代码**：
```python
def exist(board, word):
    m, n = len(board), len(board[0])
    visited = [[False] * n for _ in range(m)]
    
    def backtrack(i, j, k):
        # 找到单词
        if k == len(word):
            return True
        
        # 越界或不匹配
        if i < 0 or i >= m or j < 0 or j >= n:
            return False
        if visited[i][j] or board[i][j] != word[k]:
            return False
        
        # 标记访问
        visited[i][j] = True
        
        # 四个方向搜索
        found = (backtrack(i+1, j, k+1) or
                 backtrack(i-1, j, k+1) or
                 backtrack(i, j+1, k+1) or
                 backtrack(i, j-1, k+1))
        
        # 撤销标记
        visited[i][j] = False
        
        return found
    
    # 从每个位置开始搜索
    for i in range(m):
        for j in range(n):
            if backtrack(i, j, 0):
                return True
    
    return False
```

## 7. 高频判断

### 判断 1：排列、组合、子集的区别？

| 问题 | 特点 | 是否考虑顺序 | 模板 |
|------|------|-------------|------|
| **排列** | 选 k 个排成一排 | 是 | 用 visited 数组 |
| **组合** | 选 k 个 | 否 | 用 start 参数 |
| **子集** | 选任意个 | 否 | 每个状态都记录 |

**记忆技巧**：
- 排列：全选，考虑顺序，用 visited
- 组合：选 k 个，不考虑顺序，用 start
- 子集：选任意个，不考虑顺序，每步都记录

### 判断 2：如何避免重复？

读到“判断 2：如何避免重复？”时，先别急着记结论。更有效的方式是先弄清它在“算法与数据结构 - 10.递归与回溯”这一张卡片里到底解决什么问题，这样后面的分点和对比才不至于显得太散。

**排列问题**：
- 用 visited 数组标记已使用的元素
- 含重复元素时：排序 + 剪枝

**组合/子集问题**：
- 用 start 参数，只选择当前位置之后的元素
- 含重复元素时：排序 + 跳过相同元素

### 判断 3：如何剪枝优化？

**常见剪枝策略**：
1. **提前终止**：不满足条件时直接返回
2. **排序**：先排序，方便剪枝
3. **跳过重复**：`if i > start and nums[i] == nums[i-1]: continue`
4. **剩余不够**：剩余元素不够时直接返回

### 判断 4：递归 vs 迭代？

| 对比项 | 递归 | 迭代 |
|--------|------|------|
| **代码** | 简洁 | 复杂 |
| **空间** | O(h) 栈空间 | O(1) 或 O(h) |
| **性能** | 可能栈溢出 | 不会栈溢出 |
| **适用** | 树、回溯 | 循环、尾递归 |

**选择建议**：
- 树、回溯：递归
- 简单循环：迭代
- 深度很大：迭代

### 判断 5：如何避免栈溢出？

**方法**：
1. **改用迭代**：手动维护栈
2. **尾递归优化**：某些语言支持
3. **增加栈深度**：`sys.setrecursionlimit()`
4. **记忆化搜索**：避免重复计算

### 判断 6：回溯的时间复杂度？

| 问题 | 时间复杂度 | 说明 |
|------|-----------|------|
| **排列** | O(N!) | N 个位置，每个位置的选择递减 |
| **组合** | O(C(N,K)) | 从 N 个中选 K 个 |
| **子集** | O(2^N) | 每个元素选或不选 |
| **N 皇后** | O(N!) | 每行选一个位置 |

### 判断 7：常见面试问题

这一节看起来往往容易变成一串结论，其实先抓住“判断 7：常见面试问题”在算法与数据结构里的作用会更稳。后面的内容可以理解成围绕这个问题展开的几种常见判断和做法。

**Q1：递归和循环的区别？**
- 递归：函数调用自己，代码简洁，但有栈空间开销
- 循环：重复执行代码块，空间复杂度低，但代码可能复杂

**Q2：如何将递归改为迭代？**
- 手动维护栈
- 用队列实现 BFS
- 用动态规划

**Q3：回溯和动态规划的区别？**
- 回溯：暴力搜索所有解，时间复杂度高
- 动态规划：保存子问题的解，避免重复计算

**Q4：如何判断是否需要回溯？**
- 需要找所有解
- 需要遍历所有可能
- 有明确的选择和撤销过程

## 9. 面试技巧

### 口头回答模板

**问：什么是回溯算法？**

答：回溯算法是一种通过尝试所有可能的解来寻找问题答案的算法。它的核心思想是：在搜索过程中，如果发现当前路径不满足条件，就回退到上一步，尝试其他路径。回溯算法本质上是深度优先搜索加上剪枝优化。它适用于排列、组合、子集等需要遍历所有可能解的问题。虽然时间复杂度通常很高（指数级），但通过剪枝可以显著提升性能。

**问：排列、组合、子集有什么区别？**

答：这三个问题的核心区别在于是否考虑顺序和选择数量：
- 排列：选 N 个元素排成一排，考虑顺序，[1,2] 和 [2,1] 是不同的排列
- 组合：选 K 个元素，不考虑顺序，[1,2] 和 [2,1] 是同一个组合
- 子集：选任意个元素，不考虑顺序，包括空集

在实现上，排列用 visited 数组标记已使用的元素，组合和子集用 start 参数避免重复选择。

### 常见追问

这里真正要说明的是“常见追问”为什么值得单独拿出来讲。把它放到“算法与数据结构 - 10.递归与回溯”这张卡片的上下文里看，后面的要点和区别会更容易读顺。

**Q：如何优化回溯算法？**
- 剪枝：提前终止不满足条件的分支
- 排序：方便跳过重复元素
- 记忆化：保存已计算的结果

**Q：回溯和 DFS 的区别？**
- DFS 是遍历所有节点
- 回溯是寻找所有解，需要撤销选择
- 回溯可以看作是 DFS 的一种应用

**Q：如何避免回溯中的重复？**
- 排列：用 visited 数组
- 组合/子集：用 start 参数
- 含重复元素：排序 + 跳过相同元素
