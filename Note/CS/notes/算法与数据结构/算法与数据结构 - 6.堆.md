
# 算法与数据结构 - 堆

堆是一种特殊的完全二叉树，分为大根堆和小根堆。大根堆中每个节点的值都大于等于其子节点，小根堆则相反。堆最常见的应用是实现优先队列和堆排序。

## 堆的基本性质

堆是完全二叉树，可以用数组高效存储。对于数组中位置 k 的节点：
- 父节点位置：k/2
- 左子节点位置：2k
- 右子节点位置：2k+1

这里约定数组索引从 1 开始，位置 0 不使用，这样计算父子关系更清晰。

## 堆的核心操作

### 上浮（Swim）

当插入新元素或某个节点的值变大时，可能违反堆的性质。这时需要将该节点与父节点比较，如果比父节点大（大根堆），就交换它们的位置，然后继续向上比较，直到满足堆的性质。

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/99d5e84e-fc2a-49a3-8259-8de274617756.gif" width="270px"> </div><br>

```python
def swim(heap, k):
    while k > 1 and heap[k] > heap[k // 2]:
        heap[k], heap[k // 2] = heap[k // 2], heap[k]
        k = k // 2
```

### 下沉（Sink）

当删除堆顶元素或某个节点的值变小时，需要将该节点与子节点比较。选择两个子节点中较大的那个，如果该节点比它小，就交换位置，然后继续向下比较。

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/4bf5e3fb-a285-4138-b3b6-780956eb1df1.gif" width="270px"> </div><br>

```python
def sink(heap, k, n):
    while 2 * k <= n:
        j = 2 * k
        # 选择较大的子节点
        if j < n and heap[j] < heap[j + 1]:
            j += 1
        # 如果已经满足堆性质，停止
        if heap[k] >= heap[j]:
            break
        heap[k], heap[j] = heap[j], heap[k]
        k = j
```

### 插入元素

将新元素放到数组末尾，然后上浮到合适位置。时间复杂度 O(log N)。

```python
def insert(heap, val):
    heap.append(val)
    swim(heap, len(heap) - 1)
```

### 删除最大元素

将堆顶元素（最大值）与最后一个元素交换，删除最后一个元素，然后让新的堆顶元素下沉。时间复杂度 O(log N)。

```python
def delete_max(heap):
    max_val = heap[1]
    heap[1] = heap[-1]
    heap.pop()
    sink(heap, 1, len(heap) - 1)
    return max_val
```

## 堆排序

堆排序利用堆的性质进行排序，分为两个阶段：构建堆和排序。

### 构建堆

从最后一个非叶子节点开始，依次对每个节点执行下沉操作。这样从下往上、从右往左地调整，最终得到一个大根堆。

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/c2ca8dd2-8d00-4a3e-bece-db7849ac9cfd.gif" width="210px"> </div><br>

### 排序过程

反复将堆顶（最大值）与堆的最后一个元素交换，然后缩小堆的范围，对新的堆顶执行下沉操作。

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/d156bcda-ac8d-4324-95e0-0c8df41567c9.gif" width="250px"> </div><br>

```python
def heap_sort(arr):
    n = len(arr) - 1
    
    # 构建大根堆
    for i in range(n // 2, 0, -1):
        sink(arr, i, n)
    
    # 排序
    while n > 1:
        arr[1], arr[n] = arr[n], arr[1]
        n -= 1
        sink(arr, 1, n)
```

堆排序的时间复杂度是 O(N log N)，空间复杂度是 O(1)，是原地排序算法。但它不是稳定排序，且由于频繁跳跃访问数组元素，缓存命中率低，实际性能不如快速排序。

## Top K 问题

堆最经典的应用是解决 Top K 问题：从 N 个元素中找出最大（或最小）的 K 个元素。

**找最大的 K 个元素**：维护一个大小为 K 的小根堆。遍历所有元素，如果当前元素比堆顶大，就替换堆顶并调整堆。最后堆中就是最大的 K 个元素。

**找最小的 K 个元素**：维护一个大小为 K 的大根堆，逻辑类似。

```python
import heapq

def top_k_largest(nums, k):
    # Python 的 heapq 是小根堆
    heap = nums[:k]
    heapq.heapify(heap)
    
    for num in nums[k:]:
        if num > heap[0]:
            heapq.heapreplace(heap, num)
    
    return heap
```

时间复杂度 O(N log K)，空间复杂度 O(K)。这比先排序再取前 K 个（O(N log N)）更高效。

## 优先队列

优先队列是堆的直接应用。每次取出的元素是当前优先级最高（或最低）的元素。

**应用场景**：
- 任务调度：优先处理重要任务
- Dijkstra 最短路径算法
- 哈夫曼编码
- 事件驱动模拟

Python 的 `heapq` 模块提供了优先队列的实现：

```python
import heapq

# 创建空堆
heap = []

# 插入元素
heapq.heappush(heap, 3)
heapq.heappush(heap, 1)
heapq.heappush(heap, 4)

# 弹出最小元素
min_val = heapq.heappop(heap)  # 1

# 查看最小元素但不删除
min_val = heap[0]

# 将列表转换为堆
nums = [3, 1, 4, 1, 5]
heapq.heapify(nums)
```

## 什么时候用堆

这一节主要把“什么时候用堆”放回“算法与数据结构 - 6.堆”这条主线里看。先抓住它在整个算法与数据结构知识体系中的位置，再去看后面的分类和结论，会更容易把零散的判断串起来。

**适合的场景**：
- 需要频繁获取最大值或最小值
- Top K 问题
- 需要动态维护有序数据（插入和删除操作频繁）
- 优先队列

**不适合的场景**：
- 需要查找任意元素（用哈希表）
- 需要范围查询（用平衡树）
- 只需要一次性排序（用快速排序或归并排序）

## 堆 vs 其他数据结构

这一节看起来往往容易变成一串结论，其实先抓住“堆 vs 其他数据结构”在算法与数据结构里的作用会更稳。后面的内容可以理解成围绕这个问题展开的几种常见判断和做法。

**堆 vs 数组**：
- 堆：获取最值 O(1)，插入删除 O(log N)
- 有序数组：获取最值 O(1)，插入删除 O(N)
- 无序数组：获取最值 O(N)，插入删除 O(1)

**堆 vs 平衡树（如红黑树）**：
- 堆：只能获取最值，操作 O(log N)
- 平衡树：可以获取任意顺序的元素，操作 O(log N)
- 堆的常数因子更小，实现更简单

**选择建议**：
- 只需要最值：堆
- 需要有序遍历或范围查询：平衡树
- 需要频繁查找任意元素：哈希表

## 常见面试问题

**Q1：堆和二叉搜索树的区别？**

堆是完全二叉树，只保证父节点大于（或小于）子节点，兄弟节点之间没有大小关系。二叉搜索树保证左子树 < 根 < 右子树，中序遍历是有序的。堆用于快速获取最值，BST 用于有序数据的查找和遍历。

**Q2：为什么堆排序不常用？**

虽然堆排序时间复杂度是 O(N log N) 且是原地排序，但它的缓存命中率低。堆的操作需要频繁跳跃访问数组元素（父子节点的索引相差较大），而快速排序和归并排序更多是顺序访问，对缓存更友好。

**Q3：如何用堆实现中位数？**

维护两个堆：一个大根堆存储较小的一半元素，一个小根堆存储较大的一半元素。保持两个堆的大小相差不超过 1。中位数就是两个堆顶的平均值（偶数个元素）或某个堆顶（奇数个元素）。

**Q4：堆的时间复杂度？**

- 构建堆：O(N)（不是 O(N log N)，因为大部分节点的下沉距离很短）
- 插入：O(log N)
- 删除最值：O(log N)
- 查看最值：O(1)
- 堆排序：O(N log N)
