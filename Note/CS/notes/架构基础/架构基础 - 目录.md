# 架构基础 - 目录

架构这个词很容易越说越虚。图画得更大一点，组件名词再多一点，仿佛就有了架构感。可对后端来说，真正有用的架构基础并不神秘，它主要回答几件很实际的事：系统怎样分层，边界怎么划，模块怎样协作，复杂度为什么会慢慢长出来。

这一组笔记不写成“架构名词大全”，而是沿着后端最常遇到的几条主线往下走。先看分层、模块边界和依赖方向，理解为什么很多维护问题最后都能追到边界不清；再看同步、异步、串行、并行以及单体到拆分的变化，理解系统为什么会从直接路径慢慢长成复杂网络；最后回到演进与取舍，看所谓架构决策为什么很少是一次性设计，更多是在现实压力里逐步形成的。

## 架构基础在后端里的位置

架构基础不是替代软件工程，也不是替代分布式基础。软件工程更关心团队如何把代码持续做对，分布式基础更关心节点、复制、一致性和协调问题；架构基础更像中间那一层，它关心系统形状是怎么长出来的，模块关系怎样影响后面的扩展、维护和故障处理。

## 这一模块的阅读顺序

先读《架构基础 - 分层与边界》，把模块关系和依赖方向压清楚。再读《架构基础 - 同步、异步与拆分》，看系统为什么会从单体路径走向更复杂的协作方式。最后读《架构基础 - 演进与取舍》，把前面两篇重新放回真实项目的长期演进里。
