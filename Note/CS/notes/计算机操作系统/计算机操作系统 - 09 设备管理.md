## 设备管理概述

设备管理是操作系统管理计算机外部设备的功能模块，负责设备的分配、启动、完成和回收。

### 设备分类

**按数据传输单位分类**：

**块设备（Block Device）**：数据以块为单位传输（如 4KB）；可寻址，支持随机访问；传输速率高；比如硬盘、SSD、U盘

**字符设备（Character Device）**：数据以字符为单位传输；不可寻址，只能顺序访问；传输速率低；比如键盘、鼠标、打印机

**网络设备（Network Device）**：数据以数据包为单位传输，用于网络通信；比如网卡

### 设备管理的目标

- **提高设备利用率**：多个进程共享设备
- **方便用户使用**：提供统一的接口
- **提高系统吞吐量**：合理调度设备

## I/O控制方式

I/O 控制方式是指 CPU 如何控制 I/O 设备进行数据传输。

### 程序直接控制（Programmed I/O）

CPU 直接控制 I/O 设备，不断轮询设备状态。

**过程**：

1. CPU 向设备发送 I/O 命令
2. CPU 不断轮询设备状态（忙等待）
3. 设备准备好后，CPU 读取数据
4. 重复 2-3，直到传输完成

**特点**：简单，但是CPU 利用率极低（一直在等待）

**类比**：你在煮水，并一直等着水开。

### 中断驱动（Interrupt-Driven I/O）

CPU 发出 I/O 命令后，不再等待，去做其他任务。设备准备好后，通过中断通知 CPU。

**过程**：
1. CPU 向设备发送 I/O 命令
2. CPU 去做其他工作
3. 设备准备好后，发出中断信号
4. CPU 响应中断，读取数据
5. 重复 1-4，直到传输完成

**特点**：CPU 利用率提高，但每次传输都要中断

**类比**：你在煮水，设置了闹钟，然后去做其他事情，水开了闹钟响，你再去关火。

### DMA（Direct Memory Access，直接内存访问）

引入 DMA 控制器，负责在内存和设备之间直接传输数据，不需要 CPU 参与。

**过程**：

1. CPU 向 DMA 控制器发送命令（传输起始地址、传输长度）
2. CPU 去做其他事情
3. DMA 控制器直接在内存和设备之间传输数据
4. 传输完成后，DMA 控制器发出中断通知 CPU

**特点**：CPU 完全不参与数据传输，利用率最高；但是需要 DMA 控制器硬件支持

**类比**：你在煮水，请了个保姆，保姆负责看水壶，水开了保姆关火，然后通知你。

**DMA 比中断驱动快**

- 中断驱动：每传输一个单位都要中断一次，也就是说设备准备好一份数据，一般就触发一次中断，因此开销大
- DMA：传输整个数据块只中断一次，中断开销小

### 通道控制（Channel）

引入通道处理器，是一个专门负责 I/O 的处理器，有自己的指令集。

**过程**：
1. CPU 向通道发送 I/O 命令
2. 通道执行 I/O 程序，完成数据传输
3. 传输完成后，通道发出中断通知 CPU

**特点**：CPU 完全不参与 I/O，通道可以并行处理多个 I/O 请求；但是硬件成本高，主要用于大型机

## 缓冲技术

缓冲技术是在内存中开辟一块区域，用于暂存 I/O 数据，减少 CPU 和设备之间的速度差异。

**速度不匹配**：CPU 速度快（GHz），设备速度慢（KB/s）；如果 CPU 直接等待设备，利用率低

**数据粒度不匹配**：CPU、内存和外设的数据处理单位或传输方式可能不同，缓冲区可以作为中间过渡区，实现数据格式、传输粒度或到达节奏的协调。

**减少中断次数**：缓冲技术在某些 I/O 方式下可以减少中断次数。若没有足够缓冲，设备可能每产生一个较小数据单位就需要中断 CPU；而设置缓冲区后，可以等积累到一定数量再统一处理，从而减少中断开销。

### 单缓冲（Single Buffer）

在内存中开辟一个缓冲区。

**过程**：

1. 设备把数据写入缓冲区
2. CPU 从缓冲区读取数据
3. 重复 1-2

**问题**：设备写入时，CPU 不能读取；CPU 读取时，设备不能写入。

### 双缓冲（Double Buffer）

在内存中开辟两个缓冲区，交替使用。一个负责写、一个负责读，他们的职责可能转换。

**过程**：
1. 设备把数据写入缓冲区 1
2. CPU 从缓冲区 2 读取数据
3. 缓冲区 1 写满后，交换缓冲区 1 和缓冲区 2
4. 重复 1-3

**优点**：设备和 CPU 可以并行工作，提高效率

### 循环缓冲（Circular Buffer）

多个缓冲区组成一个环形队列。比如生产者-消费者模型

### 缓冲池（Buffer Pool）

系统维护一个缓冲区池，动态分配缓冲区。比如数据库系统

## 磁盘

### 磁盘结构

磁盘是最常见的块设备，理解磁盘结构有助于理解磁盘调度算法。

**磁盘的物理结构**：

- **盘面（Platter）**：一个磁盘有多个盘面
- **磁道（Track）**：盘面上的圆形带状区域，一个盘面可以有多个磁道
- **扇区（Track Sector）**：磁道上的一个弧段，一个磁道可以有多个扇区，它是最小的物理储存单位，目前主要有 512 bytes 与 4 K 两种大小
- **磁头（Head）**：与盘面非常接近，能够将盘面上的磁场转换为电信号（读），或者将电信号转换为盘面的磁场（写）
- **制动手臂（Actuator arm）**：用于在磁道之间移动磁头
- **主轴（Spindle）**：使整个盘面转动

**读写一个磁盘块的时间组成**：

- **旋转时间**：主轴转动盘面，使得磁头移动到适当的扇区上
- **寻道时间**：制动手臂移动，使得磁头移动到适当的磁道上
- **实际的数据传输时间**

其中，**寻道时间最长**，因此磁盘调度的主要目标是使磁盘的平均寻道时间最短。

**SSD 比机械硬盘快**

- 机械硬盘：需要寻道时间（移动磁头）和旋转时间（转动盘面），慢
- SSD：没有机械部件，直接通过电信号访问，快
- SSD 的随机访问速度是机械硬盘的 100 倍以上

### 磁盘调度算法

磁盘调度算法决定了磁盘请求的处理顺序，目标是减少平均寻道时间。

#### 先来先服务（FCFS, First Come First Served）

按照磁盘请求的顺序进行调度。

特点：公平、简单；但未对寻道做任何优化，平均寻道时间可能较长

**例子**：

- 当前磁头在 53 号磁道
- 请求队列：98, 183, 37, 122, 14, 124, 65, 67
- 访问顺序：98 → 183 → 37 → 122 → 14 → 124 → 65 → 67
- 总移动距离：(98-53) + (183-98) + (183-37) + (122-37) + (122-14) + (124-14) + (124-65) + (67-65) = 640

#### 最短寻道时间优先（SSTF, Shortest Seek Time First）

优先调度与当前磁头所在磁道距离最近的磁道。

**特点**：平均寻道时间比 FCFS 低，不够公平，可能出现饥饿现象（两端的磁道请求可能一直等待）

**例子**：
- 当前磁头在 53 号磁道
- 请求队列：98, 183, 37, 122, 14, 124, 65, 67
- 访问顺序：65 → 67 → 37 → 14 → 98 → 122 → 124 → 183
- 总移动距离：(65-53) + (67-65) + (67-37) + (37-14) + (98-14) + (122-98) + (124-122) + (183-124) = 236

#### 电梯算法（SCAN，扫描算法）

电梯总是保持一个方向运行，直到该方向没有请求为止，然后改变运行方向。

**过程**：
- 磁头沿一个方向移动，处理沿途的所有请求
- 到达最外侧或最内侧磁道后，改变方向
- 重复上述过程

**特点**：解决了 SSTF 的饥饿问题，所有请求都会被满足；但两端的磁道等待时间较长

**例子**：
- 当前磁头在 53 号磁道，向外移动
- 请求队列：98, 183, 37, 122, 14, 124, 65, 67
- 访问顺序：65 → 67 → 98 → 122 → 124 → 183 → 37 → 14
- 总移动距离：(65-53) + (67-65) + (98-67) + (122-98) + (124-122) + (183-124) + (183-37) + (37-14) = 331
