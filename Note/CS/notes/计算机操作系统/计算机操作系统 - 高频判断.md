## 进程与线程

进程和线程是操作系统中最容易混淆的概念。核心区别是：进程是资源分配的基本单位，线程是独立调度的基本单位。

**进程 vs 线程**

进程拥有独立的资源和地址空间，进程间通信需要 IPC（管道、消息队列等），创建和切换开销大，但进程崩溃不影响其他进程。线程共享进程资源和地址空间，可以直接读写共享内存，创建和切换开销小，但线程崩溃可能导致整个进程崩溃。

记忆技巧：进程重（资源多），线程轻（资源少）。

**多进程 vs 多线程**

选择多进程还是多线程，取决于具体场景。多进程的优势是稳定性好（一个进程崩溃不影响其他进程）、安全性好（进程间隔离）、可以利用多核 CPU。多线程的优势是创建和切换开销小、通信方便（共享内存）、资源占用少。

选择建议：CPU 密集型 + 多核可以用多进程或多线程，I/O 密集型用多线程更合适，需要高稳定性用多进程，需要频繁通信用多线程。

**协程 vs 线程**

协程和线程的核心区别在于调度者。线程由操作系统内核调度，切换开销较大（需要陷入内核），可以真正并行，适合 CPU 密集型。协程由用户程序调度，切换开销很小（用户态切换），只能并发不能并行，适合 I/O 密集型。

记忆技巧：线程内核调度可并行，协程用户调度只并发。

## 上下文切换与模式切换

上下文切换和模式切换是两个不同的概念，容易混淆。

**上下文切换 vs 模式切换**

上下文切换是从进程/线程 A 切换到进程/线程 B，需要切换进程，需要切换页表（进程切换），开销大，由时间片用完、阻塞等触发。模式切换是用户态和内核态之间的切换，不切换进程，不切换页表，开销小，由系统调用、中断、异常触发。

关键区别：上下文切换是切换执行的进程/线程，模式切换是切换 CPU 的特权级。

关系：上下文切换一定包含模式切换，模式切换不一定包含上下文切换。

**系统调用一定会触发上下文切换吗**

不一定。系统调用只是从用户态切换到内核态（模式切换），不一定切换进程。例如 getpid() 只有模式切换，没有上下文切换。但如果系统调用需要等待 I/O（如 read），可能切换到其他进程。

## 用户态与内核态

用户态和内核态的区别在于权限和可访问的资源。

**用户态 vs 内核态**

用户态权限受限，只能执行非特权指令，只能访问用户空间，运行应用程序，崩溃只影响当前进程。内核态权限最高，可以执行所有指令，可以访问所有空间，运行操作系统内核，崩溃可能导致系统崩溃。

**什么操作会触发用户态到内核态的切换**

会触发：系统调用（如 read、write、fork）、异常（如缺页异常、除零错误）、外部中断（如时钟中断、I/O 中断）。

不会触发：
- 普通函数调用
- 普通的算术运算
- 访问自己的内存空间

## 并发与并行

**并发 vs 并行**

| 对比项 | 并发（Concurrency） | 并行（Parallelism） |
|--------|-------------------|-------------------|
| 定义 | 多个任务交替执行 | 多个任务同时执行 |
| 时间维度 | 宏观上同时，微观上交替 | 真正的同时 |
| 硬件要求 | 单核即可 | 需要多核 |
| 目的 | 提高系统响应性 | 提高系统吞吐量 |

形象比喻：
- 并发：一个人同时做多件事（快速切换）
- 并行：多个人同时做多件事

示例：
```
单核 CPU：
- 只能并发，不能并行
- 通过快速切换实现"同时"执行多个任务

多核 CPU：
- 既可以并发，也可以并行
- 多个核心真正同时执行多个任务
```

## 同步与异步、阻塞与非阻塞

**同步 vs 异步**

| 对比项 | 同步 | 异步 |
|--------|------|------|
| 定义 | 主动等待或轮询结果 | 被动接收通知 |
| 关注点 | 消息通知机制 | 消息通知机制 |
| 获取结果 | 主动获取 | 被动接收 |
| 是否阻塞 | 可能阻塞 | 不阻塞 |

记忆技巧：同步是我问你答（主动），异步是你通知我（被动）。

**阻塞 vs 非阻塞**

| 对比项 | 阻塞 | 非阻塞 |
|--------|------|--------|
| 定义 | 调用者等待结果 | 调用者立即返回 |
| 关注点 | 调用者的状态 | 调用者的状态 |
| 等待方式 | 挂起等待 | 立即返回，可能需要轮询 |
| CPU 占用 | 不占用（挂起） | 可能占用（轮询） |

记忆技巧：阻塞是等（挂起），非阻塞是不等（立即返回）。

**阻塞/非阻塞 vs 同步/异步**

这是两个不同维度的概念：
- 阻塞/非阻塞：描述调用者的状态（等不等）
- 同步/异步：描述通知机制（谁主动）

| 组合 | 说明 | 示例 |
|------|------|------|
| 同步阻塞 | 调用者等待，主动获取结果 | 传统的 read() |
| 同步非阻塞 | 调用者轮询，主动获取结果 | 非阻塞 read() + 轮询 |
| 异步阻塞 | 调用者等待通知 | 很少见 |
| 异步非阻塞 | 调用者继续执行，被动接收通知 | Linux AIO |

## 互斥量、信号量、锁

**互斥量 vs 信号量 vs 锁**

| 对比项 | 互斥量（Mutex） | 信号量（Semaphore） | 锁（Lock） |
|--------|----------------|-------------------|-----------|
| 用途 | 互斥访问 | 同步 + 互斥 | 互斥访问 |
| 取值 | 0 或 1 | 非负整数 | 锁定/未锁定 |
| 所有权 | 有（谁加锁谁解锁） | 无 | 有 |
| 可重入 | 可以（递归锁） | 不可以 | 可以（递归锁） |

互斥量（Mutex）：
- 二元信号量（取值 0 或 1）
- 用于保护临界区
- 谁加锁谁解锁

信号量（Semaphore）：
- 可以取任意非负整数
- 可以用于同步（如生产者-消费者）
- 没有所有权概念

锁（Lock）：
- 广义的概念，包括互斥锁、读写锁、自旋锁等
- 互斥锁 ≈ 互斥量

**自旋锁 vs 互斥锁**

| 对比项 | 自旋锁 | 互斥锁 |
|--------|--------|--------|
| 等待方式 | 忙等待（循环检查） | 睡眠等待 |
| CPU 占用 | 占用 | 不占用 |
| 上下文切换 | 无 | 有 |
| 适用场景 | 锁持有时间短 | 锁持有时间长 |
| 适用环境 | 多核 CPU | 单核或多核 |

选择建议：
- 临界区很小：自旋锁
- 临界区较大：互斥锁
- 单核 CPU：互斥锁（自旋锁会浪费 CPU）

**读写锁**

特点：
- 允许多个读者同时读
- 只允许一个写者写
- 读写互斥

适用场景：读多写少，提高并发性能。

## 死锁

**死锁的四个必要条件**

必须同时满足：
1. 互斥：资源不能共享，只能被一个进程使用
2. 占有和等待：进程持有资源的同时等待其他资源
3. 不可抢占：资源不能被强制剥夺
4. 环路等待：存在进程资源的循环等待链

破坏任意一个条件，就可以预防死锁。

**死锁预防 vs 死锁避免 vs 死锁检测**

| 方法 | 时机 | 策略 | 代价 |
|------|------|------|------|
| 死锁预防 | 事前 | 破坏四个必要条件之一 | 资源利用率低 |
| 死锁避免 | 事中 | 动态检查是否安全（银行家算法） | 需要预知资源需求 |
| 死锁检测 | 事后 | 检测到死锁后恢复 | 需要额外开销 |

选择建议：
- 死锁概率低：鸵鸟策略（忽略）
- 资源充足：死锁预防
- 资源紧张：死锁避免
- 无法预知：死锁检测

## 内存管理

**物理地址 vs 虚拟地址**

| 对比项 | 物理地址 | 虚拟地址 |
|--------|----------|----------|
| 定义 | 内存的真实地址 | 程序使用的逻辑地址 |
| 范围 | 受物理内存限制 | 可以超过物理内存 |
| 唯一性 | 全局唯一 | 每个进程独立 |
| 转换 | 无需转换 | 需要通过页表转换 |

虚拟地址的优势：
- 进程隔离（每个进程有独立的地址空间）
- 内存保护（防止越界访问）
- 可以使用比物理内存更大的地址空间

**内部碎片 vs 外部碎片**

内部碎片：
- 已分配的内存块内部未使用的空间
- 如分配 4KB 的页，只用了 1KB，剩余 3KB 就是内部碎片
- 分页系统会产生
- 解决方法：减小分配单位

外部碎片：
- 未分配的内存块之间的小块空间
- 虽然总空间足够，但不连续，无法分配
- 分段系统会产生
- 解决方法：内存紧缩（compaction）

## 分页与分段

**分页 vs 分段**

| 对比项 | 分页 | 分段 |
|--------|------|------|
| 目的 | 实现虚拟内存 | 逻辑划分 |
| 大小 | 固定（如 4KB） | 可变 |
| 地址空间 | 一维 | 二维（段号 + 段内偏移） |
| 对程序员 | 透明 | 需要显式划分 |
| 碎片 | 内部碎片 | 外部碎片 |
| 共享 | 困难 | 容易 |

记忆技巧：
- 分页：物理划分，固定大小，透明
- 分段：逻辑划分，可变大小，显式

**页表 vs TLB**

页表（Page Table）：
- 存储在内存中
- 完整的虚拟地址到物理地址的映射
- 访问速度慢

TLB（Translation Lookaside Buffer）：
- 存储在 CPU 中的缓存
- 缓存最近使用的页表项
- 访问速度快

地址转换过程：
```
1. 查找 TLB
   ↓
2. TLB 命中？
   是 → 直接获取物理地址
   否 → 查找页表
   ↓
3. 查找页表
   ↓
4. 更新 TLB
   ↓
5. 获取物理地址
```

## 虚拟内存

**虚拟内存的作用**

主要作用：
1. 扩大地址空间：可以运行比物理内存大的程序
2. 进程隔离：每个进程有独立的地址空间
3. 内存保护：防止进程访问其他进程的内存
4. 共享内存：多个进程可以共享同一块物理内存

**页面置换算法对比**

| 算法 | 特点 | 优点 | 缺点 |
|------|------|------|------|
| FIFO | 先进先出 | 简单 | 可能淘汰常用页 |
| LRU | 最近最久未使用 | 性能好 | 实现复杂，开销大 |
| LFU | 最不经常使用 | 考虑访问频率 | 可能淘汰刚加载的页 |
| Clock | 时钟算法 | 折中方案 | 性能介于 FIFO 和 LRU 之间 |
| OPT | 最佳置换 | 理论最优 | 无法实现（需要预知未来） |

实际应用：
- Linux：LRU 的近似算法
- 大多数系统：Clock 算法或其变种

## 文件系统

**inode 包含什么**

包含：
- 文件类型和权限
- 所有者和组
- 文件大小
- 时间戳（atime, mtime, ctime）
- 硬链接计数
- 数据块指针

不包含：
- 文件名（存储在目录中）
- 文件内容（存储在数据块中）

**硬链接 vs 软链接**

| 特性 | 硬链接 | 软链接 |
|------|--------|--------|
| inode | 共享同一个 | 有自己的 inode |
| 数据 | 共享同一份 | 存储路径字符串 |
| 跨文件系统 | 不可以 | 可以 |
| 链接目录 | 不可以 | 可以 |
| 原文件删除 | 不影响 | 失效 |

记忆技巧：
- 硬链接：硬绑定，共享 inode，限制多
- 软链接：软引用，独立 inode，灵活

## I/O 模型

**五种 I/O 模型**

| I/O 模型 | 数据准备阶段 | 数据复制阶段 | 是否阻塞 | 是否同步 |
|----------|-------------|-------------|---------|---------|
| 阻塞 I/O | 阻塞 | 阻塞 | 是 | 是 |
| 非阻塞 I/O | 非阻塞（轮询） | 阻塞 | 部分阻塞 | 是 |
| I/O 多路复用 | 阻塞（等待就绪） | 阻塞 | 是 | 是 |
| 信号驱动 I/O | 非阻塞 | 阻塞 | 部分阻塞 | 是 |
| 异步 I/O | 非阻塞 | 非阻塞 | 否 | 否 |

关键区别：
- 前四种都是同步 I/O（数据复制阶段会阻塞）
- 只有异步 I/O 是真正的异步（整个过程都不阻塞）

**select vs poll vs epoll**

| 特性 | select | poll | epoll |
|------|--------|------|-------|
| 数量限制 | 1024 | 无 | 无 |
| 时间复杂度 | O(n) | O(n) | O(1) |
| 跨平台 | 是 | 是 | 否（仅 Linux） |
| 性能 | 低 | 中 | 高 |

选择建议：
- 跨平台：select 或 poll
- Linux 高性能：epoll
- 连接数少：三者都可以

**为什么 epoll 比 select 快**

原因：

1. 不需要遍历：select 需要遍历所有文件描述符，epoll 只返回就绪的文件描述符
2. 不需要复制：select 每次都要复制文件描述符集合，epoll 只在添加时复制一次
3. 数据结构：select 使用位图有数量限制，epoll 使用红黑树无数量限制
4. 回调机制：epoll 使用回调，文件描述符就绪时自动加入链表，select 需要轮询检查

## 易混淆概念总结

**进程 vs 线程 vs 协程**
- 进程：资源分配单位，重量级
- 线程：调度单位，轻量级
- 协程：用户态调度，超轻量级

**并发 vs 并行**
- 并发：交替执行（单核可以）
- 并行：同时执行（需要多核）

**同步 vs 异步**
- 同步：主动等待
- 异步：被动接收通知

**阻塞 vs 非阻塞**
- 阻塞：调用者等待
- 非阻塞：调用者立即返回

**上下文切换 vs 模式切换**
- 上下文切换：切换进程/线程
- 模式切换：切换特权级

**用户态 vs 内核态**
- 用户态：权限受限
- 内核态：最高权限

**互斥量 vs 信号量**
- 互斥量：二元，用于互斥
- 信号量：多元，用于同步

**分页 vs 分段**
- 分页：固定大小，透明
- 分段：可变大小，显式

**硬链接 vs 软链接**
- 硬链接：共享 inode
- 软链接：独立 inode

**物理地址 vs 虚拟地址**
- 物理地址：真实地址
- 虚拟地址：逻辑地址
