## 虚拟内存

程序想要连续的、大量的内存，然而物理内存并不一定总能连续的分配，也不一定足够大。虚拟内存的目的是为了让物理内存扩充成更大的逻辑内存，从而让程序获得更多的可用内存。

为了更好的管理内存，操作系统将内存抽象成地址空间。每个程序拥有自己的地址空间，这个地址空间被分割成多个块，每一块称为一页。这些页被映射到物理内存，但不需要映射到连续的物理内存，也不需要所有页都必须在物理内存中。当程序引用到不在物理内存中的页时，由硬件执行必要的映射，将缺失的部分装入物理内存并重新执行失败的指令。

从上面的描述中可以看出，虚拟内存允许程序不用将地址空间中的每一页都映射到物理内存，也就是说一个程序不需要全部调入内存就可以运行，这使得有限的内存运行大程序成为可能。例如有一台计算机可以产生 16 位地址，那么一个程序的地址空间范围是 0\~64K。该计算机只有 32KB 的物理内存，虚拟内存技术允许该计算机运行一个 64K 大小的程序。

### 分页系统地址映射

内存管理单元（MMU）管理着地址空间和物理内存的转换，其中的页表（Page table）存储着页（程序地址空间）和页框（物理内存空间）的映射表。

一个虚拟地址分成两个部分，一部分存储页面号，一部分存储偏移量。

下图的页表存放着 16 个页，这 16 个页需要用 4 个比特位来进行索引定位。例如对于虚拟地址（0010 000000000100），前 4 位是存储页面号 2，读取表项内容为（110 1），页表项最后一位表示是否存在于内存中，1 表示存在。后 12 位存储偏移量。这个页对应的页框的地址为 （110 000000000100）。

### 页面置换算法

在程序运行过程中，如果要访问的页面不在内存中，就发生缺页中断从而将该页调入内存中。此时如果内存已无空闲空间，系统必须从内存中调出一个页面到磁盘对换区中来腾出空间。页面置换算法和缓存淘汰策略类似，可以将内存看成磁盘的缓存。在缓存系统中，缓存的大小有限，当有新的缓存到达时，需要淘汰一部分已经存在的缓存，这样才有空间存放新的缓存数据。

页面置换算法的主要目标是使页面置换频率最低（也可以说缺页率最低）。

#### 最佳

> OPT, Optimal replacement algorithm

所选择的被换出的页面将是最长时间内不再被访问，通常可以保证获得最低的缺页率。是一种理论上的算法，因为无法知道一个页面多长时间不再被访问。

举例：一个系统为某进程分配了三个物理块，并有如下页面引用序列：

```html
7，0，1，2，0，3，0，4，2，3，0，3，2，1，2，0，1，7，0，1
```

开始运行时，先将 7, 0, 1 三个页面装入内存。当进程要访问页面 2 时，产生缺页中断，会将页面 7 换出，因为页面 7 再次被访问的时间最长。

#### 最近最久未使用

> LRU, Least Recently Used

虽然无法知道将来要使用的页面情况，但是可以知道过去使用页面的情况。LRU 将最近最久未使用的页面换出。为了实现 LRU，需要在内存中维护一个所有页面的链表。当一个页面被访问时，将这个页面移到链表表头。这样就能保证链表表尾的页面是最近最久未访问的。因为每次访问都需要更新链表，因此这种方式实现的 LRU 代价很高。

```html
4，7，0，7，1，0，1，2，1，2，6
```

#### 最近未使用

> NRU, Not Recently Used

每个页面都有两个状态位：R 与 M，当页面被访问时设置页面的 R=1，当页面被修改时设置 M=1。其中 R 位会定时被清零。可以将页面分成以下四类：

- R=0，M=0
- R=0，M=1
- R=1，M=0
- R=1，M=1

当发生缺页中断时，NRU 算法随机地从类编号最小的非空类中挑选一个页面将它换出。NRU 优先换出已经被修改的脏页面（R=0，M=1），而不是被频繁使用的干净页面（R=1，M=0）。

#### 先进先出

> FIFO, First In First Out

选择换出的页面是最先进入的页面。该算法会将那些经常被访问的页面换出，导致缺页率升高。

#### 第二次机会算法

FIFO 算法可能会把经常使用的页面置换出去，为了避免这一问题，对该算法做一个简单的修改：

当页面被访问 (读或写) 时设置该页面的 R 位为 1。需要替换的时候，检查最老页面的 R 位。如果 R 位是 0，那么这个页面既老又没有被使用，可以立刻置换掉；如果是 1，就将 R 位清 0，并把该页面放到链表的尾端，修改它的装入时间使它就像刚装入的一样，然后继续从链表的头部开始搜索。

#### 时钟

> Clock

第二次机会算法需要在链表中移动页面，降低了效率。时钟算法使用环形链表将页面连接起来，再使用一个指针指向最老的页面。

### 分段

虚拟内存采用的是分页技术，也就是将地址空间划分成固定大小的页，每一页再与内存进行映射。

下图为一个编译器在编译过程中建立的多个表，有 4 个表是动态增长的，如果使用分页系统的一维地址空间，动态增长的特点会导致覆盖问题的出现。

分段的做法是把每个表分成段，一个段构成一个独立的地址空间。每个段的长度可以不同，并且可以动态增长。

### 段页式

程序的地址空间划分成多个拥有独立地址空间的段，每个段上的地址空间划分成大小相同的页。这样既拥有分段系统的共享和保护，又拥有分页系统的虚拟内存功能。

- 对程序员的透明性：分页透明，但是分段需要程序员显式划分每个段。

- 地址空间的维度：分页是一维地址空间，分段是二维的。

- 大小是否可以改变：页的大小不可变，段的大小可以动态改变。

- 出现的原因：分页主要用于实现虚拟内存，从而获得更大的地址空间；分段主要是为了使程序和数据可以被划分为逻辑上独立的地址空间并且有助于共享和保护。

## 补充

### 虚拟内存的作用

虚拟内存允许程序不用将地址空间中的每一页都映射到物理内存，程序不需要全部调入内存就可以运行。这解决了几个问题：一是可以运行比物理内存大的程序，二是为每个进程提供独立的地址空间从而实现内存保护，三是多个进程可以共享同一物理内存（如共享库）。

### 分页和分段的选择

分页的主要目的是实现虚拟内存，页的大小固定，对程序员透明，会产生内部碎片。分段的主要目的是逻辑划分，段的大小可变，需要程序员显式划分，会产生外部碎片。现代操作系统主要使用分页，因为分页更适合实现虚拟内存，而分段的逻辑划分功能可以通过其他方式实现。

### 页面置换算法的实际应用

理论上最优的是 OPT 算法，但它需要预知未来，无法实现。LRU 算法性能接近最优，但实现开销大，需要维护页面访问顺序。实际系统通常使用 Clock 算法，它是 LRU 的近似，性能较好且实现简单。Linux 使用的就是 Clock 算法的变种。

### 多级页表的权衡

单级页表简单但占用空间大，每个进程都需要完整的页表。多级页表按需分配，只为实际使用的地址空间分配页表，节省空间，但访问内存的次数增加。TLB 的存在缓解了多级页表带来的性能损失，因为大部分页表访问都能命中 TLB。

### 缺页中断的开销

缺页中断的主要开销在于磁盘 I/O，这是毫秒级的操作，比内存访问慢几个数量级。因此减少缺页率对系统性能至关重要。提高程序的局部性（时间局部性和空间局部性）可以有效减少缺页率。

### 写时复制的应用

写时复制在 fork() 系统调用中非常有用。传统的 fork() 需要复制父进程的整个地址空间，开销很大。使用写时复制后，父子进程共享物理内存，只有在写入时才复制相应的页面。由于很多情况下子进程会立即调用 exec() 替换地址空间，写时复制避免了不必要的复制，大大提高了 fork() 的效率。

### Q1：什么是虚拟内存？为什么需要虚拟内存？

**虚拟内存的定义：**
将物理内存抽象成更大的逻辑内存，让程序认为自己拥有连续的、足够大的内存空间。

**为什么需要虚拟内存？**

| 原因 | 说明 | 好处 |
|------|------|------|
| **内存不足** | 物理内存有限，程序可能很大 | 可以运行比物理内存大的程序 |
| **内存碎片** | 物理内存分配不连续 | 提供连续的虚拟地址空间 |
| **内存保护** | 进程可能访问其他进程内存 | 每个进程独立地址空间 |
| **内存共享** | 多个进程需要共享代码/数据 | 可以映射到同一物理内存 |

**核心机制：**
1. 地址空间分页
2. 页表映射（虚拟地址 → 物理地址）
3. 按需调页（需要时才加载）
4. 页面置换（内存不足时换出）

**记忆口诀：**
虚拟内存 = 让程序以为内存很大 + 实际按需分配

### Q2：分页和分段的区别？

读到“Q2：分页和分段的区别？”时，先别急着记结论。更有效的方式是先弄清它在“计算机操作系统 - 内存管理”这一张卡片里到底解决什么问题，这样后面的分点和对比才不至于显得太散。

| 对比项 | 分页（Paging） | 分段（Segmentation） |
|--------|---------------|---------------------|
| **目的** | 实现虚拟内存 | 逻辑划分，便于共享和保护 |
| **大小** | 固定大小（如 4KB） | 可变大小 |
| **地址空间** | 一维（页号 + 偏移） | 二维（段号 + 段内偏移） |
| **透明性** | 对程序员透明 | 需要程序员显式划分 |
| **碎片** | 内部碎片 | 外部碎片 |
| **共享** | 以页为单位 | 以段为单位（更灵活） |

**内部碎片 vs 外部碎片：**

- **内部碎片**：分配的内存比需要的多（分页）
  - 例如：需要 3.5KB，分配 4KB，浪费 0.5KB
  
- **外部碎片**：有足够内存但不连续（分段）
  - 例如：有 10KB 空闲，但分散在多个小块中

**记忆技巧：**
- 分页：切蛋糕，每块一样大
- 分段：切西瓜，按需要切

### Q3：页面置换算法有哪些？

| 算法 | 策略 | 优点 | 缺点 | 实现难度 |
|------|------|------|------|---------|
| **最佳（OPT）** | 置换最长时间不用的页 | 缺页率最低 | 无法实现（需要预知未来） | 理论算法 |
| **LRU** | 置换最近最久未使用的页 | 性能好，接近最佳 | 实现开销大 | 困难 |
| **FIFO** | 置换最先进入的页 | 实现简单 | 可能置换常用页 | 简单 |
| **Clock** | FIFO + 第二次机会 | 性能好，开销小 | 比 LRU 稍差 | 中等 |
| **NRU** | 按 R/M 位分类置换 | 实现简单，性能尚可 | 不够精确 | 简单 |

**算法对比：**

```
性能：OPT > LRU > Clock > NRU > FIFO
实现难度：FIFO < NRU < Clock < LRU < OPT（不可实现）
```

**LRU 实现方式：**
1. 链表：每次访问移到表头（开销大）
2. 计数器：记录访问时间（需要额外空间）
3. 栈：访问时压栈（实现复杂）

**实际应用：**
- Linux：Clock 算法（LRU 近似）
- 数据库：LRU 变种
- Redis：LRU 近似算法

### Q4：什么是缺页中断？

**定义：**
访问的页面不在物理内存中，触发中断，将页面从磁盘加载到内存。

**缺页中断处理流程：**

```
1. 访问虚拟地址
   ↓
2. 查页表，发现页面不在内存（有效位 = 0）
   ↓
3. 触发缺页中断
   ↓
4. 操作系统接管
   ↓
5. 检查地址是否合法
   ↓
6. 选择一个空闲页框（如果没有，执行页面置换）
   ↓
7. 从磁盘读取页面到内存
   ↓
8. 更新页表
   ↓
9. 重新执行访问指令
```

**缺页中断的开销：**
- 保存现场
- 查找页面位置
- 磁盘 I/O（最耗时，毫秒级）
- 更新页表
- 恢复现场

**如何减少缺页率？**
1. 增加物理内存
2. 使用更好的页面置换算法
3. 提高局部性（时间局部性、空间局部性）
4. 预取（提前加载可能用到的页）

### Q5：什么是多级页表？为什么需要多级页表？

**问题背景：**

假设：
- 32 位地址空间 = 4GB
- 页大小 = 4KB
- 页表项大小 = 4B

需要的页表项数量 = 4GB / 4KB = 1M 个
页表大小 = 1M × 4B = 4MB

**问题：**
每个进程都需要 4MB 的页表，太大了！

**多级页表的解决方案：**

将页表分成多级，只为实际使用的地址空间分配页表。

**二级页表示例：**

```
虚拟地址（32位）：
┌──────────┬──────────┬──────────┐
│ 一级索引 │ 二级索引 │  偏移量  │
│  10位    │  10位    │  12位    │
└──────────┴──────────┴──────────┘
```

**优点：**
1. 节省空间：只为使用的地址空间分配页表
2. 灵活：可以按需分配

**缺点：**
1. 访问内存次数增加（多次查表）
2. 实现复杂

**实际应用：**
- x86-32：二级页表
- x86-64：四级页表

### Q6：什么是 TLB？

这一节看起来往往容易变成一串结论，其实先抓住“Q6：什么是 TLB？”在计算机操作系统里的作用会更稳。后面的内容可以理解成围绕这个问题展开的几种常见判断和做法。

**TLB（Translation Lookaside Buffer）：**
- 中文：快表、页表缓存
- 作用：缓存最近使用的页表项
- 位置：CPU 内部

**为什么需要 TLB？**

没有 TLB：
```
访问一个数据 = 查页表（1次内存访问）+ 访问数据（1次内存访问）= 2次
```

有 TLB：
```
TLB 命中：访问一个数据 = 1次内存访问
TLB 未命中：访问一个数据 = 2次内存访问
```

**TLB 命中率：**
- 典型命中率：95% - 99%
- 命中时间：1 个时钟周期
- 未命中时间：10 - 100 个时钟周期

**有效访问时间计算：**

```
EAT = TLB命中率 × TLB访问时间 + TLB未命中率 × (TLB访问时间 + 页表访问时间 + 内存访问时间)
```

**示例：**
- TLB 访问时间 = 1ns
- 内存访问时间 = 100ns
- TLB 命中率 = 98%

```
EAT = 0.98 × 1 + 0.02 × (1 + 100 + 100)
    = 0.98 + 0.02 × 201
    = 0.98 + 4.02
    = 5ns
```

**TLB 刷新时机：**
- 进程切换（切换地址空间）
- 修改页表
- 显式刷新指令

### Q7：什么是内存碎片？如何解决？

读到“Q7：什么是内存碎片？如何解决？”时，先别急着记结论。更有效的方式是先弄清它在“计算机操作系统 - 内存管理”这一张卡片里到底解决什么问题，这样后面的分点和对比才不至于显得太散。

**内部碎片：**
- 定义：分配的内存比需要的多
- 原因：固定大小分配（分页）
- 示例：需要 3.5KB，分配 4KB，浪费 0.5KB

**外部碎片：**
- 定义：有足够空闲内存，但不连续
- 原因：可变大小分配（分段）
- 示例：需要 10KB，有 12KB 空闲，但分散在 3 个 4KB 块中

**对比：**

| 类型 | 产生原因 | 影响 | 解决方法 |
|------|---------|------|---------|
| **内部碎片** | 固定大小分配 | 浪费内存 | 减小页大小 |
| **外部碎片** | 可变大小分配 | 无法分配 | 内存紧缩、分页 |

**解决方法：**

1. **内部碎片：**
   - 减小页大小（但会增加页表大小）
   - 使用多种页大小（如 4KB + 2MB + 1GB）

2. **外部碎片：**
   - 内存紧缩（移动进程，合并空闲区）
   - 分页（将分段改为分页）
   - 伙伴系统（按 2 的幂次分配）

### Q8：什么是写时复制（Copy-on-Write）？

**定义：**
多个进程共享同一物理内存，只有在写入时才复制。

**应用场景：**

**1. fork() 系统调用：**

传统方式：
```
父进程 fork() → 复制整个地址空间 → 子进程
```

写时复制：
```
父进程 fork() → 共享物理内存（只读） → 子进程
                ↓
            某个进程写入
                ↓
            复制该页面
```

**优点：**
- 节省内存
- 加快 fork() 速度
- 很多情况下子进程会立即 exec()，不需要复制

**2. 共享库：**
多个进程共享同一个库的代码段。

**实现机制：**
1. 页表项标记为只读
2. 写入时触发缺页中断
3. 操作系统复制页面
4. 更新页表，标记为可写

### Q9：什么是交换空间（Swap）？

**定义：**
磁盘上的一块区域，用于存放从内存换出的页面。

**作用：**
1. 扩展内存：让系统可以运行更多程序
2. 休眠支持：将内存内容保存到磁盘

**工作流程：**

```
内存不足
  ↓
选择页面换出（页面置换算法）
  ↓
写入交换空间
  ↓
释放物理内存
  ↓
需要时再换入
```

**性能影响：**
- 磁盘 I/O 很慢（毫秒级 vs 纳秒级）
- 频繁交换会导致"抖动"（Thrashing）

**抖动（Thrashing）：**
- 定义：系统大部分时间在换页，而不是执行程序
- 原因：内存不足，频繁缺页
- 解决：增加内存、减少进程数、改进算法

### Q10：段页式内存管理是什么？

**定义：**
结合分段和分页的优点。

**结构：**
```
程序
  ↓ 分段
段（代码段、数据段、栈段）
  ↓ 分页
页（固定大小）
  ↓ 映射
物理内存
```

**地址转换：**

```
虚拟地址：
┌────────┬────────┬────────┐
│ 段号   │ 页号   │ 偏移量 │
└────────┴────────┴────────┘
    ↓        ↓        ↓
  段表    页表    物理地址
```

**优点：**
1. 分段的优点：逻辑划分、共享、保护
2. 分页的优点：固定大小、无外部碎片、虚拟内存

**缺点：**
1. 实现复杂
2. 地址转换需要访问两次表（段表 + 页表）

**实际应用：**
- Intel x86：段页式（但现代操作系统很少用分段）
- Linux：主要用分页，分段功能弱化

### 虚拟内存

这一节看起来往往容易变成一串结论，其实先抓住“虚拟内存”在计算机操作系统里的作用会更稳。后面的内容可以理解成围绕这个问题展开的几种常见判断和做法。

**核心概念：**
- 虚拟地址空间 → 页表 → 物理内存
- 按需调页：需要时才加载
- 页面置换：内存不足时换出

**关键机制：**
1. 地址转换（MMU + 页表）
2. 缺页中断（页面不在内存）
3. 页面置换（选择换出哪个页）

### 分页 vs 分段

这一节看起来往往容易变成一串结论，其实先抓住“分页 vs 分段”在计算机操作系统里的作用会更稳。后面的内容可以理解成围绕这个问题展开的几种常见判断和做法。

**记忆口诀：**
- 分页：固定大小，透明，内部碎片
- 分段：可变大小，显式，外部碎片

**选择建议：**
- 实现虚拟内存：分页
- 逻辑划分、共享：分段
- 现代系统：主要用分页

### 页面置换算法

**性能排序：**
OPT > LRU > Clock > NRU > FIFO

**实际应用：**
- 理论最优：OPT（不可实现）
- 性能最好：LRU（开销大）
- 实际常用：Clock（LRU 近似）

**记忆技巧：**
- OPT：预知未来（理论）
- LRU：看过去（最近最久未使用）
- FIFO：看顺序（先进先出）
- Clock：FIFO + 第二次机会

### 多级页表

读到“多级页表”时，先别急着记结论。更有效的方式是先弄清它在“计算机操作系统 - 内存管理”这一张卡片里到底解决什么问题，这样后面的分点和对比才不至于显得太散。

**为什么需要：**
- 单级页表太大（每个进程 4MB）
- 多级页表按需分配（节省空间）

**代价：**
- 访问内存次数增加
- 实现复杂

**优化：**
- TLB 缓存（减少页表访问）

### TLB

**作用：**
缓存页表项，加速地址转换。

**效果：**
- 命中率 95% - 99%
- 命中时：1 次内存访问
- 未命中：2 次内存访问（多级页表更多）

### 内存碎片

这一节看起来往往容易变成一串结论，其实先抓住“内存碎片”在计算机操作系统里的作用会更稳。后面的内容可以理解成围绕这个问题展开的几种常见判断和做法。

**内部碎片：**
- 分配的比需要的多
- 分页产生
- 解决：减小页大小

**外部碎片：**
- 有内存但不连续
- 分段产生
- 解决：内存紧缩、分页

## 易错点

### 1. 虚拟内存 ≠ 交换空间

**错误认识：**
虚拟内存就是硬盘上的交换空间。

**正确理解：**
- 虚拟内存：一种内存管理技术
- 交换空间：虚拟内存的一部分实现

虚拟内存包括：
- 地址空间抽象
- 页表映射
- 按需调页
- 页面置换
- 交换空间（可选）

### 2. 页表大小计算

**常见错误：**
混淆页表项数量和页表大小。

**正确计算：**
```
页表项数量 = 地址空间大小 / 页大小
页表大小 = 页表项数量 × 页表项大小
```

**示例：**
- 32 位地址空间 = 4GB
- 页大小 = 4KB = 2^12
- 页表项大小 = 4B

```
页表项数量 = 2^32 / 2^12 = 2^20 = 1M
页表大小 = 2^20 × 4B = 4MB
```

### 3. 缺页中断 vs 一般中断

**区别：**

| 类型 | 缺页中断 | 一般中断 |
|------|---------|---------|
| **类型** | 内中断（异常） | 外中断 |
| **是否可恢复** | 是（重新执行指令） | 否 |
| **处理时间** | 长（毫秒级，涉及磁盘 I/O） | 短（微秒级） |

### 4. LRU 实现误区

**错误认识：**
LRU 很容易实现。

**正确理解：**
精确的 LRU 实现开销很大：
- 链表方式：每次访问都要移动节点
- 计数器方式：需要大量额外空间
- 栈方式：实现复杂

实际系统使用 LRU 近似算法（如 Clock）。

### 5. 分页 vs 分段的混淆

**常见错误：**
认为分页和分段是互斥的。

**正确理解：**
- 可以只用分页（现代系统）
- 可以只用分段（早期系统）
- 可以结合使用（段页式）

现代操作系统主要用分页，分段功能弱化。

### 6. 页大小的选择

**不是越小越好：**

页太小：
- 内部碎片少 正确
- 页表太大 错误
- 缺页中断频繁 错误

页太大：
- 页表小 正确
- 内部碎片多 错误
- 局部性差 错误

**典型选择：**
- 4KB（最常见）
- 2MB（大页）
- 1GB（巨页）

### 7. 写时复制的触发条件

**错误认识：**
fork() 后立即复制内存。

**正确理解：**
fork() 后共享内存，只有在写入时才复制。

**注意：**
- 读操作：不触发复制
- 写操作：触发复制
- exec()：不需要复制（替换地址空间）
